## 存放位置约定

| 类别 | 存放方案 | 适用场景 |
|------|---------|---------|
| **项目专用 key** | `brain/.tools/.env.{用途}` | graphify、未来项目脚本类 |
| **系统级凭证** | macOS Keychain | GitHub PAT、iCloud、邮箱 |
| **多设备共享** | 1Password / Bitwarden（可选） | 跨机器共享、团队协作 |
| **临时调试** | shell session env，**禁止**写入文件 | 一次性测试 |

### 绝对禁止

- ❌ 明文 key 写进 git commit / log / chat 历史 / 任何 markdown
- ❌ key 复制粘贴到对话框、Slack、邮件
- ❌ 共用同一把 key 跨多个项目（粒度按"服务 × 用途"切分）

### E-001 · DeepSeek API Key 暴露（2026-05-14）

- **凭证**：清单 #1，`sk-3fe87...3733`（完整 key 见 `.env.graphify`）
- **暴露面**：
  - Cowork 对话历史（一次性，对话结束后不可恢复）
  - 截断前缀（仅前 8 位 + 省略号）出现在 `logs/2026-05-14-graphify-接通-DVA-渊图.md:65`，已 commit `1112b41` push 到 GitHub remote
- **风险评估**：
  - DeepSeek 账号是预付费模式，余额低
  - 截断前缀（8 位）理论上无法暴力恢复完整 key
  - 攻击者即使拿到 key 最大损失 = 当前余额
- **处置决策**（Doctor · 2026-05-14）：
  - **接受风险，不轮换**，理由：预付费账户余额低，损失上限可控
  - 不清理 GitHub remote 上的截断前缀（前 8 位无害）
- **后续触发条件**（任一满足则必须立即轮换）：
  - DeepSeek 改成账单制 / 接入 Anthropic 主账户
  - 余额充值至 ¥100+
  - 发现异常调用（非本机 IP、非 graphify 模式的请求）
- **复盘教训**：
  - 凡是要在对话里贴 key，先用 `.env` 文件 + `source` 引用，避免明文进对话
  - 写 log 时 key 字段必须脱敏成 `sk-***`，不留前缀
