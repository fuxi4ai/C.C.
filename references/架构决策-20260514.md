## D6 · 对标 OpenViking 的记忆系统取舍（2026-05-23）

**背景**：看了字节开源 Agent 记忆系统 OpenViking 的设计概览（详见 [[OpenViking-Agent记忆系统]]），对标 brain，决定借鉴哪些、明确不做哪些。

**决策：借五项，拒三项**

借鉴（已落地）：
- A1 分级加载标准化（L0/L1/L2，借三层加载思想）
- A2 经验库（borrow agent 自我记忆 cases/patterns/tools/strategies，补 brain 只记错误的缺口）
- A3 写入时级联复查（用现成 backlinks 缓解级联更新）
- B1 brain 原生记忆固化 skill（borrow session-commit + consolidation，双重门控）
- B2 半自动 session commit（brain-save Step 3.5，borrow MemoryExtractor 但人在环）

明确不做（C 档）：
- ❌ 向量/语义递归检索（维持 D2「纯文件 + ripgrep」；规模小 + 关键词 anchor 已够，向量徒增基建/不确定性/故障面）
- ❌ 每条笔记三文件（L0/L1/L2 物理拆分）——项目级两层已拿到大部分收益，下沉到每条笔记维护成本过高
- ❌ 用单一工作记忆替换不可变会话日志——日志是工作史，价值在审计/回溯，不替换

**理由**：brain 是"人在环 + 数百条笔记 + Obsidian 手工策展"模型，与 OpenViking"自主 agent + 海量记忆 + 省 token"目标不同；机械照搬会过度工程。规模涨 10 倍再重估向量。

**影响**：见 [[CLAUDE]] 新增三节（分级加载约定 / 经验沉淀规则 / 级联复查规则）+ [[经验库]] + brain-save v2.1 + brain-consolidate skill。
