# Graphify 集成

> 把代码库的结构转成 Obsidian 可视化的知识图谱。

## 设计

| | 路径 |
|---|---|
| 工具 | `graphify extract <src> --out <dst>` (v0.7.16, npm 全局包) |
| 入口脚本 | `brain/.tools/run-graphify.sh` |
| 输出 | `brain/graphify/{项目名}/`（Obsidian 子 vault 形态） |
| 触发 | 手动 `bash run-graphify.sh <项目>` 或 `--all` |

## 建议跑图谱的项目

| 项目 | 源 | 理由 |
|------|----|------|
| DVA | `Claude/Projects/DVA/` | 多脚本互调，结构复杂值得视化 |
| 渊图 | `Database/行业研究/` | 121 篇研报，链接密集 |
| 龙鱼五力 | `Claude/Projects/龙鱼五力/` | RULES.md + 多个分析脚本 |
| O MY HTML | `Claude/Projects/O MY HTML/` | 前端组件库，节点关系丰富 |

**不建议**（纯文档类）：海螺姑娘、政治经济学、Optical communication、司南

## 操作

### 前置

```bash
# 1. graphify 在 PATH
export PATH="$PATH:$HOME/.local/bin"
graphify --help    # 应列出 install/extract/path/explain 等命令

# 2. LLM API key（extract 需要语义分析，必须）
export ANTHROPIC_API_KEY=sk-ant-...
# 或：
export KIMI_API_KEY=sk-...
```

### 首次

```bash
# 列待跑项目
bash ~/Documents/Claude/brain/.tools/run-graphify.sh

# 跑一个（首次：5-30 分钟，看代码量）
bash ~/Documents/Claude/brain/.tools/run-graphify.sh DVA

# 跑全部
bash ~/Documents/Claude/brain/.tools/run-graphify.sh --all
```

### 产物结构

```
brain/graphify/{项目名}/graphify-out/
├── graph.json          # 节点 + 边的 JSON 图
├── report.md           # 聚类分析报告
└── GRAPH_TREE.html     # D3 可视化
```

### 增量更新（无需 LLM）

```bash
cd brain/graphify/{项目名}
graphify update <项目源码路径>
```

### 查看图谱

- HTML 可视化：双击 `brain/graphify/{项目}/graphify-out/GRAPH_TREE.html`
- 查询：`graphify query "<问题>" --graph brain/graphify/{项目}/graphify-out/graph.json`
- 解释节点：`graphify explain "节点名" --graph ...`
- 最短路径：`graphify path "A" "B" --graph ...`

## 与 brain 的协作约定

写入 `brain/CLAUDE.md` 已有规则：
> 有图谱的项目，CC 应优先：
> 1. 查询 graphify/{项目}/graph.json 理解代码结构
> 2. 查询 vault 了解决策和上下文
> 3. 最后才直接读源代码

## GOTCHAS

- 大代码库首次跑可能耗时 1-2 分钟
- graphify 缓存在 `graphify/{项目}/cache/`，已在 .gitignore 忽略
- 若 graphify 报 "node version too old"，先升级 Node 到 ≥ 18
