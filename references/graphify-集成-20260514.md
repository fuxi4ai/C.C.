# Graphify 集成

> 把代码库的结构转成 Obsidian 可视化的知识图谱。

## 设计

| | 路径 |
|---|---|
| 工具 | `graphify` (v0.7.16, npm 全局包) |
| 入口脚本 | `brain/.tools/run-graphify.sh` |
| 输出 | `brain/graphify/{项目名}/`（Obsidian 子 vault 形态） |
| 触发 | 手动 `bash run-graphify.sh <项目>` 或 `--all` |

## 建议跑图谱的项目

| 项目 | 源 | 理由 |
|------|----|------|
| DVA | `Claude/Projects/DVA/` | 多脚本互调，结构复杂值得视化 |
| 渊图 | `Database/行业研究/` | 121 篇研报，链接密集 |
| 龙鱼五力 | `Claude/Projects/龙鱼五力/` | RULES.md + 多个分析脚本 |
| O MY HTML | `Claude/Projects/O MY HTML/` | 前端组件库，节点关系丰富 |

**不建议**（纯文档类）：海螺姑娘、政治经济学、Optical communication、司南

## 操作

### 首次

```bash
# 确保 graphify 在 PATH
export PATH="$PATH:$HOME/.local/bin"
graphify --version    # 应显示 v0.7.16

# 列待跑项目
bash ~/Documents/Claude/brain/.tools/run-graphify.sh

# 跑一个
bash ~/Documents/Claude/brain/.tools/run-graphify.sh DVA

# 跑全部
bash ~/Documents/Claude/brain/.tools/run-graphify.sh --all
```

### 在 Obsidian 中打开

`brain/graphify/{项目名}/` 是一个独立 vault。可以：
- 在 brain 主 vault 中开"vault 切换"切到子 vault
- 或在 Obsidian 直接 open new vault → 指向 `brain/graphify/DVA/`

### 增量更新

代码改动后重跑 `run-graphify.sh <项目>` 即可，graphify 会增量重建。

## 与 brain 的协作约定

写入 `brain/CLAUDE.md` 已有规则：
> 有图谱的项目，CC 应优先：
> 1. 查询 graphify/{项目}/graph.json 理解代码结构
> 2. 查询 vault 了解决策和上下文
> 3. 最后才直接读源代码

## GOTCHAS

- 大代码库首次跑可能耗时 1-2 分钟
- graphify 缓存在 `graphify/{项目}/cache/`，已在 .gitignore 忽略
- 若 graphify 报 "node version too old"，先升级 Node 到 ≥ 18
