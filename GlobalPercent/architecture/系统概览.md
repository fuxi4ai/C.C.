# GlobalPercent · 系统概览

> 全球宏观情绪温度计 v2。CC 进入该项目时第一篇要读的文件。
> 2026-06-29 从「游离在记忆库外」捞回 brain（早期 CLI 所建，未登记、无日志，故 /resume 一度搜不到）。

## 项目使命

拿 Polymarket + Kalshi 两个预测市场的**公开只读**数据，把每个合约价格读成概率，按宏观模块归类，算出一个**定义清晰、可复现的 0–100 GlobalPercent 指数**——原始上游项目（`simonlin1212/globalpercent`）叫这名却从没真算出任何百分比，只做了分类+列表。Doctor 先隔离审查上游（确认无后门/无 hook），再彻底重构为可独立 clone-and-run 的应用。

## 定位铁律（写进代码 docstring）

**它量的是「全市场对宏观事件有多激烈 / 分歧」的注意力-张力温度，不是涨跌买卖信号，也不是任何单一事件的概率。** 预测市场散户多数亏损，仅供研究叠加层使用。

## 指数构成

三个透明的、资金加权的分量，每个 0–100：

- **Tension（张力）**：成交量加权的分布熵。高 = 人群分裂 / 结果胶着（不确定性集中在 50/50 附近）。
- **Momentum（动量）**：成交量加权的 24h 概率位移大小，`|Δ| / momentum_ref(0.10)` 截顶到 1。高 = 在快速重定价。
- **Attention（注意力）**：该模块资金占最热模块的份额。

```
模块温度 = wT·Tension + wM·Momentum + wA·Attention   （默认 0.5 / 0.3 / 0.2）
GlobalPercent（headline）= CORE 模块的成交量加权温度
                         → 攒满 30 个历史点后，再表达为「相对自身历史的百分位」
                         → 不足 30 点时报原始值并标 calibrating
```

CORE 模块（喂总指数）：货币政策 / 宏观经济 / 地缘政治 / 政治选举 / 股指大宗 / AI科技。
REFERENCE 模块（仅展示、不进总指数）：加密 / 体育 / 娱乐 / 其他。

## 模块构成（代码）

- `app/index.py` — ★指数数学（纯函数、可测）：三分量 + CORE 加权 headline + 百分位校准注入点。
- `app/normalize.py` — overround 归一化（sum→1）+ favorite-longshot 去偏（默认关）+ 归一化熵。
- `app/taxonomy.py` — ★唯一分类器：关键词优先、native category 兜底；模块/cluster cap。
- `app/sources/{base,polymarket,kalshi}.py` — 统一 Market schema + 各源整形（单一整形路径）。
- `app/aggregator.py` — fetch→cap→index→snapshot→publish 管道 + 自适应自动刷新循环。
- `app/store.py` — SQLite 指数历史 + JSON 快照 + asyncio 锁（替原模块级全局变量）。
- `app/{api,main}.py` — FastAPI 路由 + 托管单文件 ECharts 仪表盘。
- `app/{translate,publish,config}.py` — 可选中文翻译（默认关、零密钥可跑）/ 可选快照发布（文件/gist）/ env 驱动配置。
- `web/{index,viewer}.html` — 单文件仪表盘 + 独立查看器（读已发布快照、不依赖后端）。
- `deploy/com.globalpercent.plist` — macOS launchd 常驻模板。
