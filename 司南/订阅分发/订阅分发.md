# 订阅分发

> 司南子项目：把内部研究产物转化为**可安全示人、可持续送达读者**的订阅内容线。核心命题＝保密（算法/库/代号零泄漏）× 可信（脱敏信源披露）× 易懂（非专业读者）× 自包含（脱库单文件、断网可读、移动端优先）。

## 通用范式

- [[分发版构建范式]]（permanent）——脱敏三件套 / 单文件内嵌 / QA 断言 / 独立定时任务 + G-X51 防悬挂前置。新领域子线一律从此范式起步。

## 边界

- 本子项目管"分发产线与产物"；**不管**渠道运营（公众号/邮件列表/托管）与商业化——那是将来另立的事。
- 增量铁律：分发线永远只读内部产物，绝不反向改动任何内部产线。
