# 司南 · 项目护航方法论增量（2026-05-14 → 2026-07-02）

> v1.0 只有「四大护法文档 + 5 条执行原则」。此后两个月，brain 各项目实践长出六大主题的增量。本笔记是归纳层：每条给一句话方法论 + 出处指针（G-X 编号见 [[通用教训]]，EXP 见 [[经验库]]）。已按「难复得三问」过滤，只收可跨项目复用的。

## 一、治理与验收：人在环的两权分立

1. **propose-then-confirm**（2026-05-23 全局）：重大改动先提案→出具体改动清单/逐行 diff→Doctor 批准→才执行。「批准做某事」≠「批准具体怎么改」。两段式 diff 审批已成标准动作（渊图 patch dry-run、烛照 dim4 回溯 27 行 diff）。见 [[Doctor协作偏好]]。
2. **打勾权与起草权分离**（G-X4 + G-X10）：完成幻觉是 LLM 结构性盲区——所有自检都是"CC 用 CC 的标准评 CC"。故验收打 ✓ 权 Doctor 独占；起草权可下放（brain-prd：多轮逼问对齐→CC 起草 PRD 含可验证交付标准→Doctor 审改批）。自动化不豁免：确定性公式自动执行 ≠ CC 自动打 ✓（G-X23）。
3. **独立审查者范式**：审查者≠执行者，且不喂结论。三种成型形态——① 日常互审（句芒每日审九儿入库，只报不改）；② 三路独立 subagent 同任务对账 + 分歧点二次重点核查（渊图 dedup：抓 12 组真重复、驳回 1 误并，单路必错）；③ 多路只读子代理分维度审计（EXP-20260613-001）。审查结论的「根因」也要回源码验证（G-X11）。
4. **双重门控**：高危流程 skill 只定义流程、绝不自动跑；每次先产出「拟改动 diff」过目后才落盘（brain-consolidate 范式）。

## 二、数据真实性与来源治理

5. **数据真实性铁律**（2026-06-07 · 跨项目最高优先级）：宁可缺数不可假数。缺口用 `待核验` 显式标注；占位/样本数据绝不入库、绝不进交付物；取数受阻报告缺失交回 Doctor，不硬编码"先跑起来"。见 [[中文表述语域规范]] 同级的 brain/CLAUDE.md 铁律节。
6. **数据随源走**：每个数值带 `source + asof + credibility(P0/P1/P2)`；检索顺序 渊图→官方/权威站→一般 web，不要一上来就 web 搜。
7. **先核实再 promote + 克制挂接**（G-X18）：外部语料先入「视角层」（tracked），核实通过才候选，promote 进 canonical 须 Doctor 显式。带数字的断言自算量级核对（"特高压含铝 30 吨/3km"自算≈97 吨识破）；"独家/锁死"类找不到一手条款就标作者推断。
8. **按时效性分库**（G-X30）：高时效（行情/价格）与慢变结构知识分库；价格走独立价格层，不污染图谱。
9. **口径冲突先比时效与粒度**（渊图 FIX-20260626-001）：新旧口径矛盾多为「粒度+时效错配」非真矛盾——比 data_vintage、分粒度精修、加口径演进注记，不盲删旧述（投知君君"设备与两长"口径演进同法）。
10. **命名规范铁律**（NAMING-20260628-001 + G-X22）：易混实体 id 用可唯一辨识全称，禁裸拼音简称；配自动检测脚本进 batch 后校验。张冠李戴修正须 name/aliases/边/desc 全字段同改（FIX-20260623-001）。

## 三、写入安全与守恒：防数据损毁

11. **safe-wrapper 范式**：高危写操作包一层安全壳——自动备份 + 格式闸（错输入拒收）+ 强制输出位（防漏 --output）+ 落盘后读盘断言只增不减（kg_merge_safe）。2026-07-02 推广到 DVA：report-write-guard（`totalX 从 N>0 写成 0 一律拒绝` 守恒断言 + 补库模式禁写分析产物）。
12. **快照基线**：高价值产物生成成功后立即快照 `_snapshot_YYYYMMDD/`；改库前备份 `bak_YYYYMMDD_<事由>`。事故实证：update-all 曾把老毛聊交易全部分析产物静默清零（DVA BUG-20260702-002）。
13. **读盘核验铁律**：工具回执/合并日志不可信（Δ+0 幂等噪音、"已落盘"假象 G-X8、dry-run 动作标签≠真合入）——落盘后必重读盘断言目标数据真在（NOTE-20260617-001、NOTE-20260628-001）。
14. **深合并防薄冲富**（G-X35）：merge/patch 工具对 properties/_meta 类累加 dict 用子键并集深合并，非整块替换；改主链前先补回归单测。
15. **改动前的两向检查**：纵向——改生成物白改，先改源（G-X27）；横向——改共享 tag/字段前 grep 枚举所有下游消费者，牵连大走派生 fallback 不动本体（G-X29）；沉淀事实改动跑级联复查 check-cascade（wikilink 反链）。

## 四、沙箱/环境边界：FUSE 家族纪律

16. **FUSE 写限制家族**（G-X2/7/26/34/37/48 同源）：凡需独占锁/unlink/rename 的写（git 写、zip、clone、sqlite WAL 切换/checkpoint/VACUUM、rm 覆盖）沙箱挂载盘做不了——一律 `/tmp` 完成再 cp 回，或构造命令交 Doctor 终端。sqlite 标准姿势：/tmp 副本写好→整库放回（EXP-20260622-001）。
17. **平铺挂载路径纪律**（G-X45 + 渊图 G020/G021）：gateway 沙箱把目录平铺到 `mnt/{名}`，脚本靠 `parents[N]`/向上找 Documents 必算错——用 env-var 覆盖或软链重建，路径落盘前 assert 绝对前缀 fail-fast（EXP-20260617-001-P）。定时任务 SKILL 须写明挂载前置 + 挂载失败即报告退出。
18. **断言环境事实前先实测**：文件系统布局先挂载实地核（G-X39）；库内既有 GOTCHAS/日志先 grep 再下"连不通/做不到"结论（G-X31）；规则文件里过时的"做不到"是待验证假设非现行约束（G-X41）。

## 五、自动化任务护航（2026-06~07 新主题）

19. **链路两半互设哨兵**：上下游定时任务（如抓取半/提炼半）各自检测对方停摆——下游查源数据 mtime 超阈值即在报告中提示并附补跑命令；谁断了都能被对面看见。
20. **看门狗设计三原则**（INFRA-20260702-001 事故沉淀）：① 停跑判据用「产出目录/DB 有无新写入」而非「日志增长」——进度条缓冲让日志静止≠卡死；② kill 前核对锁内 pid 是否同一实例——pkill 按模式匹配会误杀新起的健康进程；③ wrapper 加 trap，中断时回写锁状态，防残锁误导巡检。
21. **fail closed + 防状态掩盖**：任务失败要显式失败；警惕 `exit 0 + success_with_warnings` 掩盖全程失败（DYD 网络全挂仍报成功案）。健康检查看表级/字段级 max(date)，别看文件 mtime（G-X24，recap_health 范式）。
22. **确定性逻辑沉代码**（G-X25）：可判定的规则别只活在 SKILL 自然语言里——沉为版本化脚本（可复现/可测/有主/可 gate），SKILL 只写何时用/什么算合格/什么红线（G-X10 三段式）。
23. **跨 agent 分工先核能力边界**（G-X44/47）：gateway 沙箱开放出网 vs 官方逐域加白、per-task 模型路由等隐性能力，分工决策前实测，别按假设排班。

## 六、上下文与知识工程

24. **L0/L1/L2 分级加载**（2026-05-23）：stub 一句话定位→概览决策要不要深入→满足深读触发才加载全文；锚点分轻量/重型两档。防上下文污染（EXP-20260523-001）。
25. **GOTCHAS/经验库双轨**：踩坑→GOTCHAS 自动记（无需提示）；趟出路→经验库半自动（/save 时列候选、Doctor 点选）。升格门槛=难复得三问（持久？难复得？大概率再用？）。
26. **状态行优先**：引用 GOTCHAS/issue 先看状态行，✅ 的别当待办（G-X12）；翻 ✅ 前先核 canonical/git 实况，注释与计数会滞后（G-X28）。
27. **做完即登记**（G-X42）：项目完成那刻进项目总览+建骨架+落日志，否则成 /resume 与锚点都搜不到的游离项目。
