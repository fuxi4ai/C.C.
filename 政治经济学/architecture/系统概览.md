# 政治经济学 · 系统概览

> 项目最顶层视角。CC 进入该项目时第一篇要读的文件。

## 项目使命

打磨**认识框架本身**，而非生产结论。

好的分析不从结论出发寻找支撑，而由证据驱动框架进化。框架的精密度、洞见力与解释力，比任何具体结论都更有持久价值——"思维的碰撞本身即意义，对错是副产品"（Doctor 2026-05-08 确立）。

**项目不是**：新闻评论 / 立场辩护 / 结论生产机器。
**项目是**：用康德的认识论谦逊（物自体不可知）、波普尔的可证伪性、罗生门式多视角三角测量，对政治经济现象做严格的、可证伪的、可对抗审查的分析。

## 模块构成

**1 · 四大金刚文档**（Projects/政治经济学/ 根目录）
- `PRD.md`：愿景、原则、方法论。分析优先级原则（行为>利益>结构>叙事>人设）在此确立。
- `CLAUDE.md`：对 Claude 的操作规范。三层结构规程（L1 作者原命题不可改 / L2 质疑标注 / L3 我方分支）和多作者合成规程在此固化。
- `INDEX.md`：学术概念、F/H/SP 框架、案例的索引图。
- `GOTCHAS.md`：G-01~G-14 全文（brain 侧只做索引，不复制）。

**2 · 分析工具层**
- `credibility-matrix.md`：框架五维可信度矩阵。
- `predictions-register.md`：可证伪预测登记本（图尔钦克利动力学方法论标杆）。
- `sp05-lobby-weight.md`：以色列游说权重分析（SP-05 专属工具）。

**3 · 知识图谱可视化**
- `graph-data.js` + `validate-graph-sync.js` + `知识图谱.html`：F/H/G/SP 间关系的可视化与同步验证器。

**4 · 案例研究层 case-studies/**（项目合成层，介于 raw 与 INDEX 之间的双向桥接）
- 按文明组织（CS-XX_<文明名>/），含共享基础（历史时间线、学者谱系、sources）+ 框架应用（A01/A02...）+ CHANGELOG。
- 援引时**必带版本号**。

**5 · 原始材料层 raw/**
- `YYYY-MM-DD_[来源类型]_[主题].md` 命名规范。
- 核心防火墙：raw 收录 ≠ 承认为证据。

**6 · 产出层 outputs/**
- 文明基因报告（v1.0 含 md/json/html 三态）等可对外分享的成品。
