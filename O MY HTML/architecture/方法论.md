## 一、设计前置（动手前必须想清楚）

### 1.1 城市四字骨架

| ?重庆 | ?山江雾城 |  |

### 1.2 季节 × 时刻 × 母题（情绪定调器）· **v1.1 升级三轴**

> v1.1 关键修订：母题分**主流 / 小众**双轨。大众认知里的标准母题先做，小众文人母题留作进阶版。

| 城市 | 主流母题（先做） | 小众母题（进阶） |
|------|---------------|--------------|
| 杭州 | **春 · 白居易《忆江南》"江南忆，最忆是杭州"** | 冬 · 张岱《湖心亭看雪》"上下一白" |
| 北京 | **薄暮 · 紫禁中轴"暖金灯火次第点亮"** | 晨雾 · 老胡同烟火 |
| 长沙 | 秋 · 毛泽东《沁园春·长沙》"万山红遍" | 夏 · 烈日湘江 |
| 苏州 | **春 · 白居易"江南忆，其次忆吴宫"** | 秋 · 张继《枫桥夜泊》"江枫渔火" |
| 重庆 | 雾都黄昏 · "山城夜话" | 火锅夏夜 · "麻辣腾雾" |
| 西安 | 长安月夜 · "长安一片月，万户捣衣声" | 黄昏大雁塔 |

**操作要点（v1.1 新增）**：
- **时刻** 锚比季节锚更有诗意——"薄暮"、"晨光"、"月夜"、"雪后初晴"、"雨后"、"灯火初上" 都是好选择
- **主流母题**走"大众认知 + 官方文宣标签"路径；**小众母题**走文人趣味
- 系列化时建议先做主流版本，再做小众版本（小众放后期作"高级款"）

### 1.3 地标 9 选 + **视觉灵魂原则**（v1.1 新增）

#### 9 选标准

1. 与四字骨架对位（每个字 2-3 个地标支撑）
2. 形态可识别（外观独特，不会被 AI 画成泛中式）
3. 文化锚点（与季节/母题关联）
4. 视觉层次（远/中/近三层都有）

#### ⭐ **视觉灵魂原则**（v1.1 新增）

每个城市必须有**一个绝对视觉中心**，占画面 35-45%——这是城市的"视觉灵魂"。

| 城市 | 视觉灵魂 | 占画幅目标 |
|------|---------|----------|
| 北京 | **紫禁城建筑群（沿中轴线纵贯）** | 40% |
| 杭州 | **西湖湖面** | 40% |
| 重庆 | **嘉陵江-长江交汇处** | 35% |
| 西安 | **明城墙方形闭环 + 钟楼** | 40% |
| 苏州 | **古城护城河 + 拙政园** | 35% |
| 长沙 | **湘江南北轴 + 橘子洲** | 35% |

**反面教训（我们 card03 错误）**：杭州·雪让西湖只占 25%，被环湖建筑挤压。对标春景版西湖 40%——后者绝对的视觉中心感强 100%。

**操作要点**：prompt 里必须显式强调"X 居中铺展" / "Y 占画面主体" / "Z 为绝对视觉中心"。

### 1.4 配色契约（accent budget）

整张图的色彩必须**预算化**。

**v1.1 升级**：分**数值版**和**形容词版**双轨。

#### A. 数值版（严格审美，单色为主时用）

| 用途 | 占比 |
|------|------|
| 主底色（米白/月白/沙白） | 80-90% |
| 季节主色（秋红/桃粉/翠绿/雪白）| 8-12% |
| 印章红 / 暖灯点缀 | 2-5% |
| 墨黑阴影 | 2-5% |
| **任何蓝绿紫（除非季节需要）** | **0%** |

#### B. 形容词版（多色丰富时用，参照牛马 AIGC）

> "浅灰米白底色，哑光浮雕质感，光影清透柔和，超高细节"
> "暖金色灯火，红墙黄瓦温润光泽，青砖胡同"

**操作要点**：
- 杭州·雪、长沙·秋走 A 数值版（单色严格）
- 北京·薄暮、杭州·春走 B 形容词版（多色丰富）

### 1.5 字体 / 题款 / 印章规划

**必须矢量后期上字**——AI 写字（无论中英）必错。

| 元素 | 字体推荐 |
|------|---------|
| 西文主标 | Trajan Pro / Cinzel · 大写宽字距 200+ |
| 西文副标 / 简介 | Cormorant Garamond Italic · 9pt 衬线 |
| 中文主标 | 方正小标宋 / 思源宋 · 横排 |
| 中文副标 | 方正藏隶 / 仿宋 |
| 地标标注 | 思源黑 ExtraLight · 6-7pt |
| 印章 | 朱文方印矢量 · 4-5 字 |
| **GPS 坐标**（v1.1 新增） | Helvetica Light · 9pt · 含 ° ' " 符号 |

**操作要点**：
- prompt 里**预留空白**给标题块（"empty space at upper-left for title block"）
- AI 生图时只画浮雕主体，所有文字一律不让 AI 写

### 1.6 地域非视觉文化符号（v1.1 新增章）

> 牛马 AIGC 北京 prompt 的最大启发："青砖胡同里飘散**柿子树影与鸽哨余韵**"——鸽哨是声音意象，不可视，但 AI 收到这两个字会自动给整张图加上"北京式安静午后"的语义氛围。

**每个城市必须配置一组非视觉符号**（声音/气味/触感/光影动词），写进 prompt 的"氛围"段。

| 城市 | 非视觉符号候选 |
|------|-------------|
| 北京（参照） | 鸽哨余韵 / 柿子树影 / 烤白薯炉香 |
| 杭州 | 桂雨摇落 / 古琴泛音 / 龙井初炒香气 / 蚕茧丝织声 |
| 苏州 | 评弹声 / 雨打芭蕉 / 蒸糕的雾 / 园林滴水声 |
| 重庆 | 汽笛长鸣 / 麻辣腾雾 / 老茶馆茶碗碰瓷 / 索道嗡鸣 |
| 西安 | 秦腔余韵 / 羊肉泡馍香 / 鼓楼钟声 / 黄土麦秆 |
| 长沙 | 嗦粉声 / 茶颜叫卖 / 湘江木船橹声 / 桂花酒香 |
| 泉州 | 海风咸 / 海螺号 / 蚵仔煎香 / 古榕落叶 |

**写法**：在 prompt 末尾"氛围"段嵌入 2-3 个。例：
> "薄暮时分天光澄净，**桂雨摇落，琴音泛起**，西湖岸暖灯次第点亮..."

## 二、prompt 工程

### 2.1 prompt 长度甜区

| 长度 | 表现 |
|------|------|
| < 500 字节 | 信息密度不足 |
| **1200-1800 字节（约 250-300 字英文 / 200-250 字中文）** | **甜区** |
| > 2200 字节 | 容易踩 safety / 节流 |
| > 3000 字节 | 必崩 |

### 2.2 prompt 两套范式（v1.1 关键升级）

#### A. **中文二段范式 · 牛马 AIGC 实测款** ⭐ 默认推荐

```
[城市名 + 主题]: 3D 立体微缩浮雕地图海报，主题为[四字骨架]，[时刻]，
[配色形容词三组]，[质感词]，[精度声明：超高细节]。

核心场景：[地形铺展] + [主轴线/视觉灵魂占画幅] + [古代地标列举] +
[现代地标错落分布]。

[氛围段名 · 如"古都氛围/江南氛围/山城氛围"]：[时刻 + 光色] +
[材质细节] + [水/植被] + [地域非视觉符号 2-3 个]。
```

**北京实证样本**（牛马 AIGC，已证有效）：
> 3D 立体微缩浮雕地图海报，主题为北京古都中轴，立体皇城，浅灰米白底色，哑光浮雕质感，光影清透柔和，超高细节。
> 核心场景：平缓舒展的华北平原地形，中轴线纵贯南北，恢弘的紫禁城建筑群居中铺展，天坛祈年殿、太和殿、钟鼓楼、正阳门箭楼、北海白塔、颐和园佛香阁、万里长城如龙蜿蜒山脊，鸟巢、水立方、央视大楼、国贸 CBD 现代地标错落点缀。
> 古都氛围：薄暮时分天光澄净，暖金色灯火沿中轴线次第点亮，红墙黄瓦折射温润光泽，什刹海与筒子河水面倒映灯火，青砖胡同里飘散柿子树影与鸽哨余韵。

#### B. 英文三段范式（备用，特殊需求时用）

```
1. 媒介与构图（1 句）—— Hand-carved bas-relief paper diorama / 45° aerial atlas view / 9:16 vertical
2. 三段构图（3 行）—— Top band / Middle band / Bottom band 各 1 行
3. 单人叙事钩（1 句）—— A small figure with vermillion umbrella ...
4. palette 契约（1 行）—— Six tiny vermillion-red accents only: ...
5. 风格血缘（1 句）—— David Macaulay / Hattie Newman paper craft
```

#### 两种范式什么时候用谁

- **优先中文二段**：中国城市/地名（地名翻译损耗大）+ 多色丰富画面 + 需要文化氛围
- **英文三段**：特殊单色严格 palette + scale figure 叙事钩 + 国际化需求 + 极简风

### 2.3 视角与媒介词双轨（v1.1 关键升级）

| 视角 | prompt 媒介词 | 输出风格 | 用例 |
|------|------------|---------|------|
| ⭐ **3D 立体微缩 diorama** | "**3D 立体微缩浮雕地图海报**" / "elevated 3D miniature diorama photography" | 真实模型摄影感，单点透视，前低后高 | 北京中轴 / 杭州春 / 重庆山城 |
| 平面浮雕 bas-relief | "**hand-carved bas-relief paper diorama**" | 平面浅刻，纸雕感，均匀光照 | 杭州·雪（小众文人风）/ 长沙·秋（古地图气）|

**默认推荐 3D 微缩**——视觉信息量、物理重力感、光照真实性全面碾压平面浮雕。**bas-relief 留给追求古地图/文人画气质的子项目**。

### 2.4 已知触发词清单（高度警惕，不变）

| 词 / 词组 | 触发风险 | 替代 |
|----------|---------|------|
| `twin towers` | 9/11 联想 | `modern skyline` |
| `Lone figure` | 孤独叙事 | `small figure` / `tiny silhouette` |
| `no other people` | 反人物 negative | 删 |
| `muted crimson` | 血色联想 | `soft red` |
| `Christmas red-green` | 节日触发 | 删 |
| 含 `human` 单独出现 | 偶尔升级 | `figure` / `silhouette` |

**核心规律**：OpenAI gpt-image-2 比 gpt-image-1 严格得多；中文 prompt 因为不直接触发英文 safety classifier，反而往往更宽松（牛马 AIGC 实证）。

### 2.5 程度词陷阱

| ❌ 推到极端 | ✅ 克制执行 |
|-----------|----------|
| deeper / deepest | subtle / very mild |
| stronger / aggressive | slight / restrained |
| richer / intense | gentle / soft |
| more / significantly | a little / lightly |

要"Camera Raw 滤镜效果"用 subtle 档；用 strong 档会直接重画。

## 三、API 工程（APIYI · gpt-image-2）

### 3.1 调用栈

**不要用 Python `requests`**——会被系统代理 + 默认 UA + HTTP 版本多重干扰。

**正确做法**：`subprocess.run([curl, ...])`。

### 3.2 必备 headers

```python
[
    "-H", f"Authorization: Bearer {KEY}",
    "-H", "Content-Type: application/json",
    "-H", "User-Agent: curl/8.7.1",
    "-H", "Accept: */*",
]
```

### 3.3 三种 endpoint

| Endpoint | 方法 | Body |
|---------|------|------|
| `/v1/images/generations` | JSON | `{model, prompt, size, quality, n}` |
| `/v1/images/edits` | multipart | `-F image=@file -F prompt=...` |
| `/v1/images/variations` | multipart | `-F image=@file` |

generations 不支持 `response_format` 参数。

### 3.4 retry 策略

```
异常类型              重试  退避       立即放弃
─────────────────────────────────────────────────
curl 52 Empty reply    是    10/25/60s
HTTP 502/503/504       是    10/25/60s
HTTP 4xx (含 400)      是    60s（仅 1 次）  第 2 次仍 4xx 放弃
HTTP 200               -     -          解析响应
curl timeout >220s     是    10/25/60s
```

### 3.5 响应结构

```python
b64 = data.get("b64_json")              # gpt-image-2 新格式
img_url = data.get("url")
if not b64 and not img_url:
    arr = data.get("data") or []        # OpenAI 经典格式
    if arr:
        b64 = arr[0].get("b64_json")
        img_url = arr[0].get("url")
```

### 3.6 批量抽卡

```bash
N=5 python3 tools/gen_xxx.py
```

卡间 sleep ≥ 30 秒，单 session ≤ 10 张。

## 四、APIYI / OpenAI safety 矩阵（实测）

**真因假说**：APIYI 转接层在 quality×size 升级时上游 key 配额/路由限制触发，错误被网关统一包装为 `moderation_blocked + shell_api_error`。

### 4.1 SAFE / 踩雷 路径

| Quality \ Size | 1024×1024 | 1024×1536 | 1024×1792 |
|---|---|---|---|
| low | ✅ | （未测） | ✅ |
| medium | ✅ | ❌ | ❌ |
| high | ❌ | ❌ | （未测必拦） |

**结论**：只有 **low + 1024×1024** 与 **low + 1024×1792** 两条路径稳。

### 4.2 关键 size 选型

- **1024×1024**（square）—— Instagram、抖音封面通用
- **1024×1792**（4:7 portrait）—— OpenAI 官方支持，近似 9:16，**抖音/小红书首选**
- **1024×1536** —— **非官方 size，APIYI 必拦，避免**

### 4.3 错误识别速查

| 错误 | 真因 | 处理 |
|------|------|------|
| `curl: (52) Empty reply` | 上游 timeout | retry 10/25/60s |
| `HTTP 502 Bad Gateway` | APIYI 网关 timeout | retry 10/25/60s |
| `HTTP 400 + shell_api_error + moderation_blocked` | APIYI 节流伪装 | retry 60s 一次；第 2 次仍 400 = 真 prompt 错 |
| `HTTP 400 + Unknown parameter` | 真参数错 | 修参数，不重试 |
| `HTTP 401` | key 过期 | 检查 .env |

## 五、抽卡策略

### 5.1 流程

```
1. 跑首张测样（low + square 1024×1024）→ 验风格基线
2. 风格对 → 升 portrait（low + 1024×1792）批量 N=5
3. 4-5 张里挑最佳（6 维评分见 5.2）
4. 选定底图进 Phase 2 调色
```

### 5.2 6 维评分（v1.1 新增"视觉灵魂"维度）

| 维度 | 权重 | 评分细则 |
|------|------|---------|
| **视觉灵魂占画幅**（v1.1 新增） | 25% | 城市核心（西湖 / 紫禁城 / 城墙）是否占 35-45% |
| 风格凝聚力 | 20% | palette 守纪律 / 整体气质统一 |
| 地标位置准确 | 20% | 关键地标位置不错乱 |
| 字错率 | 10% | AI 写字错率，低 = 加分（后期可修但越少越好）|
| 题款气质 | 10% | 古朴 / 文化精度 |
| 红色 accent 密度 | 15% | 目标点是否到位且克制 |

### 5.3 选型原则

- **视觉灵魂 + 风格凝聚力** 双满足是必要条件，缺一个就该重抽
- **AI 自发加分元素**（如"西湖十景"副标 / GPS 坐标 / 罗盘中文化）优先
- **位置错的地标可后期 inpaint，但视觉灵魂错全图重抽**

## 六、后期调色（image-2 edit endpoint）

### 6.1 能做什么 / 不能做什么

- ✅ 加深阴影 / 加暗角 / 增对比（精准范围内）
- ✅ 局部 inpaint（加 mask）
- ❌ 精确数学滤镜（Camera Raw 那种 100% 可控）
- ❌ 保证完全不动构图

### 6.2 subtle 档调色 prompt 模板

```
Apply a very subtle tonal enhancement to this [media description].
PRESERVE the [base color] of the [protected areas] — do not [adjust] these.
Only slightly [adjust action] in the [target areas] to [achieve effect].
Add a very mild [side effect] (no more than 10%).
Maintain the original [palette].
Like applying a Camera Raw filter with [Blacks -10, Contrast +8, Vignette -5, Texture +5]
— subtle and restrained, not aggressive.
Keep all labels, landmarks, characters, compass, title block exactly as they are.
```

三条铁律：
1. **subtle / very mild / slight** —— 程度词必须克制
2. **PRESERVE / do not + 保护区域** —— 显式锁定不变区域
3. **Camera Raw 数值锚** —— 给 image-2 具体参考点

## 九、失败模式速查表

| 现象 | 第一反应 |
|------|---------|
| Empty reply 一次 | 不慌，retry（脚本自动） |
| 连续 3 次 Empty reply | 节点波动，等 5-10 分钟 |
| HTTP 400 moderation_blocked | 先核对 size 是否官方 + 是否 quality 越级；再排查 prompt 触发词；最后才考虑真 safety |
| 图比例错 | size 参数没传对，gpt-image-2 仅认官方 sizes |
| AI 写字错 | 不重抽，矢量后期上字 |
| **视觉灵魂占比偏小**（v1.1 新增） | 改 prompt 强调"X 占画面主体"，重抽 |
| **平面浮雕感太弱**（v1.1 新增） | 媒介词改用 "3D 立体微缩浮雕" 触发 diorama 风 |
| 整体风格散 | prompt 太长，压到 ≤ 1800 字节 |
| 调色后变石刻 / 重画 | 程度词太极端，换 subtle / very mild |
| 大批量抽卡半数失败 | 卡间 sleep 加到 60s |
