# 城市浮雕图谱系列 · 方法论 v1.0

> 从《长沙·秋》溯源 + 《杭州·雪》0→1 走通沉淀。
> 适用：任何"AI 文生图 + 后期 + 系列化"的城市/地标主题项目。
> 工具栈：gpt-image-2 via APIYI · curl subprocess · 后期 PS / Affinity（或 image-2 edit endpoint）

---

## 一、设计前置（动手前必须想清楚）

### 1.1 城市四字骨架

每个城市必须能用 4 字概括其地理 + 文化结构。这决定了浮雕图的**主轴**与**地标取舍**。

| 城市 | 四字骨架 | 解读 |
|------|---------|------|
| 长沙 | 山水洲城 | 岳麓-湘江-橘子洲-老城（官方文宣已提炼） |
| 杭州 | 湖山江城 | 西湖-群山-钱塘江-古城+运河（项目内提炼） |
| ?重庆 | ?山江雾城 | 待提炼 |
| ?苏州 | ?水巷园城 | 待提炼 |

**操作要点**：四字必须横跨地理 + 文化双轴，光地形或光文化都不够。

### 1.2 季节 × 文学母题（情绪定调器）

季节选择决定 palette；文学母题决定叙事钩。**两者必须同时提炼**。

| 城市 | 季节 | 母题 | palette | 叙事钩 |
|------|------|------|---------|--------|
| 长沙 | 秋 | 《沁园春·长沙》"万山红遍" | 米白 + 枫秋红 | 橘子洲毛主席像 |
| 杭州 | 冬 | 《湖心亭看雪》"上下一白" | 米白 + 朱红 | 撑伞女子立断桥头朝湖心亭 |

**操作要点**：母题必须能塞进 prompt（"上下一白"→ "frozen white-on-white"），且为整张图情绪定锚。

### 1.3 地标 9 选

> **克制比饱满更重要**。AI 越是堆地标越容易构图崩。

**选地标的 4 个标准**（按权重）：
1. 与四字骨架对位（每个字都要有 2-3 个地标支撑）
2. 形态可识别（外观独特，不会被 AI 画成"泛中式建筑"）
3. 文化锚点（与季节/母题关联）
4. 视觉层次（远/中/近三层都有，避免全堆在一块）

**反例**：第一次草拟杭州地标 13 个 → 第三轮强行裁到 9 个 → 实际 AI 自发画了 15+，但效果反而散了。
**教训**：prompt 写 9 个是好事，让 AI 自己加超 12 个就要警惕重复标签 / 位置错乱。

### 1.4 配色契约（accent budget）

整张图的色彩必须**预算化**——不是"用什么色"，是"每种色占百分之几"。

**长沙·秋 / 杭州·雪 共用契约**：

| 用途 | 占比 |
|------|------|
| 米白 / 月白 / 沙白底（浮雕基底） | 85% |
| 季节主红（枫红 / 朱砂） | 8-10% |
| 印章红 / 暖灯 / 人物点缀 | 3-5% |
| 墨黑 / 灰阴影 | 2-5% |
| **任何蓝绿紫** | **0%**（明确禁止） |

**操作要点**：
- 水面**不画蓝**——画银灰 / 月白
- 天空不画云不画色——直接留白或薄雾
- 6-8 个红点是上限，多一个都过

### 1.5 字体 / 题款 / 印章规划

**必须矢量后期上字**——AI 写字（无论中英）一定出错。

| 元素 | 字体推荐 | 说明 |
|------|---------|------|
| 西文主标 | Trajan Pro / Cinzel | 大写宽字距 200+ |
| 西文副标 / 简介 | Cormorant Garamond Italic | 9pt 衬线 |
| 中文主标 | 方正小标宋 / 思源宋 | 横排 |
| 中文副标 | 方正藏隶 / 仿宋 | 比主标小 1/3 |
| 地标标注 | 思源黑 ExtraLight | 6-7pt |
| 印章 | 朱文方印矢量 | 4-5 字最优 |

**操作要点**：
- prompt 里**预留空白**给标题块（"empty space at upper-left for title block"）
- AI 生图时只画浮雕主体，所有文字一律不让 AI 写

## 二、prompt 工程（这套是核心 KnowHow）

### 2.1 prompt 长度甜区

| 长度 | 表现 |
|------|------|
| < 500 字节 | 信息密度不足，AI 自由发挥过多 |
| **1200-1500 字节** | **甜区**，信息密集且不触发 token-edge 触发器 |
| > 2000 字节 | 容易踩 safety / 节流 / token 超 |
| > 3000 字节 | 必崩 |

杭州·雪最终版 1487 字节。

### 2.2 prompt 五段结构

固定结构，每个城市套用：

```
1. 媒介与构图（1 句）—— hand-carved bas-relief paper diorama / 45° aerial atlas view / 9:16 vertical
2. 三段构图（3 行）—— Top band / Middle band / Bottom band 各 1 行
3. 单人叙事钩（1 句）—— 极简，避免"lone / alone / facing"等孤独叙事词
4. palette 契约 + accent budget（1 行）—— Six tiny vermillion-red accents only: ...
5. 风格血缘（1 句）—— David Macaulay / Hattie Newman paper craft / Song dynasty city map
```

**操作要点**：
- 删除 Negative 段——OpenAI gpt-image-2 对"no X, no Y"列表的 multi-factor classifier 容易误判
- 反向意图通过**正向描述**实现："matte finish, single light source, desaturated muted palette" 替代 "no glow, no neon, no saturated"

### 2.3 已知触发词清单（高度警惕）

| 词 / 词组 | 触发风险 | 替代 |
|----------|---------|------|
| `twin towers` | 9/11 联想（一律拦） | `modern skyline` |
| `Lone figure` | 孤独叙事 | `small figure` / `tiny silhouette` |
| `no other people` | 反人物 negative | 删 |
| `muted crimson` | 血色联想 | `soft red` |
| `Christmas red-green` | 节日触发 | 删 |
| 含 `human` 单独出现 | 偶尔升级 | `figure` / `silhouette` |

**核心规律**：OpenAI gpt-image-2 比 gpt-image-1 严格得多，对所有"暴力 / 政治 / 孤独 / 节日"语义有 false positive 倾向。

### 2.4 程度词陷阱

| ❌ image-2 会推到极端 | ✅ image-2 能克制执行 |
|---------------------|--------------------|
| deeper / deepest | subtle / very mild |
| stronger / aggressive | slight / restrained |
| richer / intense | gentle / soft |
| more / significantly | a little / lightly |

**操作要点**：要"Camera Raw 滤镜效果"用 subtle 档；用 strong 档会直接重画。

## 三、API 工程（APIYI · gpt-image-2）

### 3.1 调用栈（避坑过的）

**不要用 Python `requests`**——会被以下多重干扰：
- 系统代理变量（Clash / Stash 的 TUN 模式）
- 默认 UA `python-requests/X.X` 被 APIYI openresty 拦
- HTTP/1.1 vs HTTP/2 协议层差异

**正确做法**：用 `subprocess.run([curl, ...])`，因为 curl 已被多种环境验证可用，且不依赖任何 Python 库。

### 3.2 必备 headers

```python
[
    "-H", f"Authorization: Bearer {KEY}",
    "-H", "Content-Type: application/json",        # generations
    "-H", "User-Agent: curl/8.7.1",                # 避免 python-requests 默认 UA 被拦
    "-H", "Accept: */*",
]
```

### 3.3 三种 endpoint 的 payload 差异

| Endpoint | 方法 | Body | 用途 |
|---------|------|------|------|
| `/v1/images/generations` | JSON | `{model, prompt, size, quality, n}` | 从零生成 |
| `/v1/images/edits` | multipart | `-F image=@file -F prompt=... -F ...` | 喂图调色/重画 |
| `/v1/images/variations` | multipart | `-F image=@file` | 同主题变体（少用） |

**注意 generations**：不支持 `response_format` 参数，传了会 400。

### 3.4 retry 策略

```
异常类型              重试  退避       立即放弃
─────────────────────────────────────────────────
curl 52 Empty reply    是    10/25/60s
HTTP 502/503/504       是    10/25/60s
HTTP 4xx (含 400)      是    60s（仅重试 1 次）  第 2 次仍 4xx 放弃
HTTP 200               -     -          解析响应
curl timeout >220s     是    10/25/60s
```

**4xx 也 retry 是因为 APIYI 把节流伪装成 400 + `shell_api_error / moderation_blocked`**，60s 冷却后多数能通。如果是真 prompt 错，第 2 次仍 4xx 才放弃，浪费 1 次额度但换来准确分类。

### 3.5 响应结构（三种兼容）

```python
b64 = data.get("b64_json")            # gpt-image-2 新格式（顶层）
img_url = data.get("url")
if not b64 and not img_url:
    arr = data.get("data") or []      # OpenAI 经典格式
    if arr:
        b64 = arr[0].get("b64_json")
        img_url = arr[0].get("url")
```

### 3.6 批量抽卡

```bash
N=5 python3 tools/gen_xxx.py
```

**间隔规则**：
- 卡间 sleep **≥ 30 秒**（否则 APIYI 节流，5 张失败 3 张）
- 单 session 内总卡数建议 ≤ 10

## 四、APIYI / OpenAI safety 矩阵（实测）

**真因假说**（未完全证实，待官方 OpenAI key 验证）：APIYI 转接层在 quality×size 升级时上游 key 配额 / 路由限制触发，错误被网关统一包装成 `moderation_blocked + shell_api_error`，**不是 OpenAI 真发的内容审查**。

### 4.1 已知 SAFE 路径

| Quality | Size | 状态 |
|---------|------|------|
| low | 1024×1024 | ✅ 稳 |
| low | 1024×1792 | ✅ 稳（推荐 portrait） |
| medium | 1024×1024 | ✅ 通但风格散 |

### 4.2 已知踩雷路径

| Quality | Size | 状态 |
|---------|------|------|
| medium | 1024×1536 | ❌ 必拦 |
| medium | 1024×1792 | ❌ 必拦 |
| high | 任何 size | ❌ 必拦 |

### 4.3 关键 size 选型

- **1024×1024**（square）—— Instagram、抖音封面通用
- **1024×1792**（4:7 portrait）—— OpenAI 官方支持，近似 9:16，**抖音/小红书首选**
- **1024×1536**（2:3 portrait）—— **非 OpenAI 官方 size**，APIYI 实测必拦，避免

### 4.4 错误识别速查

| 错误现象 | 真因 | 处理 |
|---------|------|------|
| `curl: (52) Empty reply` | 上游 timeout / 节点 reset | retry 10/25/60s |
| `HTTP 502 Bad Gateway` | APIYI 网关 upstream timeout | retry 10/25/60s |
| `HTTP 400 + shell_api_error + moderation_blocked` | APIYI 节流伪装 | retry 60s 一次；第 2 次仍 400 = 真 prompt 问题 |
| `HTTP 400 + Unknown parameter` | 真参数错（如传了 response_format） | 修参数，不重试 |
| `HTTP 401` | key 过期 / 域名错 | 检查 .env |

## 五、抽卡策略

### 5.1 流程

```
1. 跑首张测样（quality=low）→ 看风格基线（浮雕 / palette / 三段构图）是否对
2. 若风格对 → 批量抽卡 N=5（同参数同 prompt）
3. 4-5 张里挑最佳（5 维评分见下）
4. 选定底图进 Phase 2 后期
```

### 5.2 5 维评分表

| 维度 | 权重 | 评分细则 |
|------|------|---------|
| **风格凝聚力** | 30% | 浮雕厚度 / 米黄底 / 整体气质统一 |
| **地标位置准确** | 25% | 湖心亭 / 三潭印月 / 双堤 / 各塔位置 |
| **字错率** | 15% | AI 写字错率，低 = 加分 |
| **题款气质** | 15% | 古朴 / 文化精度（"壬寅冬製"等） |
| **红色 accent 密度** | 15% | 6-8 个目标点是否到位 |

### 5.3 选型原则

- **风格凝聚力 > 细节精度**：精度高但风格散的卡比风格统一但缺细节的卡更弱
- **AI 自发加分元素**优先（如"南宋臨安府"副标 / 西湖十景题款 / 罗盘中文方位）
- **位置错的地标可后期 inpaint，但风格错全图重抽**

## 六、后期调色（image-2 edit endpoint）

### 6.1 能做什么

- 加深阴影 / 加暗角 / 增对比（**精准范围内**）
- 局部 inpaint（加 mask）
- 微改色调 / palette 暖冷偏

### 6.2 不能做什么

- 精确数学滤镜（Camera Raw 那种 100% 可控）
- 保证完全不动构图（即使 prompt 说 keep，也可能微动）

### 6.3 调色 prompt 模板（subtle 档）

```
Apply a very subtle tonal enhancement to this [media description].
PRESERVE the [base color] of the [protected areas] — do not [adjust] these.
Only slightly [adjust action] in the [target areas] to [achieve effect].
Add a very mild [side effect] (no more than 10%).
Maintain the original [palette].
Like applying a Camera Raw filter with [Blacks -10, Contrast +8, Vignette -5, Texture +5]
— subtle and restrained, not aggressive.
Keep all labels, landmarks, characters, compass, title block exactly as they are.
```

**关键三条**：
1. **subtle / very mild / slight** —— 程度词必须克制
2. **PRESERVE / do not + 保护区域** —— 显式锁定不变区域
3. **Camera Raw 数值锚** —— 给 image-2 一个具体参考点

```
Phase 0 · 设计前置
  0.1 城市四字骨架
  0.2 季节 × 文学母题
  0.3 9 地标选型
  0.4 palette + accent budget
  0.5 字体 / 题款 / 印章规划

  1.1 验证 API 凭证 + 写最小脚本
  1.2 跑首张测样（low + 1024×1024 验证风格基线）

  3.1 矢量补西文标题块
```

## 八、失败模式速查表

| 现象 | 第一反应 |
|------|---------|
| 一次 Empty reply 不通 | 不慌，retry 一次（脚本自动） |
| 连续 3 次 Empty reply | 节点波动，等 5-10 分钟换时段 |
| HTTP 400 moderation_blocked | **先核对**参数（size 是否官方 / 是否 high quality）；再排查 prompt 触发词；最后才考虑真 safety |
| Image 出来比例错 | size 参数没传对，gpt-image-2 仅认官方 sizes |
| AI 写字错 | 不要重抽，矢量后期上字 |
| 地标位置错 | 可接受范围内继续；离谱重抽 |
| 整体风格散 | 检查 prompt 是不是太长，压到 ≤ 1500 字节 |
| 调色后变石刻 / 重画 | 程度词太极端，换成 subtle / very mild |
| 大批量抽卡半数失败 | 卡间 sleep 加到 60s |
