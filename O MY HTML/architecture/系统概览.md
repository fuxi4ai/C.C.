> 项目最顶层视角。CC 进入该项目时第一篇要读的文件。

## 项目使命

把 AI 产出从 Markdown 文本升级为"可排版 + 可视化 + 可交互"的 HTML 产物——让读者按自己的思考链（而非文档编号）消费内容，发挥 HTML 的立体信息结构（北极星见 `Projects/O MY HTML/PRD.md`）。并衍生出"AI 文生图 + 矢量后期"的城市/文明微雕可视化工作流。一句话：做 Doctor 各研究项目（渊图 / 龙鱼五力 / PEC / DVA）的**可视化与设计交付工具箱**。

## 当前阶段

- **状态**：🟡 维护（与 [[CLAUDE]] 项目列表一致）
- **最后活跃**：2026-06-26（design-system 新增「暖色·卡片页」页面原型层：styleguide + boilerplate，承 jiuyin-warm·指向 GAI artifact）

## 模块构成

- **设计师工具箱.html（Taste Skill Loader）**：主入口，注入 Emil / Kami / visual-eye / impeccable 等设计品味 skill，按"天工开物 / 启用设计模式"咒语启用设计模式。
- **微雕图谱系列**（城市浮雕 / 旅行微雕 / 文明微雕）：AI 文生图 + 矢量后期的系列化工作流，方法论见 [[方法论]] v1.1；城市浮雕是政经/文化比较研究的可视化基底。
- **可视化原型**（文明基因互动地图 / 报告矩阵、中际旭创首屏仪表盘等）：把研报 / 知识图谱数据落成可交互 HTML，接渊图 / 龙鱼 / PEC。
- **组件与模式库**（`components/_template` + `prototypes/`）：组件化沉淀，单文件 A → 演进 B，接入生产管线。
- **设计参考库 `design-ref/`**（visual-eye / impeccable / kami / taste-skills / emil）：设计品味与构图原理参考。
- **生图管线**：各微雕子项目 `tools/` 下的 `gen_*.py` / `edit_*.py`（如 `城市浮雕/杭州·雪/tools/`），via APIYI gpt-image-2 的 curl subprocess 调用；顶层 `tools/` 放设计模式开关等。调用栈见 [[方法论]] §三/§四。

## 关键依赖

- 上游：数据源（龙鱼五力样本报告、渊图知识图谱、PEC、DVA）；设计品味库（Emil / Kami / visual-eye / impeccable，KV 摄影构图）；APIYI gpt-image-2（生图，转接模式见 [[apiyi-transit-mode]]）。
- 下游：为渊图 / 龙鱼五力 / PEC / DVA 提供可视化与设计交付载体（HTML 产物 / 微雕海报）。
