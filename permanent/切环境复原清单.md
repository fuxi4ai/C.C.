# 切环境复原清单

> 切工作环境（桌面官方 ↔ gateway/Claude-3p ↔ Claude Code CLI ↔ 换机）时，**注册表/缓存/侧栏是按环境独立的，不自动带过来**。新环境形如白纸，须按本清单逐类复原。这是「切环境该补什么」的**总索引**——各类的真源镜像见对应笔记。
>
> 缘起：2026-07-01 订阅恢复切回官方工作区，复原项散落多处，artifacts 与网络白名单差点漏（见 logs/2026-07-01-切回官方-重建定时任务与skill）。

## 五类复原项

### 1 · 定时任务（Scheduled Tasks）
- **判据**：新环境 `list_scheduled_tasks` 返回 0 = 没带过来。
- **真源**：[[定时任务清单]]（cron + 通知 + 完整 prompt 指针）。
- **复原**：照清单逐个 `create_scheduled_task`（cron/prompt 照搬，ad-hoc 任务省 cron→Manual only）。任务文件在**受保护目录** `~/Documents/Claude/Scheduled/`（沙箱挂不了）→ 需 Doctor 终端 `cat` 出各 `SKILL.md` 全文。建完 `list` 核对 `cronExpression` 字段（别信摘要行）。

### 2 · Skills（自制）
- **判据**：可用 skill 列表里缺 `brain-*` 等自制项。
- **真源**：[[已装skill清单]] + `brain/.skills/*.skill` 包。
- **复原**：Cowork 走 Settings→Capabilities 手动选 `.skill` 安装（我这边装不了）。⚠️ **旧镜像可能过时**（无引号 frontmatter 违 Cowork rule 4 → 装不进 / 缺新内容）——先从现源重打包（python zipfile 直写，避挂载盘 zip 限制 G-X48）再发。**改 SKILL.md 源后必重打 `.skill`**，否则镜像与源漂移、下次又踩坑。

### 3 · Artifacts（侧栏看板）
- **判据**：新环境侧栏空；定时任务 `update_artifact` 推送前需本体存在。
- **共 4 个**：`global-asset-inventory`（海螺资产看板）、`zhuzhao-jiuyin-daily`（烛照日报）、`baize-weekly-dashboard`（白泽周报）、Brain Vault Dashboard 星图（带「审查并修复」按钮，触发 repair-finance-chain）。
- **真源/复原**：见 [[artifact存储机制]] + 各生成脚本（烛照 `gen_daily_report.py`/sync、白泽 `sync_to_cowork_artifact.py`、海螺 `build_asset_dashboard.py`、星图 dashboard 生成器）。多数可由对应定时任务或脚本重推。

### 4 · 网络白名单
- **判据**：官方沙箱**默认拒绝 + 逐域名加白**；gateway/Claude-3p **开放出网**、无需白名单。
- **真源**：[[定时任务清单]]「网络白名单」节 + [[通用教训]] G-X44。
- **复原**：切回官方后，取数域名（`api.tushare.pro`/`hq.smm.cn`/`api.deepseek.com`/`stockanalysis.com`/Yahoo/抖音 等）须逐个加白；否则取数任务标缺不编数（有 curl 自检兜底）。对一个取数任务点 **Run now** 既验出网、又预批工具权限。

### 5 · 文件夹挂载
- **判据**：沙箱默认可能只挂 Brain，够不到 Database/Projects/AI4ME。
- **复原**：各任务 prompt 内 `request_cowork_directory` 按需挂。⚠️ `~/Documents` **整挂会被 `Claude/Scheduled` 保护目录拦截**（`overlaps a protected host location`）→ 必须**分目录挂**（Brain / Database / 各 Project / AI4ME 各挂各的）。平铺挂载路径算错时靠脚本 env 后门（G-X45）。
