# 切环境复原清单

> 切工作环境（桌面官方 ↔ gateway/Claude-3p ↔ Claude Code CLI ↔ 换机）时，**注册表/缓存/侧栏是按环境独立的，不自动带过来**。新环境形如白纸，须按本清单逐类复原。这是「切环境该补什么」的**总索引**——各类的真源镜像见对应笔记。
>
> 缘起：2026-07-01 订阅恢复切回官方工作区，复原项散落多处，artifacts 与网络白名单差点漏（见 logs/2026-07-01-切回官方-重建定时任务与skill）。

## 五类复原项

### 1 · 定时任务（Scheduled Tasks）
- **判据**：新环境 `list_scheduled_tasks` 返回 0 = 没带过来。
- **真源**：[[定时任务清单]]（cron + 通知 + 完整 prompt 指针）。
- **复原**：照清单逐个 `create_scheduled_task`（cron/prompt 照搬，ad-hoc 任务省 cron→Manual only）。任务文件在**受保护目录** `~/Documents/Claude/Scheduled/`（沙箱挂不了）→ 需 Doctor 终端 `cat` 出各 `SKILL.md` 全文。建完 `list` 核对 `cronExpression` 字段（别信摘要行）。

### 2 · Skills（自制）
- **判据**：可用 skill 列表里缺 `brain-*` 等自制项。
- **真源**：[[已装skill清单]] + `brain/.skills/*.skill` 包。
- **复原**：Cowork 走 Settings→Capabilities 手动选 `.skill` 安装（我这边装不了）。⚠️ **旧镜像可能过时**（无引号 frontmatter 违 Cowork rule 4 → 装不进 / 缺新内容）——先从现源重打包（python zipfile 直写，避挂载盘 zip 限制 G-X48）再发。**改 SKILL.md 源后必重打 `.skill`**，否则镜像与源漂移、下次又踩坑。

### 3 · Artifacts（侧栏看板）
- **判据**：`list_artifacts` 返回 0（manifest 在 `~/Library` 按环境独立、切环境清空）。
- **共 4 个**：`global-asset-inventory`（海螺资产看板）、`zhuzhao-jiuyin-daily`（烛照日报）、`baize-weekly-dashboard`（白泽周报）、`yuantu-starry-skies`（渊图行业知识星河·Three.js 暗色）。⚠️ **Brain Vault Dashboard 星图（`brain-vault-dashboard`）2026-06-24 已退役、并入 GAI，不在此列、不重建**（见海螺 `BVD退役盘点单`）。前三个各有定时任务刷新，星河无任务刷新、随渊图手动重推。
- **复原（2026-07-01 实战验证·最优法＝抢救原本体重注册）**：
  1. `list_artifacts`=0 但 `create_artifact` 报「folder already exists」→ Documents 下 `Claude/Artifacts/{id}/` **孤儿文件夹**残留（manifest 清了、Documents 本体没清）。`Claude/Artifacts` 是**受保护目录**、CC 挂不了也清不了。
  2. **孤儿文件夹里的 `index.html` 就是最新本体**——请 Doctor 终端把 4 个 `cp` 到已挂载处（如 `AI4ME/_artifact_rescue/`）抢救出来。
  3. Doctor 终端 `rm -rf ~/Documents/Claude/Artifacts/{4个id}` 清孤儿。
  4. CC 从抢救本体逐个 `create_artifact`（保真度最高，胜过重跑脚本）。`list_artifacts` 核对 4 个就位。
- **备选**：`update_artifact` 在 manifest 空时报 not found、不可用；重跑生成脚本（烛照 `gen_daily_report.py`、白泽 `sync_to_cowork_artifact.py`、海螺 `build_asset_dashboard.py`）出的是新数据版，仅在抢救不到本体时走。机制底细见 [[artifact存储机制]]。

### 4 · 网络白名单
- **判据**：官方沙箱**默认拒绝 + 逐域名加白**；gateway/Claude-3p **开放出网**、无需白名单。
- **真源**：[[定时任务清单]]「网络白名单」节 + [[通用教训]] G-X44。
- **复原**：切回官方后，取数域名（`api.tushare.pro`/`hq.smm.cn`/`api.deepseek.com`/`stockanalysis.com`/Yahoo/抖音 等）须逐个加白；否则取数任务标缺不编数（有 curl 自检兜底）。对一个取数任务点 **Run now** 既验出网、又预批工具权限。

### 5 · 文件夹挂载
- **判据**：沙箱默认只挂 Brain，够不到 Database/Projects/AI4ME。
- **复原**：`request_cowork_directory` 按需挂。⚠️ `~/Documents` **整挂会被受保护目录拦截**（`overlaps a protected host location`）→ 必须**分目录挂**；平铺挂载路径算错时靠脚本 env 后门（G-X45）。

- **为什么不能整挂 Documents（2026-07-01 查实·别再纠结）**：`~/Documents` 里有两个**应用级受保护目录** `Claude/Scheduled`（定时任务）+ `Claude/Artifacts`（侧栏看板），挂载系统拒绝任何"盖住受保护位置"的父目录，故 `~/Documents` 与 `~/Documents/Claude` 都挂不了。**这俩位置应用写死、不可配置**（官方文档无搬家开关；`create_scheduled_task` 明写存 `~/Documents/Claude/Scheduled/`）。手动搬/软链均**不推荐**（破坏应用活目录 + sandbox 不跟随跨界软链）。把工作文件迁到 `Documents/C.C.` 换整挂：实测 **300+ 文件、1500+ 处硬编码绝对路径**要改，投入产出严重倒挂，**不做**。想根治只能给 Anthropic 提"数据目录可配 / 挂载可持久"功能请求。
- **✅ 5 挂载配方（最接近"整挂"、够用）**：`Claude/Brain` + `Claude/Projects`（**一条覆盖全部项目**：海螺/白泽大宗/烛照九阴/剑酒青丘/DVA/O MY HTML/PEC/司南/星空/称象/MiroFish/…）+ `Database` + `AI4ME` + `Vault`。这 5 个都不含受保护子目录、各自可挂，合起来＝ Documents 工作集全覆盖。
- **挂载策略（2026-07-01 定）**：默认只挂 Brain（它是"地图"，含 CLAUDE.md/anchors/各清单）；其余 CC **开工前预判 + 批量请求**（少打断）。定时任务在调度沙箱里 `request_cowork_directory` **自动挂载可行**（G-X45 实证）。
