# 分发版构建范式

> 来源：金融项目分发优化 PRD（`logs/checkpoints/2026-07-06_金融项目分发优化_PRD.md`），烛照九阴·日报试点打样沉淀。目标：四个可视化项目（日报/周报/龙鱼标的表/星空）共用此范式产出**脱库·脱敏·自包含·移动端适配**的读者版单文件 HTML。

## 一 · 架构原则（Doctor 2026-07-06 对齐裁定）

1. **增量铁律**：分发产线只读内部产物、只写 `AI4ME/订阅分发-outputs/`；内部生成器/定时班/数据库零改动。执行前后 md5 快照兜底（`config/md5_baseline.txt`）。
2. **单文件自包含**：字体子集、图片、图表库全内嵌（data: URI），断网可完整渲染；license 注释里的 URL 一并剥离。体积上限 5MB。
3. **独立定时任务**生成（不挂进内部班）：prompt 必带 **G-X51 防悬挂前置**——先 `ls` 探测挂载，未挂载直接结束，绝不调用会弹批准的工具。
4. **中性对外品牌**：内部代号绝不对外（日报 → 观潮·市场日评）；品牌/免责/方法说明集中在 `dist/brand.json`。
5. **信号只给分数与多空方向**：不出现 买入/卖出/加减仓 等操作字样；分数取整到个位（防多期对照反推）；页脚固定风险提示「不构成投资建议」。
6. **可信度靠脱敏信源**：帕米尔→行业专家电话会议资料、FRED→美联储公开经济数据、tushare→交易所公开行情数据、外资研报→国际一线投行研究报告（`dist/source_map.json`）。
7. **方法披露到"可信不可复制"**：维度名称+加权方式可露；具体权重、阈值、降级核查/兑现折扣类细则不露（QA 红线词把关）。

## 二 · 管线（试点实现，可直接复用）

```
内部产物 HTML ──> tools/build_dist_report.py ──> dist/观潮市场日评_YYYYMMDD.html
                    │ 1 剥 license URL 注释块
                    │ 2 页脚整体重写（库名署名 → 品牌+免责）
                    │ 3 REWRITES 有序替换表（长词优先）
                    │ 4 分数取整（评分/综合/中文后括号分三个受限正则）
                    │ 5 注入 方法与信源说明块（id="score-note"）
                    │ 6 注入 移动端 CSS + 图表 resize 监听
                    └─> tools/dist_qa.py 全项静态断言（51 项，PASS + exit 0 才算产出）
```

## 三 · 试点踩坑（复用时先读）

- **禁词分两层**：短 ASCII 词（CC/recap 等）在内嵌图表库压缩码/base64 里必然误伤 → `blacklist.txt` 用 `v:` 前缀标"仅可见文本检查"，中文词全文件检查。
- **数值替换须限定上下文**：`(57.5)→(58)` 这类括号分正则必须限定前缀中文字符，否则会改写内嵌 JS 的数值参数、破坏图表库。
- **source_map 与 REWRITES 必须对齐**：QA 的"对外口径必须出现"断言用的是 source_map 的值——两处写法不一致会假 FAIL。
- **license 注释是外链的唯一残留源**：ECharts/zrender banner 含 apache/github URL，剥注释即可，不必动代码体。
- **沙箱无 headless 渲染能力**（arm64 缺系统库且包源 403）：渲染实测走 Claude in Chrome 扩展；扩展未连接时先做完静态断言，渲染留待。

## 四 · 复用到其余三项目的差异注记

| 项目 | 特殊点 |
|---|---|
| 白泽大宗·周报 | 信源以 FRED/外盘为主，source_map 权重高；周频，定时任务改周日/周一 |
| 龙鱼五力·标的表 | **保密面最重**：边际变动连续可见可反推评分函数——需先单独划披露红线（分档/延迟/抽样均为候选），未划线前不做 |
| 渊图·星空 | 纯前端最轻，重点是资产内嵌与移动端 WebGL 性能；无信号合规问题 |
