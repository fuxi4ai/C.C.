# 渊图国产公司量化评分方法

**当前版本**：v1.1（2026-05-28 借鉴龙鱼五力 v3.2.1 演进）
**v1.0 基线**：保留在 §9 版本演化日志（不删除）

---

## 0 · 进化引子（核心元规则）

**这不是终态，是基线。** 本方法论永远处于 v1.x 状态——只要有任何一条"现实信号 vs 评分结果"的明显错配，就触发版本迭代。

进化的三档触发机制：

| 档次 | 触发条件 | 调整范围 | 决策路径 |
|------|---------|---------|---------|
| **轻调** (v1.x→v1.x+1) | 单个公司评分与实证（如出货量/IPO 估值/产业链反馈）显著背离；或某维度区分度变差（80%+ 公司集中在同档） | 单维度参数微调、加分项细化 | CC 在评分时发现 → 当场提议 → Doctor 一句话确认 |
| **重构** (v1.x→v2.0) | 评分维度发现根本性缺失（如忽视了"工艺良率波动韧性"）；或与其他渊图模块（如图谱距离算法、tau 锚集）发生不一致 | 新增/删除维度、改权重分布、改公式 | CC 提议设计稿 → Doctor 审阅签字 → 落盘新版本 |
| **范式跃迁** (v2.x→v3.0) | 产业链发生范式转移（如 EUV 解禁、韬定律失败、新技术革命）；或评分对象类型扩展（如纳入海外公司/AI 应用公司） | 整体重设计 | 不在本方法论范围内，需独立立项 |

**反进化的边界**（什么时候**不**该改）：
- 一两个个案与直觉背离 → 个案处理（写入甄别注记），不改方法论
- Doctor 的临时性偏好 → 临时调整本次评分，不入方法论
- 短期市场情绪 → 不影响产业链结构性评分

**进化记录纪律**：每次版本迭代必须在文末「九 · 版本演化日志」区追加条目，含：迭代日期 / 触发事件 / 调整内容 / 评估改进效果（如有）。**绝不悄悄改动，绝不删除历史版本**。

## 1 · 适用范围

**适用**：
- 某条产业链（韬定律、玻璃芯载板、HBM 供应链等）下中国大陆公司的浮现度分析
- 图谱已沉淀 P1+ 信源的公司节点
- 产业链结构性分析（不是股价/估值预测）

**不适用**：
- 投资建议（评分≠估值）
- 海外公司（评分体系基于国产替代视角，海外公司单建对位体系）
- 学术机构纳入时需独立标记（不参与梯队排名，单列）
- 公司品格甄别（用 _meta 标记，不混入评分主体）

## 2 · 六维评分（v1.0 当前规范）

### 2.1 维度定义

| 维度 | 权重 | 量纲 | 评分依据 | 数据源 |
|------|------|------|---------|--------|
| **τ-key** 产业链关键度 | 30 | 0-10 | 距离产业链核心锚（韬定律 4 大组件 / HBM4 核心 / CXL 主体等）的图距：0-1→10 / 2→7-9 / 3→4-6 / 4+→1-3 | `kb.py` 图距 + tau_anchors 集 |
| **稀缺性** 不可替代性 | 25 | 0-10 | 国内独家=10 / 双家=8 / 三家寡头=6-7 / 多家竞争=4-5 / 跟随=1-3 | 帕米尔纪要 + 帕米尔级研报 |
| **国产替代价值** | 20 | 0-10 | 突破国际三巨头/卡脖子环节：破垄断+对位=10 / 部分突破=6-8 / 跟随=3-5 / 无替代价值=0-2 | 国际三巨头收入占比 + BIS 出口管制清单 + 国产替代官方政策 |
| **量产/商业化成熟度** | 10 | 0-10 | 规模量产 SOP=10 / 量产 yield 待提=7-8 / 研发后期+pilot=5-6 / prototype=3 / 概念=1 | 财报 + IPO 招股书 + 帕米尔产业纪要 |
| **图谱浮现度** | 10 | 0-10 | log2(degree+1) 标准化封顶 10 | 渊图 canonical 节点度数 |
| **信息甄别加分** | 5 | −5~+5 | 公司品格典范 +5 / 常规 0 / 信披瑕疵 −5 | `_meta.公司品格典范` / `_meta.信披诚信瑕疵` |

### 2.2 评分公式

```
总分 = τ-key × 3.0 + 稀缺×2.5 + 替代×2.0 + 成熟×1.0 + 浮现×1.0 + 甄别×1.0
最高: 100  最低: -5（信披瑕疵且各维度全 0）
```

### 2.3 梯队映射

| 区间 | 梯队 | 含义 |
|------|------|------|
| ≥ 85 | 🏆 第一梯队 · 产业链核心枢纽 | 国内独家或近独家 + 直接处于产业链核心组件 + 量产成熟 |
| 75-85 | 🥈 第二梯队 · 关键支柱 | 国产替代价值极高 + 量产或接近 |
| 65-75 | 🥉 第三梯队 · 重要参与者 | 产业链有清晰位置但非独家 |
| 55-65 | 第四梯队 · 外围参与者 | 间接相关或量级偏小 |
| < 55 | 第五梯队 · 边缘/警示 | 含信披瑕疵节点单列标记 |

### 2.4 P1 内部细分（v1.1 新增 · 借鉴龙鱼五力数据源优先级）

P1 单档过粗，无法区分"IPO 招股书"与"行业站 datasheet 转引"。v1.1 把 P1 细分为四档：

| 子级 | 信源类型 | 信源示例 | 信源权重（用于多源冲突仲裁） |
|------|---------|---------|----------------------------|
| **P1-A** | 一手原始文件 | IPO 招股书 / 二次问询函回复 / SEC filing / 央企公告 / 官方白皮书 / IEEE 论文 / chinaxiv 论文 | 1.0 |
| **P1-B** | 权威机构研报 | 帕米尔级纪要 / Yole / JPMorgan / Morgan Stanley / TrendForce / SemiAnalysis / EE Times | 0.85 |
| **P1-C** | 一线财经媒体多源 | 一线财经媒体 + 多源交叉（21经济网/财联社/钛媒体/第一财经/Reuters/CNBC/新浪财经/证券时报）| 0.7 |
| **P1-D** | 行业站官方页/datasheet | 厂商官方 datasheet / Adhesives 产品页 / Newsroom 发稿 / BusinessWire 转载 | 0.6 |

**多源冲突仲裁规则**：当多源对同一事实给出不同表述时，按 A>B>C>D 顺序仲裁；同级冲突时按多源数量+独立程度判断；任何子级冲突都必须在节点 `_meta` 中显式记录。

**与 P2 边界**：自媒体（即使影响力大）、单源未交叉、缺乏官方背书的转述 → 全部归 P2，不算 P1。

### 2.5 数据时效约束（v1.1 新增 · 借鉴龙鱼五力 6 个月红线）

| 信源 vintage | 处理 |
|--------------|------|
| ≤ 90 天 | 全权重，无折扣 |
| 90-180 天 | 该维度评分 × **0.85 折扣**，节点 `_meta.信源时效折扣` 显式注记 |
| 180-365 天 | 该维度评分 **必须重新核实**；若无法核实则标"—"（数据缺失），不计权重不给 0 |
| > 365 天 | **该维度评分作废**，必须强制重新核实或标"—" |

**例外**：结构性事实（如"华为是 CXL 创始 9 家之一"这种历史事件）不受时效约束，仅注明历史发生日期即可。受时效约束的是变化性事实（财务/产能/客户/yield/估值/政策状态等）。

### 2.6 打分开口原则（v1.1 新增 · 借鉴龙鱼五力"敢高敢低"）

**在维度内部使用全 0-10 区间，避免回归均值。**

具体落地：
- 若某维度天然高分（如华为 HiSilicon 的 τ-key 10/10），**不要因为"平均化"担忧打 8**
- 若某维度天然低分（如纯应用方公司的稀缺性 1-2 分），**不要因为"鼓励"担忧打 4**
- 一份评分表中**期望见到分布的两端**（10 分和 0 分），不期望全部集中在 5-7
- CC 评分时应有意识检查：若某维度 80% 公司集中在同一档（5/6/7），**主动反思是否在自我审查**

**反模式**：把所有公司维度评分都打成 5-7 的"安全分布"，让总分集中在 60-75 → 这是回归均值偏差，本质是 CC 的自我审查或缺乏数据驱动的判断

### 2.7 量化语言强制（v1.1 新增 · 借鉴龙鱼五力 RULES）

**关键判断必须有数字阈值，禁止纯定性"高/中/低"描述。**

具体落地（每个维度的评分依据中）：

| 维度 | 禁用表述 | 必须使用 |
|------|---------|---------|
| τ-key | "距离韬定律核心很近" | "图距 = 1，supplies 韬定律核心节点 2 条" |
| 稀缺性 | "国内市占率高" | "国内市占 85%，行业第二 5%（具体数字 + 排名）" |
| 国产替代价值 | "替代价值高" | "已通过三星 5nm 工艺认证（带产品名+年份）+ Synopsys 直接对标" |
| 量产成熟度 | "量产成熟" | "2025 年报营收 13.25 亿（+8.4% YoY，财报数）" |
| 浮现度 | "图谱里出现很多" | "节点度=18，τ 锚连接 5 条"（具体数字） |

**例外**：当数据缺失时，应标"信源待补强"并标空（"—"），不允许用"高/中/低"模糊估算。

## 3 · 评分流程（标准操作）

### 3.1 准备阶段

1. **明确产业链锚**：哪些是 tau_anchors（如韬定律 4 大组件 + 关键产品节点 + EDA 概念）
2. **筛选公司池**：从 canonical 中筛 `type=company` 且与产业链相关的中国大陆公司
3. **排除规则**：
   - 排除明确外资（TSMC/Samsung/Intel 等）
   - 排除港资台资（除非已部分在大陆运营且与产业链直接相关）
   - 排除与产业链关系仅为应用方且公司本身不在产业链上的（如消费品/电池等）

### 3.2 评分阶段

1. 计算图距 → τ-key
2. 查询帕米尔纪要/IPO 招股书 → 稀缺性 + 成熟度
3. 对照国际三巨头/BIS 清单 → 替代价值
4. 从 canonical 提取 → 浮现度
5. 查 `_meta` 甄别字段 → 甄别加分
6. **数据缺失处理**：若某维度数据不足以打分，标"—"且**不计权重平均**（不是给 0，因为 0 含义是"无价值"而非"无数据"）

### 3.3 复核阶段

- 三人原则（CC 自评 + 至少一次 Doctor 抽查 + 一次集体对照）
- **常识反推**：评分排序明显与产业链直觉不一致时，停下来检查
- **甄别注记同步**：信披瑕疵/叙事股权/业内关联但无 P1 持股 必须有 _meta 注记，评分**只对甄别加分项扣分**，不影响其他维度

### 3.4 落盘阶段

- 评分表存 `raw/<产业链名>中国公司量化评分-<date>.md`
- 关键变化（梯队跃迁/新晋）回写到对应产业链的 brain/permanent 笔记
- 评分体系版本号标注（v1.0 / v1.1 ...）

## 3.5 结构化输出接口（v1.1 新增 · 与龙鱼五力协同 · Box）

**问题背景**：龙鱼五力 v3.2.1 在分析 A 股标的时强制调用渊图 `kg_context.matched_nodes` 和 `matched_rels`，但当前评分输出仅有"总分 + 梯队 + 关键叙事"非结构化文本，龙鱼五力的"竞争格局（25 分）"维度无法精准提取。

**v1.1 标准输出 JSON schema**（每家评分公司必产）：

```json
{
  "company": "company_Empyrean",
  "scoring_version": "v1.1",
  "scoring_date": "2026-05-28",
  "industry_chain": "韬定律",
  "company_type": "国产替代型",
  "dimensions": {
    "tau_key": {"score": 10, "evidence": "图距=1，supplies 3DICEDAFlow + HiSilicon", "p_grade": "P1-A", "vintage_days": 0},
    "scarcity": {"score": 9, "evidence": "国内唯一 3DIC 全流程", "p_grade": "P1-A", "vintage_days": 60},
    "replacement": {"score": 9, "evidence": "ALPS 8/5/4nm 认证", "p_grade": "P1-B", "vintage_days": 120, "discount": 0.85},
    "maturity": {"score": 7, "evidence": "2025 营收 13.25 亿+8.4%", "p_grade": "P1-A", "vintage_days": 30},
    "graph_degree": {"score": 6, "raw_degree": 12, "evidence": "log2(12+1)=3.7 → 6"},
    "integrity": {"score": 0, "evidence": "常规"}
  },
  "weighted_total": 86.0,
  "tier": "🏆 第一梯队",
  "wuli_consumable_fields": {
    "competition_landscape_input": "国内唯一 3DIC 全流程，700+ 客户，含华为海思深度合作",
    "tech_demand_input": "EDA 软件 2025 营收 +27.02% YoY，技术服务 +74.9%",
    "policy_input": "2025-12 战略并购思尔芯，国产 EDA 整合潮代表",
    "valuation_anchor": "SZ301269（创业板）"
  }
}
```

**关键字段说明**：
- `dimensions.*.p_grade`：每个评分依据信源的 P1 子级（P1-A/B/C/D，详见 §2.4）
- `dimensions.*.vintage_days`：该信源距今天的天数（自动决定折扣，详见 §2.5）
- `dimensions.*.discount`：vintage 折扣系数（仅当 90 < vintage ≤ 180 天时出现）
- `wuli_consumable_fields`：**专为龙鱼五力调用设计的预提取字段**——竞争格局/技术需求/政策/估值锚直接对位龙鱼五力 6 维中的 4 维

**调用方式**（龙鱼五力侧）：
```python
# 龙鱼五力 five_forces_engine_v3.py 在调用 kg_context 时
from yuantu_scoring import get_company_score
score = get_company_score(ts_code="301269.SZ")
if score:
    kg_context.matched_score = score.weighted_total
    kg_context.tier = score.tier
    kg_context.competition_landscape_input = score.wuli_consumable_fields.competition_landscape_input
    # 龙鱼五力维度 3「竞争格局」打分时直接拼接，避免重复梳理
```

**协同价值**：
- 渊图评分 → 龙鱼五力的"输入预处理"，减少重复劳动
- 龙鱼五力评分 → 渊图可反向把 A 股估值/财务节点回写为产业链验证（如：评分高的公司是否估值也高？信息回路闭合）
- 两套体系**层叠协作**而非竞争：渊图看产业链结构，龙鱼五力看投资视角，两者用不同维度组合服务不同决策场景

## 4 · 已知局限（v1.0 自我披露，v1.1 部分缓解 / 部分继承）

**先说出来，避免后续被动**：

1. **τ-key 维度依赖图谱距离**：图谱节点完备度有限，未充分入库的公司其 τ-key 可能被低估（如华虹半导体因图谱节点缺失而推断估算）· **v1.1 状态**：继承未解决
2. **稀缺性是动态变量**：随竞争对手出现而下降（华大九天的 3DIC 全流程独家地位会随概伦/广立微/芯和半导体补齐而下降）· **v1.1 状态**：通过 vintage 90/180 天约束 + 季度复评机制部分缓解
3. **国产替代价值受出口管制周期影响**：2025-05→07 BIS 短周期管制证明替代价值可在数月内波动 · **v1.1 状态**：vintage ≤ 90 天约束 + 触发性即时复评机制部分缓解
4. **量产成熟度对非上市公司不透明**：青禾晶元/SiCarrier 等非上市公司的量产细节难核 · **v1.1 状态**：通过 P1-A/B/C/D 细分（强制要求 P1-A 一手源）部分缓解；非上市公司若无 P1-A 一手源，该维度标"—"不强行打分
5. **图谱浮现度有"先发偏差"**：早入库公司因被多次引用而度数高，新入库公司度数偏低（如芯华章度=0 是 patch 漏建边导致而非真无关）· **v1.1 状态**：继承未解决，留待 v1.2+ 通过"度数 ÷ 入库时长"正规化处理
6. **甄别加分单向性**：当前只对"典范"+5 与"瑕疵"−5，未覆盖中间灰度状态 · **v1.1 状态**：继承未解决，留待 v1.2 扩展为 ±10 五档
7. **学术机构与商业公司同表打分**：北大集成电路学院（学术 originator）以同公式评分会被低估 · **v1.1 状态**：继承未解决，留待 v2.0 重构（单列学术机构子表）
8. **评分对"潜力"敏感度低**：当前体系偏向"已实证"，对早期 prototype（如北大 EDA 工具）的颠覆潜力评估不足 · **v1.1 状态**：继承未解决，留待 v2.0 场景化适配权重（"技术原创型"权重组）
9. **信源时效性**：评分基于截至该日的图谱状态 · **v1.1 状态**：通过 §2.5 vintage 约束规则化，已解决
10. **维度内打分定性偏强**：v1.0 仅 0-10 文字描述，缺乏阈值约束 · **v1.1 状态**：通过 §2.7 量化语言强制部分缓解，但完整解决需 v1.2 子项细分
11. **回归均值偏差**：v1.0 未显式强调"敢高敢低" · **v1.1 状态**：通过 §2.6 打分开口原则显式化，已解决

## 5 · 已识别的可能进化方向（v1.1+ 候选）

下列方向是 v1.0 落盘时已识别的可能进化项，**尚未实施**——等触发条件成熟时按 §0 元规则推进：

### 5.1 维度细化候选

- **τ-key 权重分拆**：把"距离图谱锚"和"是否在产业链核心组件路径"分成两个子维度，避免单一图距过度概括语义
- **稀缺性时序化**：引入"稀缺性轨迹"（5 年前/3 年前/当前/未来 3 年）替代单点稀缺性
- **成熟度细分**：把"研发→pilot→量产 ramp→规模 SOP"细分为 5 档而非当前 3 档
- **甄别加分细化**：从 ±5 二档扩到 ±10 五档，覆盖"严重瑕疵 −10 / 轻微 −5 / 中性 0 / 轻微典范 +5 / 严重典范 +10"
- **新维度**："工艺良率波动韧性"（应对出口管制反复）/ "客户分散度"（避免单客户依赖）/ "海外营收占比"（衡量国际竞争力）

### 5.2 算法进化候选

- **非线性加权**：当前线性加权可能掩盖"短板效应"——如某公司在 τ-key 满分但量产为 0，仍可能拿 70 分。可引入"任一维度 < 3 则总分封顶 60"等约束
- **图距算法改进**：当前 BFS 简单图距未考虑边权重和边类型差异——supplies/competes_with/measured_by 三种边对"产业链关键度"贡献不同
- **置信度区间**：每个维度评分配 [low, high] 区间而非单点，最终输出"评分 ± 不确定性"
- **聚类自适应**：根据评分分布自动调整梯队边界，避免固定边界导致"边界附近的公司频繁跨级"

### 5.3 评分对象扩展候选

- **学术机构子表**：单独的"学术 originator 评分表"，维度为"originator 价值 / 产业转化潜力 / 国际声誉"
- **海外公司对位表**：对位中国公司的国际同行评分，便于"国产替代差距"量化
- **AI 应用层公司**：如智谱/月之暗面/MiniMax 等"用 AI 算力"而非"造 AI 算力"的公司——需要独立评分维度
- **公司联盟评分**：CXL/UALink 等联盟作为"集合体"的评分

### 5.4 与渊图其他模块集成候选

- **与 `kb.py --signal` 集成**：把"市场信号节点"作为评分加分维度（公司若位于多个 demand_surge/supply_shock 节点的上游则加分）
- **与 `signal_impact_chain.py` 集成**：把"下游影响链长度"作为产业链关键度的辅助证据
- **与 quarterly_recheck 集成**：评分自动跟随图谱季度复检更新

### 5.5 v1.2 候选 · 产业链场景子方法论范式（2026-05-29 试点）

v1.2 计划把**维度内子项细分**（T1 候选）实现为**按产业链场景的专属子方法论**。每个场景子方法论提供：
- 该产业链特有的 N 维框架（替代主方法论的通用 6 维）
- 每维度内的 scoring_scaffold（带数字阈值）
- 与主方法论 6 维的归一映射公式

**首个试点**：[[CPO时代企业评估框架]]（2026-05-29 落盘）——光通信场景 8 维框架

**v1.2 完整规范的待定项**：① 主方法论 6 维 ↔ 子方法论 N 维的归一公式 ② 子方法论之间的横向比较规则（如 EDA 公司与封装公司是否可同表打分）③ 当某公司跨产业链时（如华为既在 EDA 又在工艺）用哪个子方法论

## 6 · 反模式与禁忌

### 6.1 不要做的事

- ❌ **看到 Doctor 不喜欢的评分就改评分体系参数**：本方法论是对产业结构的客观描述，不是为了"评分 = Doctor 直觉"
- ❌ **用评分判断股价**：评分高 ≠ 股价高（见英维克 48.5 vs 历史股价波动）
- ❌ **跨产业链横向比较单家公司评分**：同一公司在韬定律评 92 不代表在医疗芯片产业链也 92
- ❌ **隐藏评分体系版本**：每份评分表必须标 v1.x，不能让读者以为"评分=终态真理"
- ❌ **把评分作为入库优先级唯一依据**：评分仅是产业链结构的一个视角，入库优先级还要考虑信源 P 级、图谱完整性、甄别需求等

### 6.2 评分体系的元防御

- 当某公司被打高分后**反向作为入库参考**——容易形成"高分→更多入库→度数更高→评分自我强化"的死循环。**防御**：度数维度仅占 10 分权重，且配合"度数/入库时长"正规化
- 信息甄别加分项**仅服务于产业品格判别**，不影响产业能力评分——叙事股权≠商业能力的公司（如盛合晶微、SiCarrier、鸿芯微纳）业务能力照常打分
