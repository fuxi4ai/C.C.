# 渊图国产公司量化评分方法

## 0 · 进化引子（核心元规则）

**这不是终态，是基线。** 本方法论永远处于 v1.x 状态——只要有任何一条"现实信号 vs 评分结果"的明显错配，就触发版本迭代。

进化的三档触发机制：

| 档次 | 触发条件 | 调整范围 | 决策路径 |
|------|---------|---------|---------|
| **轻调** (v1.x→v1.x+1) | 单个公司评分与实证（如出货量/IPO 估值/产业链反馈）显著背离；或某维度区分度变差（80%+ 公司集中在同档） | 单维度参数微调、加分项细化 | CC 在评分时发现 → 当场提议 → Doctor 一句话确认 |
| **重构** (v1.x→v2.0) | 评分维度发现根本性缺失（如忽视了"工艺良率波动韧性"）；或与其他渊图模块（如图谱距离算法、tau 锚集）发生不一致 | 新增/删除维度、改权重分布、改公式 | CC 提议设计稿 → Doctor 审阅签字 → 落盘新版本 |
| **范式跃迁** (v2.x→v3.0) | 产业链发生范式转移（如 EUV 解禁、韬定律失败、新技术革命）；或评分对象类型扩展（如纳入海外公司/AI 应用公司） | 整体重设计 | 不在本方法论范围内，需独立立项 |

**反进化的边界**（什么时候**不**该改）：
- 一两个个案与直觉背离 → 个案处理（写入甄别注记），不改方法论
- Doctor 的临时性偏好 → 临时调整本次评分，不入方法论
- 短期市场情绪 → 不影响产业链结构性评分

**进化记录纪律**：每次版本迭代必须在文末「九 · 版本演化日志」区追加条目，含：迭代日期 / 触发事件 / 调整内容 / 评估改进效果（如有）。**绝不悄悄改动，绝不删除历史版本**。

## 1 · 适用范围

**适用**：
- 某条产业链（韬定律、玻璃芯载板、HBM 供应链等）下中国大陆公司的浮现度分析
- 图谱已沉淀 P1+ 信源的公司节点
- 产业链结构性分析（不是股价/估值预测）

**不适用**：
- 投资建议（评分≠估值）
- 海外公司（评分体系基于国产替代视角，海外公司单建对位体系）
- 学术机构纳入时需独立标记（不参与梯队排名，单列）
- 公司品格甄别（用 _meta 标记，不混入评分主体）

## 2 · 六维评分（v1.0 当前规范）

### 2.1 维度定义

| 维度 | 权重 | 量纲 | 评分依据 | 数据源 |
|------|------|------|---------|--------|
| **τ-key** 产业链关键度 | 30 | 0-10 | 距离产业链核心锚（韬定律 4 大组件 / HBM4 核心 / CXL 主体等）的图距：0-1→10 / 2→7-9 / 3→4-6 / 4+→1-3 | `kb.py` 图距 + tau_anchors 集 |
| **稀缺性** 不可替代性 | 25 | 0-10 | 国内独家=10 / 双家=8 / 三家寡头=6-7 / 多家竞争=4-5 / 跟随=1-3 | 帕米尔纪要 + 帕米尔级研报 |
| **国产替代价值** | 20 | 0-10 | 突破国际三巨头/卡脖子环节：破垄断+对位=10 / 部分突破=6-8 / 跟随=3-5 / 无替代价值=0-2 | 国际三巨头收入占比 + BIS 出口管制清单 + 国产替代官方政策 |
| **量产/商业化成熟度** | 10 | 0-10 | 规模量产 SOP=10 / 量产 yield 待提=7-8 / 研发后期+pilot=5-6 / prototype=3 / 概念=1 | 财报 + IPO 招股书 + 帕米尔产业纪要 |
| **图谱浮现度** | 10 | 0-10 | log2(degree+1) 标准化封顶 10 | 渊图 canonical 节点度数 |
| **信息甄别加分** | 5 | −5~+5 | 公司品格典范 +5 / 常规 0 / 信披瑕疵 −5 | `_meta.公司品格典范` / `_meta.信披诚信瑕疵` |

### 2.2 评分公式

```
总分 = τ-key × 3.0 + 稀缺×2.5 + 替代×2.0 + 成熟×1.0 + 浮现×1.0 + 甄别×1.0
最高: 100  最低: -5（信披瑕疵且各维度全 0）
```

### 2.3 梯队映射

| 区间 | 梯队 | 含义 |
|------|------|------|
| ≥ 85 | 🏆 第一梯队 · 产业链核心枢纽 | 国内独家或近独家 + 直接处于产业链核心组件 + 量产成熟 |
| 75-85 | 🥈 第二梯队 · 关键支柱 | 国产替代价值极高 + 量产或接近 |
| 65-75 | 🥉 第三梯队 · 重要参与者 | 产业链有清晰位置但非独家 |
| 55-65 | 第四梯队 · 外围参与者 | 间接相关或量级偏小 |
| < 55 | 第五梯队 · 边缘/警示 | 含信披瑕疵节点单列标记 |

## 3 · 评分流程（标准操作）

### 3.1 准备阶段

1. **明确产业链锚**：哪些是 tau_anchors（如韬定律 4 大组件 + 关键产品节点 + EDA 概念）
2. **筛选公司池**：从 canonical 中筛 `type=company` 且与产业链相关的中国大陆公司
3. **排除规则**：
   - 排除明确外资（TSMC/Samsung/Intel 等）
   - 排除港资台资（除非已部分在大陆运营且与产业链直接相关）
   - 排除与产业链关系仅为应用方且公司本身不在产业链上的（如消费品/电池等）

### 3.2 评分阶段

1. 计算图距 → τ-key
2. 查询帕米尔纪要/IPO 招股书 → 稀缺性 + 成熟度
3. 对照国际三巨头/BIS 清单 → 替代价值
4. 从 canonical 提取 → 浮现度
5. 查 `_meta` 甄别字段 → 甄别加分
6. **数据缺失处理**：若某维度数据不足以打分，标"—"且**不计权重平均**（不是给 0，因为 0 含义是"无价值"而非"无数据"）

### 3.3 复核阶段

- 三人原则（CC 自评 + 至少一次 Doctor 抽查 + 一次集体对照）
- **常识反推**：评分排序明显与产业链直觉不一致时，停下来检查
- **甄别注记同步**：信披瑕疵/叙事股权/业内关联但无 P1 持股 必须有 _meta 注记，评分**只对甄别加分项扣分**，不影响其他维度

### 3.4 落盘阶段

- 评分表存 `raw/<产业链名>中国公司量化评分-<date>.md`
- 关键变化（梯队跃迁/新晋）回写到对应产业链的 brain/permanent 笔记
- 评分体系版本号标注（v1.0 / v1.1 ...）

## 4 · 已知局限（v1.0 自我披露）

**先说出来，避免后续被动**：

1. **τ-key 维度依赖图谱距离**：图谱节点完备度有限，未充分入库的公司其 τ-key 可能被低估（如华虹半导体因图谱节点缺失而推断估算）
2. **稀缺性是动态变量**：随竞争对手出现而下降（华大九天的 3DIC 全流程独家地位会随概伦/广立微/芯和半导体补齐而下降）
3. **国产替代价值受出口管制周期影响**：2025-05→07 BIS 短周期管制证明替代价值可在数月内波动
4. **量产成熟度对非上市公司不透明**：青禾晶元/SiCarrier 等非上市公司的量产细节难核
5. **图谱浮现度有"先发偏差"**：早入库公司因被多次引用而度数高，新入库公司度数偏低（如芯华章度=0 是 patch 漏建边导致而非真无关）—— 需配合"度数 ÷ 入库时长"做正规化
6. **甄别加分单向性**：当前只对"典范"+5 与"瑕疵"−5，未覆盖中间灰度状态（如"业内关联无 P1 持股"应该给中性 0 还是轻微负？）
7. **学术机构与商业公司同表打分**：北大集成电路学院（学术 originator）以同公式评分会被低估（成熟度天然低），建议下次单列学术机构子表
8. **评分对"潜力"敏感度低**：当前体系偏向"已实证"，对早期 prototype（如北大 EDA 工具）的颠覆潜力评估不足
9. **信源时效性**：评分基于截至该日的图谱状态，超过 90 天的数据可能失效

## 5 · 已识别的可能进化方向（v1.1+ 候选）

下列方向是 v1.0 落盘时已识别的可能进化项，**尚未实施**——等触发条件成熟时按 §0 元规则推进：

### 5.1 维度细化候选

- **τ-key 权重分拆**：把"距离图谱锚"和"是否在产业链核心组件路径"分成两个子维度，避免单一图距过度概括语义
- **稀缺性时序化**：引入"稀缺性轨迹"（5 年前/3 年前/当前/未来 3 年）替代单点稀缺性
- **成熟度细分**：把"研发→pilot→量产 ramp→规模 SOP"细分为 5 档而非当前 3 档
- **甄别加分细化**：从 ±5 二档扩到 ±10 五档，覆盖"严重瑕疵 −10 / 轻微 −5 / 中性 0 / 轻微典范 +5 / 严重典范 +10"
- **新维度**："工艺良率波动韧性"（应对出口管制反复）/ "客户分散度"（避免单客户依赖）/ "海外营收占比"（衡量国际竞争力）

### 5.2 算法进化候选

- **非线性加权**：当前线性加权可能掩盖"短板效应"——如某公司在 τ-key 满分但量产为 0，仍可能拿 70 分。可引入"任一维度 < 3 则总分封顶 60"等约束
- **图距算法改进**：当前 BFS 简单图距未考虑边权重和边类型差异——supplies/competes_with/measured_by 三种边对"产业链关键度"贡献不同
- **置信度区间**：每个维度评分配 [low, high] 区间而非单点，最终输出"评分 ± 不确定性"
- **聚类自适应**：根据评分分布自动调整梯队边界，避免固定边界导致"边界附近的公司频繁跨级"

### 5.3 评分对象扩展候选

- **学术机构子表**：单独的"学术 originator 评分表"，维度为"originator 价值 / 产业转化潜力 / 国际声誉"
- **海外公司对位表**：对位中国公司的国际同行评分，便于"国产替代差距"量化
- **AI 应用层公司**：如智谱/月之暗面/MiniMax 等"用 AI 算力"而非"造 AI 算力"的公司——需要独立评分维度
- **公司联盟评分**：CXL/UALink 等联盟作为"集合体"的评分

### 5.4 与渊图其他模块集成候选

- **与 `kb.py --signal` 集成**：把"市场信号节点"作为评分加分维度（公司若位于多个 demand_surge/supply_shock 节点的上游则加分）
- **与 `signal_impact_chain.py` 集成**：把"下游影响链长度"作为产业链关键度的辅助证据
- **与 quarterly_recheck 集成**：评分自动跟随图谱季度复检更新

## 6 · 反模式与禁忌

### 6.1 不要做的事

- ❌ **看到 Doctor 不喜欢的评分就改评分体系参数**：本方法论是对产业结构的客观描述，不是为了"评分 = Doctor 直觉"
- ❌ **用评分判断股价**：评分高 ≠ 股价高（见英维克 48.5 vs 历史股价波动）
- ❌ **跨产业链横向比较单家公司评分**：同一公司在韬定律评 92 不代表在医疗芯片产业链也 92
- ❌ **隐藏评分体系版本**：每份评分表必须标 v1.x，不能让读者以为"评分=终态真理"
- ❌ **把评分作为入库优先级唯一依据**：评分仅是产业链结构的一个视角，入库优先级还要考虑信源 P 级、图谱完整性、甄别需求等

### 6.2 评分体系的元防御

- 当某公司被打高分后**反向作为入库参考**——容易形成"高分→更多入库→度数更高→评分自我强化"的死循环。**防御**：度数维度仅占 10 分权重，且配合"度数/入库时长"正规化
- 信息甄别加分项**仅服务于产业品格判别**，不影响产业能力评分——叙事股权≠商业能力的公司（如盛合晶微、SiCarrier、鸿芯微纳）业务能力照常打分

## 7 · 与渊图三大方法论范式的联动

| 范式 | 来源 | 在评分中的体现 |
|------|------|----------------|
| **信息甄别三范式**（财报/股权/技术能力） | 2026-05-23 英维克 + 2026-05-28 盛合晶微 + 2026-05-28 华为 EDA 自研叙事 | "甄别加分"维度（信披瑕疵 −5）+ 业务能力评分不受叙事问题影响 |
| **公司品格双向范式**（典范+/瑕疵−） | 2026-05-28 沃尔德 vs 英维克 | "甄别加分"维度（典范 +5） |
| **学术 originator 范式** | 2026-05-23 MIT/PoliBa + 2026-05-28 北大 IC | 学术机构允许入榜但成熟度天然受限；建议 v1.1+ 单列学术子表 |

## 8 · 标准产出物清单

每次评分至少产出：

1. **评分表本身**（raw/<产业链>中国公司量化评分-<date>.md）
2. **梯队总结**（5 段）+ **全景特征**（5 大共同特征）
3. **评分体系版本号**（v1.x）+ **本次评分的特殊调整说明**（若有）
4. **下次复评清单**（哪些公司需触发性复评 + 触发条件）
5. **评分边界与局限**（继承本方法论 §4 + 本次特殊局限）
