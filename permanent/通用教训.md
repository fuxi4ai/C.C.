# 通用教训（跨项目）

从 DVA、O MY HTML、政治经济学等项目 GOTCHAS 提炼的**跨项目共性规律**。

---

## 代码工程

**函数存在 ≠ 函数被调用**
注册路由时，新增 case 必须同时在 switch 里注册，函数实现完整但 case 遗漏等于死码。（→ [[DVA]] BUG-20260501-001）

**移动文件后必须全局更新引用**
迁移文件到子目录后，立即 `grep -r` 所有 require/import 并同步更新，移动后立跑 `node xxx help` 验证。（→ [[DVA]] BUG-20260501-003）

**相对路径调试：先打印再运行**
含 `__dirname`/相对路径的代码，修改后用 `console.log(path.resolve(...))` 肉眼确认，再跑完整流程。（→ [[DVA]] BUG-20260424-002）

**Python `Path("~")` 不会自动展开**
需显式 `.expanduser()` 或用 `os.homedir()`（Node.js 端）展开为绝对路径后再传给 Python。（→ [[DVA]] BUG-20260505-003）

**外部资源清理放 `finally` 块**
云存储、临时文件、数据库连接的清理逻辑必须在 `finally` 块，不能依赖正常流程末尾。（→ [[DVA]] INFRA-20260504-004）

**版本升级用 `sed` 批量，替换后 `grep` 验证**
版本号更新是"一次性全局操作"，不逐文件改；替换后用 `grep -r "v旧版本"` 确认无遗漏。（→ [[DVA]] BUG-20260422-003）

## Cowork 工程（CC 专属）

**任何压缩/覆盖性写操作先在 `outputs/` 完成，再 copy 到项目目录**
挂载目录对 zip 创建/覆盖有权限限制，直接在挂载目录操作会失败并留下残留文件。（→ [[O MY HTML]]）

**`computer://` 链接只在 Cowork 聊天界面有效**
独立打开的 HTML（`file://` 或浏览器）不识别 `computer://` 协议。离开聊天的 HTML 工具改用 `file://` 或 JS 复制命令。（→ [[O MY HTML]]）

**Doctor 用 APIYI 转接模式，官方前端 UI 特性不可用**
Save skill 卡片、present_files 渲染、artifact UI 等依赖 Claude 官方聊天前端，APIYI 转接绕过了这一层，任何 UI 触发必须放到用户本地。（→ [[O MY HTML]]）

**多文件结构 skill 在 Cowork 退化为单 SKILL.md**
Cowork 的 skill 识别只挂载 SKILL.md，子目录不保留。多文件 skill 必须在项目里保留完整源码副本作为 fallback。（→ [[O MY HTML]]）

## 分析方法论（政治经济学）

**万能概念 = 不可证伪 = 无价值**
一个概念若能解释任意对立结果，立即停下来要求明确反例条件。（→ [[政治经济学]] G-01）

**"主动维持"与"被动不解决"需要不同的证据**
因果机制类型不同，对应的证据标准截然不同，不能混用。（→ [[政治经济学]] G-02）

**对话共振≠证据积累**
一次对话内概率持续单向上调是"上调偏误"，不是证据积累。主动引入独立质疑。（→ [[政治经济学]] G-03）

**受益分析必须包含完整成本-收益表**
只列收益不列成本的分析是片面清单，净收益 = 收益 - 成本。（→ [[政治经济学]] G-05）

## 数据与 API

**SDK 枚举 vs 字符串：遵循类型定义**
有枚举定义就用枚举，不传裸字符串，如 `tos.HttpMethodType.Http_Method_Get`。（→ [[DVA]] INFRA-20260504-003）

**第三方 AI 产品间接处理平台内容风险高**
字幕获取应走原始数据路径（下载 + 本地/API 转写），不依赖 AI 多模态理解，可被平台随时切断。（→ [[DVA]] INFRA-20260504-001）

**DashScope 兼容模式不是所有功能的超集**
音频转写须用原生 API；模型选型须验证 URL 要求，不能假设"任意公网 URL 均可用"。（→ [[DVA]] INFRA-20260504-002）

## 文档规范

**不删除的旧文件必须在文件头标注弃用**
给出替代命令，否则旧文件无声存活多个版本。（→ [[DVA]] BUG-20260501-005）

**每次发版将已完成任务文档视为临时文件清理**
只保留"当前仍然有效"的文档，历史规划文档随版本一起清理。（→ [[DVA]] BUG-20260501-006）

**新增统一入口时，旧手动步骤降级为"调试专用"**
不删除，但在文档中明确标注用途变更。（→ [[DVA]] BUG-20260501-007）

## Cowork 环境

**`/save` 等命令在 skill 未安装前报 "unknown skill"**
CLAUDE.md 里的文字描述不等于可执行的 slash 命令。命令要生效必须有对应的已安装 skill。在 brain-save skill 创建并安装前，用 CC 手动执行 save 行为作为替代。（→ 2026-05-14，MAX_HANDOFF P0 任务之一）
