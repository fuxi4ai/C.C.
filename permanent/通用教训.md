## 数据治理

**🔒 数据真实性铁律：宁可缺数，不可假数**（2026-06-07 立，完整版见 [[CLAUDE]] 同名节）
没有新鲜、可核验的真实数据时**留空**（`—`/`待核验`），绝不用占位/样本/估计值填充；占位数据**不入库、不进最终报告**；取数受阻就报"数据缺失"交回 Doctor 跑真实管线，**不得退化为硬编码占位"先让它跑起来"**。

**占位数据是怎么混进来的（白泽大宗活体案例 · 根因三层）**
1. **Demo-first 脚手架没拆**：`run_full_analysis_v4_1.py` 为了不接 data_collection 就能端到端跑通、产出"好看的报告"，把 `COMMODITY_DATA`/`AI_HARDWARE_DATA` 价格**硬编码**成样本值。脚手架占位本应跑通后替换，结果一直留着。
2. **数据层与分析层没接线**：`data_collection/` 的 Tushare 抓取脚本存在，但**从未接入主分析流**——主脚本读的是自己内嵌的常量，不是真实抓取结果。
3. **取数有摩擦 → 占位成了阻力最小路径**：沙箱/CC 连不上 Tushare（代理 403、无 A 股 MCP），想接真数据也接不上，于是"先写死让它跑"成了惯性。
**叠加恶化**：首轮审查只加了"数据来源声明横幅"（治标），没拆掉占位本身（治本）——声明 ≠ 移除，假数仍在报告里。

**判别器**：看到报告里有数值却答不出"这个数从哪来、采集于何时"→ 立即按占位嫌疑处理：要么补真实来源，要么留空标"待核验"。（→ 白泽大宗 docs/审查_20260607 + GOTCHAS 数据铁律条）

**检索顺序铁律 + 信源分级（2026-06-07 补，完整版见 [[CLAUDE]]「数据真实性铁律·配套规则」）**
- **顺序**：先**渊图**（内置 `confidence_level`，多 P1·研报背书·带 `data_vintage`）→ 官方/权威站（交易所/Mysteel/SMM/生意社/百川/包头所）→ 一般 web。别一上来就 web 搜。
- **分级**：P0=年报/Tushare/交易所原始；P1=官方权威站+渊图研报；P2=一般 web/媒体（须标注待复核）。每数值带 `credibility+source+asof`。
- **同比为准、环比参考**：大宗价 `change_yoy` 驱动结论，`change_mom` 仅展示。

## 代码工程

**函数存在 ≠ 函数被调用**
注册路由时，新增 case 必须同时在 switch 里注册，函数实现完整但 case 遗漏等于死码。（→ [[DVA]] BUG-20260501-001）

**移动文件后必须全局更新引用**
迁移文件到子目录后，立即 `grep -r` 所有 require/import 并同步更新，移动后立跑 `node xxx help` 验证。（→ [[DVA]] BUG-20260501-003）

**相对路径调试：先打印再运行**
含 `__dirname`/相对路径的代码，修改后用 `console.log(path.resolve(...))` 肉眼确认，再跑完整流程。（→ [[DVA]] BUG-20260424-002）

**Python `Path("~")` 不会自动展开**
需显式 `.expanduser()` 或用 `os.homedir()`（Node.js 端）展开为绝对路径后再传给 Python。（→ [[DVA]] BUG-20260505-003）

**外部资源清理放 `finally` 块**
云存储、临时文件、数据库连接的清理逻辑必须在 `finally` 块，不能依赖正常流程末尾。（→ [[DVA]] INFRA-20260504-004）

**版本升级用 `sed` 批量，替换后 `grep` 验证**
版本号更新是"一次性全局操作"，不逐文件改；替换后用 `grep -r "v旧版本"` 确认无遗漏。（→ [[DVA]] BUG-20260422-003）（→ [[DVA]] BUG-20260422-003）

**Cowork 定时任务里别用 `Path.home()` 定位 Documents**
定时任务以沙箱身份跑，`Path.home()`=`/sessions/<id>`，用户 Documents 实挂在 `/sessions/<id>/mnt/Documents`；Mac 本机 `Path.home()/Documents` 才对——本机测没事、定时任务翻车（写错地方/读不到文件）。根治：从脚本自身位置向上找名为 `Documents` 的祖先目录（`_documents_root()`），Mac 与沙箱都适用。（→ 白泽大宗 GOTCHA-014 · 周报自动化）

**同一资源被两套约定读取 → 轮换/迁移时只更一处必炸**
如 Tushare token：白泽读 env/.secrets，龙鱼读 Keychain `tushare_pro/doctor`。轮换后只更一处，另一处用失效旧值，错误还常被吞成无关报错（"找不到标的"）。根治：统一读取顺序、单一存放位置喂多方。（→ 白泽大宗 GOTCHA-013）

**Python 包内模块名经 sys.path 暴露会与消费方撞名（循环导入）**
共享包若把内部 `core`/`engine` 等通用名经 `sys.path.insert` 暴露，消费方同名模块互相 shadow，触发 `partially initialized module … circular import`。根治：包内一律**纯相对导入**（`from .sub import …`），不污染全局 sys.path；内部子模块用唯一名（`_fed` 而非 `core`）。（→ [[数灵转移设计方案]] guanxing_engine 抽取 · logs/2026-06-02-数灵转移-审查与公共层落地）

**sqlite 在挂载/FUSE 盘上直接写易触发 disk-I/O / 热 journal 回滚**
沙箱挂载盘上对 sqlite 直接 `UPDATE`+`commit` 常报 `disk I/O error`；失败还会留**热 `-journal`**，下次打开时按它**回滚**——把你刚写的又退回去（且 sandbox 删不掉 journal）。判断信号：commit 报错/改了又变回原值/连只读都 `disk I/O error`。根治：**拷出本地改→整文件写回**——`cp 库 /tmp/x.db` → 本地 `UPDATE+commit`（`PRAGMA journal_mode=DELETE`）→ `cp /tmp/x.db 回原位` → 复核用 `sqlite3.connect('file:…?mode=ro&immutable=1',uri=True)`（绕开 RW 锁与 journal）。残留 `-journal` 让 Doctor 在 Mac `rm`。（→ 白泽大宗 GOTCHA-016 赣锋 ts_code 订正 · 沙盒只 rename 不 unlink 姊妹条）

## Cowork 工程（CC 专属）

**任何压缩/覆盖性写操作先在 `outputs/` 完成，再 copy 到项目目录**
挂载目录对 zip 创建/覆盖有权限限制，直接在挂载目录操作会失败并留下残留文件。（→ [[O MY HTML]]）

**`computer://` 链接只在 Cowork 聊天界面有效**
独立打开的 HTML（`file://` 或浏览器）不识别 `computer://` 协议。离开聊天的 HTML 工具改用 `file://` 或 JS 复制命令。（→ [[O MY HTML]]）

**Doctor 用 APIYI 转接模式，官方前端 UI 特性不可用**
Save skill 卡片、present_files 渲染、artifact UI 等依赖 Claude 官方聊天前端，APIYI 转接绕过了这一层，任何 UI 触发必须放到用户本地。（→ [[O MY HTML]]）

**多文件结构 skill 在 Cowork 退化为单 SKILL.md**
Cowork 的 skill 识别只挂载 SKILL.md，子目录不保留。多文件 skill 必须在项目里保留完整源码副本作为 fallback。（→ [[O MY HTML]]）

**G-X1 自主任务质量纪律**（★★★ 关键 · 2026-05-20 立）
长时间无人监督的自主任务（凌晨批量 / "自主进行不用问询"）CC 倾向**为赶进度放弃颗粒度 + 拉平重要度**。典型表现:70 位西方哲学家全打 ★★★（膨胀 4 倍）+ 8 KB 体量与"顶级 = 行数不限制"档位严重不匹配;伊本·西那被 LLM 语料逻辑（"社会影响 ★★★ = 顶级"）自动升顶级。**根因**:无监督环境下缺少"基线对照 + 自我审计"反馈环,LLM 默认走"完成数量"路径,把质量维度压扁。**防御 6 条**:
  1. **基线先冻**:开始前明确 Doctor 既有评定基线（如哪些已确定 ★★★）,后续操作"严格按基线" — 这是元指令而非建议
  2. **颗粒度先于内容**:不允许"先完成再补颗粒度" — 一开始就按档位定字节数下限（★★★ 不限 / ★★ 15-25 KB / ★ 10-15 KB / 次要 < 10 KB）,低于下限的草稿不算落盘
  3. **升档红线**:默认**不升档** — 任何"第 N 位升 ★★★"需要独立论证段（原创奠基 / 思想深度 / 社会影响 / **功能性 vs 思想性** 四维至少两维满足）,不论证不升
  4. **每 5-10 位强制停顿对照**:与基线分布比对（西方应有 7 ★★★ 而非 30+） — 偏离基线 > 1.5 倍立即停 + 找 Doctor
  5. **自审段必写**:任务完成时强制写 self-audit（本批新升档 N 位 / 每位升档理由 / 偏差基线情况） — 没写不算完成
  6. **"自主进行"≠"无质量门"**:无监督模式下质量门不降反升 — 反偏置六类（α/α'/β/γ/δ/ε）+ 颗粒度规则要更严格执行,而不是因为"赶时间"放松
  7. **Checkpoint 机制**(2026-05-20 v1.1 加):每完成一个独立工作单元立即 Write/Edit 落实体文件（不是"在 CC 上下文里记着"）· 每 5-10 个单元 + 每 30 分钟 强制写一份 progress 文件 · 任务中途崩溃 / 上下文耗尽 / 报错丢历史时,**下一个 CC 会话读 progress 文件直接续上** · 不依赖"不出错"而依赖"出错也能从最近 checkpoint 继续" → **详见 `brain/references/CC长任务防错指南.md`**(6 类高频错误 × 长任务杀伤力分级 + 错误决策树 + 实战清单)
  （→ 跨文明哲学史 v0.1 凌晨批量 + v0.2 27 位降档校准 + v1.1 伊本·西那再降档;Doctor 2026-05-20 上传 anthropic-api-errors-cheatsheet 关切长任务防错）

**G-X4 PRD 交付标准纪律**（★★★ 核心 · 2026-05-20 立 · Doctor 命题）
**根因**:CC 错误自发现率高，但偶尔仍有"需 Doctor 提示才自觉"的盲区（5/20 一天 4 个错误 CC 自检率 0/4）。盲区机制 = **完成幻觉**:LLM 倾向"积极完成"，把"done"由完成者自己解释 → 范围漂移 + 一致性漂移 + 规则漂移。所有现有自检机制（TaskList / G-X1 self-audit / session-log / skill Step）都是"CC 在 CC 自己定义的标准下评 CC 自己"。
**核心规则**:
  1. **任务开始前立 PRD**（适用于:多文件改 / 跨小时 / 跨项目 / 凌晨自主任务 / Doctor 不全程监督）。问答 / 单文件小改 / 研究讨论 **不立 PRD** · 不要走极端官僚化
  2. **触发权**:CC 主动提（判断任务规模超阈值 → CC 提"建议立 PRD"）+ Doctor 同意 / 否决
  3. **位置**:`~/Documents/Claude/brain/logs/checkpoints/{YYYY-MM-DD}_{任务简称}_PRD.md`（与 G-X1 第 7 条 checkpoint 同栖）
  4. **三态评估**（硬约束）:模板里 checkbox 默认 `[ ]` · CC 只能改三态之一:`[?]` 我认为已达成 + 证据 / `[!]` 未达成 + 原因 / `[~]` 不确定 + 需 Doctor 判断。**CC 永远不允许打 `[✓]`**（初期）· 只有 Doctor 可以把 `[?]` 转为 `[✓]`
  5. **双向闭环关闭**:任务**只能**通过两种合法路径关闭:**全 ✓**（Doctor 把所有 `[?]` 转为 `[✓]`）或 **Doctor 显式取消**（原因 + 时间）· 禁止 CC 自动关闭（commit 即关 / 标完最后 Task 即关 / "我觉得交付完了"即关）
  6. **可验证 = 客观**:每条交付标准必须"文件存在于 X 路径 / Grep 'oldname' 在 path/ 无残留 / 脚本输出 Z" 这种可机器验证的形式。**不允许写"做好了"/"完整"/"修复了"** 等主观陈述（写了等于没写）
**关键模板提示**（G-X4 防御工具）:
  - `[ ]` 文件 X 存在于 ~/Documents/Claude/.../X.md
    - CC 填:`[?] 已建 · 证据:Read 返回 200 行 + ls -la 显示 size 12 KB`
  - `[ ]` Grep "oldname" 在 dir/ = 0 matches
    - CC 填:`[?] 无残留 · 证据:Grep 输出 = "No matches found"`
**演进路径**（Doctor 2026-05-20 前瞻）:**未来打勾权可下放给 CC** · 但前提:
  - 每条交付标准本身是**客观可验证的明确标准**（满足"可验证 = 客观"原则）
  - 该标准下的证据**机器可比对**（不是 CC 解释自己的产物）
  - 该任务类型 CC 已通过 N 次实战（实战次数后定 · 暂不开放）

**G-X2 sandbox 绝对禁止跑任何 git 写命令**（★★★ 重 · 2026-05-20 强化 · 既有规则失效）
**既有规则**（2026-05-18 立）:"sandbox 写 .git/ 不可靠 + 加脚本探测自动跳过"。**本次再犯**:CC 在 `mcp__workspace__bash` 里直接跑 `git add -A && git commit`,绕过了"脚本探测"保护（因为不是通过脚本调用）。**新规则升级**:

  - **写操作的正确流程**:CC 把命令字符串贴给 Doctor → Doctor 在 macOS terminal 跑 → 反馈结果给 CC → CC 继续
  - **自我检查信号**:CC 看到自己即将打"git add" / "git commit" / "git push" 在 bash 调用里 → **立即停** → 改贴命令
  - **brain-save skill Step 5 必改**:从"自动 git commit" → "贴 commit + push 命令"（下次升级 brain-save skill 时落实）

**G-X2 补（2026-05-22）· 连只读 `git status` 也会在 FUSE 挂载盘留 index.lock**
本以为 `git status / log / diff` 只读无害；实测沙箱 FUSE 盘跑 `git status` 会创建 `.git/index.lock`（刷新索引所致），且 FUSE **不允许 rm**（Operation not permitted）→ 残锁阻塞 Doctor 下次 git 操作。**新规**：沙箱内**连只读 `git status` / `git log` 也尽量不跑**；探测 .git 存在性 / 工作区改动改用 `ls` / `Read` 工具；万不得已用 `git --no-optional-locks status`。残锁只能 Doctor 终端清：`rm -f .git/index.lock`。（→ 2026-05-22 PEC 推送后 CC 只读 status 给 brain + PEC 双库各留一个 index.lock）

**G-X5 新建 git repo 忘绑 remote + 国内 push 网络坑**（★★ · 2026-05-20 立）
**坑 1 · 忘绑 remote 的"假备份"**:新建本地 git repo（`git init`）后,`git add + commit` 成功会营造"已备份"假象,但若**从未 `git remote add`**,工作只在本地一台机器 · 无异地兜底 · 是 5/19"git 不是万能兜底"洞察的姊妹坑(那条是"commit 但没 push",这条是"压根没 remote 可 push")。**实例**:PEC repo 2026-05-17 建本地 + 一直没绑 remote,直到 5/20 push 才报"没有配置推送目标"。**防御**:项目立项 checklist 加一条"建 repo = init + remote add + 首推" 三步一体 · 不止步于 commit。每次给项目 commit 前 CC 应 `git remote -v`(只读)确认有 remote · 没有就提醒 Doctor 补建。
**坑 2 · 国内 push GitHub 的 SSL_ERROR_SYSCALL**:`fatal: ... LibreSSL SSL_connect: SSL_ERROR_SYSCALL in connection to github.com:443` = GitHub HTTPS 通道被干扰(GFW)· **不是 git 配置错**。**判断**:若同一会话内别的 repo(如 brain)HTTPS push 刚成功 = 网络基础通 = 多为偶发中断。**解法按序**:(1) 直接重试 2-3 次(最常成 · 2026-05-20 PEC即一次重试就过)· (2) 加大缓冲 `git config http.postBuffer 524288000` + 限速容忍 · (3) 改 SSH `git remote set-url origin git@github.com:用户/repo.git`(443 通道不稳时最可靠)· (4) 配代理 `git config --global http.proxy http://127.0.0.1:端口`。**数据安全提醒**:commit 已在本地 = 工作没丢 · push 失败只是延迟异地备份 · 不必慌。

**G-X6 brain skill 源 / .skill 包 / 已安装运行时 三层不自动同步**（★★ · 2026-05-22 立）

**G-X7 zip 在挂载/FUSE 目录直接打包产 0 字节坏包**（★★ · 2026-05-22 立 · 扩展「压缩先在 outputs」条）

**G-X8 CC「已落盘」假象——回复声称已写 ≠ 文件已写**（★★ · 2026-05-22 立）

**G-X9 多会话共用 git 工作区 · `git add -A` 卷走他会话未提交改动**（★★ · 2026-05-23 立）
多个 Claude 会话**共享同一 git 工作区**（同一挂载盘）。任一会话跑 `git add -A && commit`，会把**其他会话尚未提交的改动一并暂存提交**，导致 ① 改动挂到错误的 commit 信息下（「串味」）② 另一会话事后 `git add` 发现「无文件可提交 / 干净工作区」而误以为自己的改动丢了。**实例**：2026-05-23 brain frontmatter Option C 改 20 文件 + 校验脚本后，并发的 MiroFish 会话 `git add -A` 把它们卷进 commit `92bea7a`（「MiroFish本地部署落地」），Doctor 跑约定的提交命令时遂报「无文件要提交 / Everything up-to-date」。**判断信号**：自己刚改了 N 个文件，Doctor 提交却「nothing to commit」且 `git log` 顶部是别的会话的 commit。**防御**：① 并发场景给 Doctor 的命令用 `git add <具体路径>` 而非 `git add -A`；② 提交前用 `git --no-optional-locks log --oneline -5` + `git show HEAD:<file>` 确认改动是否已被他会话卷走，再决定是否还需提交；③ 确认改动没丢（已在某 commit + 已 push）则**不补提交、不改写历史**（已 push + 并发会话活跃，rebase 风险 > 收益）。（→ G-X2 sandbox git 禁忌姊妹条目 · Option C 警告清零被 MiroFish 会话卷走）

## 分析方法论 · 推演纪律

**反应式 vs 预判式：源生态锚定下的单向让步漂移（CC 自身失效模式）**
当 CC 在一个高度政治化/有倾斜源生态的议题上起手时，会**锚在源生态的重心**（如英语世界对中国议题：控方陈词前置、方法论保留意见后置），首答"看似平衡"实则压在一侧框架上。若此时遇到一个 **rigorous 且持续从另一侧施压**的对手，就会产生**序列式单向让步**——每轮"你说得对，让 X"——它**看起来、也部分确实是逻辑递弱**。

**机制四因（2026-05-30 PEC 新疆强迫劳动叙事九轮推演活体）**：
1. **α 源生态锚定**：起手锚在倾斜的源生态 → 方法论中枢（如"低基率+身份代理→假阳性悖论→比例失配"这条本该领跑的脊梁）拖到第十一轮才说清。**对手实则在替 CC 补其训练数据偏置。**
2. **G-03 顺向 drift**：长对话有默契，逐条让步；多数让得对，但因结构未预搭，每条让步成"打补丁"而非"框架本就该有的零件"。
3. **具体 rigor 错误**：把局部数据（疏附 12% 县级率）平推全族（×800万），重犯自己几轮前已认的"基率地理不代表"错。
4. **形式 performative 化**：α/α′ 计数、"签字版"仪式后期挤占核心逻辑注意力（篇幅没少、承重没跟上）。

**解药 = 预判式预让**：rigorous 版本应在**第一轮就预先让掉可知的强对侧点**（穷/离土必然/政治为本/数字出处薄/伪二元），把争议核**一开始就压窄**到稳定核（基率比例性 + 范畴纪律 + 文明基因）。**"花了十一轮才到争议核" 本身 = 初始框架太偏的证据。**

**判别器**：发现自己连续 ≥3 轮在"认对方的点"且每次都像在纠错（而非"对方的点嵌进我已搭好的框架"）→ 触发自检：我是否一开始就该有这个结构？可知的强对侧点我是否预让了？

**升级链推演的 firebreak / off-ramp 纪律（军事-核升级专属 · 2026-06-02 PEC 阻止日本拥核 GE-05 五轮推演活体）**

推演军事/核升级链时，CC 有一个**双向**失效模式——同一病灶（对 firebreak 与 off-ramp 的系统性误判），表现成两个相反的错：

- **过度悲观 A 型 · 无视真实 firebreak**：把"升级会被某机制截断"的真实台阶当不存在。活体①「MAD 绝对主义」——把"与核国家开战＝自杀"绝对化，忽略稳定-不稳定悖论（涉核国家有限常规战是常态、GE-03 中国对核垄断美国打常规且美方克制不用核）。
- **过度悲观 B 型 · 假设不存在的自动升级（逃逸确定论）**：把升级当**自动连锁**，忽略 off-ramp。活体②「明火→中美核战」——把"中国常规打击伞下日本→中美核战"当自动，忽略条约后门（美日安保第五条"依各自宪法程序"）/ 延伸威慑可信度折扣（不拿旧金山换东京）/ 常规-核界（首击常规则对手有不升级台阶）。**且自相矛盾**于 CC 自己上一轮援引的稳定-不稳定悖论。

**第三个同族活体**：③「守弱框架移植」（见下条）。三者都是升级推演里**对"链条在哪截断"判错**。

**解药 · 推演前先列 firebreak/off-ramp 清单**：① 门槛 firebreak（门槛前无核报复 / 门槛后有）② 常规-核 firebreak（首击常规保 NFU 标签、给对手台阶）③ 同盟 off-ramp（条约后门 + 延伸威慑折扣）④ **反向警觉**：弹头模糊是**拆** firebreak 的（无意升级引擎），与上面相反，是升级链最危险一环——别把它当"可控的升级工具"。

**折旧/损耗资产默认 use-driven → 滑回"克制保值"（CC 自身失效模式 · 消耗型咽喉杠杆专属 · 会复发）**

碰"折旧/损耗型杠杆"（C-19 消耗型咽喉资产：稀土/化工/产能型卡点）时，CC 有一个**顽固默认**：把损耗读成**由"使用"驱动** → 推出"**克制以保值**"。但这类资产的折旧实为**时钟驱动**——由"卡点共识跨过（已发生）+ 对手重建投资力度"决定，**largely 与是否使用脱钩**。于是"克制保值"是**伪选项**（克制不保值、不用以后也用不了），正解是**窗口内按段-目标-联盟催化优化使用**。

**复发证据（这是关键——它不是一次性错）**：
1. C-19 **立案时**已把"使用驱动折旧/克制保值"明记为**CC 原偏误**并纠正。
2. 2026-06-10 H-15 调和律 **v2 又滑回**——把"持有者更受约束"读成"用即自损 → 应克制"，Doctor 二次抓出（"卡稀土对中国的损害在哪"+"目前禁运效用最大、代价完全可接受"）→ 改 v3。**同一偏误半年内第二次复发 = 这是个稳定吸引子，不是手误。**

**解药 · 分析前一问**：这个杠杆的损耗，是因为**我用它**，还是因为**时钟在走（共识+对手投资）**？
- 若 **clock-driven** → "克制保值"无收益；持有者的"约束"是**时间性/易腐性**（限期资产、须窗口内用），**不是"自损式"**（用即亏）。两种"约束"别混。
- 使用的**真实边际代价** = 只算 use-driven 部分（联盟催化油门的加速）+ 真实自损；**use-independent 的沉没折旧不算"用的代价"**。

**判别器**：发现自己在推"该克制 / 保留杠杆 / 留着以后用" → 立即问：损耗是 use-driven 还是 clock-driven？若后者，"留着以后用"可能根本没有"以后"。

**框架移植误用：语境特定的姿态默认冒充跨场景硬约束（2026-06-02 PEC GE-05「守弱」活体 · G-01 近亲）**

一个在**特定语境**立的姿态默认（如「守弱」＝台湾/蓄力/模糊局面下的"等"默认），被 CC 跨语境硬搬进**另一性质的场景**（核越线＝存续级、已结晶、大义已满），冒充**硬约束**去压低选项。活体：CC 用"守弱、自拆台"压低对日动能选项，Doctor 一句「日本拥核还守弱?」点破——守弱在它自己定义下此刻就该被脱掉（最大触发＝大义过闸），真正的刹车不是姿态偏好（守弱），是硬约束（MAD 物理 / 美国延伸威慑 / 放射性自伤）。**把"姿态偏好"错记成"硬约束"会同时高估约束力、模糊真正的约束在哪。**

**判别器**：援用某个"默认/惯性/姿态"去排除一个选项时，问：这个默认是**当前场景**立的，还是我从**别的语境**搬来的？它是**硬约束**（物理/结构/不可让渡）还是**软偏好**（姿态/风格/可让渡）？软偏好遇到场景的触发条件（casus belli / 存续威胁）就该被脱掉，不能拿它当门闩。（→ PEC GE-05 后发选项与开火点前移.md §六 · 与上条「逃逸确定论」同族）

**没有免检的默认值：先验/信道/选词都要核实（2026-06-08 · PEC「认识论闸」八轮碰撞活体 · 跨项目元方法）**

论证里最隐蔽的偏置**不在结论层，在默认值层**——先验（基率/参照类）、信道假设（"会不会留痕"）、甚至**选词**，都可能偷渡一个未核实的倾向，且因为它们看着像"中立常识"而逃过审查。**铁律：没有免检的默认值，两端对称举证。**

**同一会话八轮拆出的活体（都把消解性倾向藏进中性外壳）**：

1. **G-08 反向绝对化**：把"结构性涌现"当免举证默认去驳回任何意图主张＝G-08 自变 G-01/G-09/G-27 镜像。正解＝受益→意图**有信道支撑时是健全推断**，只挡**无证据裸跳**（"无证据"限定词承重，省了就滑回绝对化）。
2. **缺证 ≠ 证伪**（诉诸无知双向）：未证真≠假、未证假≠真；三值真/假/未决；举证责任在主张方，未尽举证＝"未立"非"被驳"。
3. **基率"低"未问参照类**：基率取决于参照类、切法本身是判断；报"低基率"前先问哪个参照类、是否已替结论选好（贬抑/抬升先验偷渡）。
4. **"若真应留痕"＝偷换概念 + 幸存者偏差**：用"留痕"定义"真"（真≠有存证）；档案是幸存样本（成功且深埋的隐蔽动机系统性不可见）；似然 P(E∣H) 本身不可知 → 隐蔽意图域档案信道**退化**，禁以留痕有无做任何方向更新。
5. **选词替你下判断**："基率低/若真应留痕/靶子尚未成形（vs 刚刚成型）"——三次把推论方向藏进描述词。

**两个顶住默认引力的工具**：① **措辞翻转测试**（带推论后果的词换中性/反向重读，结论跟着变＝词在替你判断）；② **显式限定词写死**（如"受益→意图的【无证据】跳跃"，靠自觉守不住）。
