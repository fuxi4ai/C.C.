## 完成的工作

- **星空数据刷新**：`canonical_to_starscape.py -i <canonical/enrich候选> -d 1 --iters 60 --radius 400 > examples/yuantu-data-v3-full.json`，从 2145/3075 刷到 **2466/3508**（-d1 滤度0孤点，源自 canonical 2950/3512 或 enrich 候选 2952/3515）；7 行业桶分布正常。含炬光改名 company_Focuslight + 光场匀化器/DUV光刻卡位节点。
- **黑屏 bug ①（双击文件黑）**：主星 `starMat` 顶点着色器（含 searchHalo、signalLayer 三处）用了 `gl_VertexID`——**WebGL2/GLSL ES 3.00 才有的内置变量**，Three.js r128 默认走 WebGL1 → shader 编译失败 → 主星几何体整个不渲染 → 全黑。改成 `fract(sin(dot(position.xy, vec2(...))) * 43758.5453)`（position 派生随机，WebGL1 兼容、相位稳定、视觉等价）。修 `prototypes/starscape-template.html` 三处。
- **黑屏 bug ②（artifact 面板黑）**：cowork artifact 面板 CSP 只放行 Chart.js/Grid.js/Mermaid，**外部 Three.js CDN（cloudflare r128 + jsdelivr examples/js）被挡** → `THREE` 未定义 → 面板黑（与 gl_VertexID 无关，是独立第二 bug）。修法：`npm install three@0.128.0` → 把 8 个 `<script src>`（core + OrbitControls + EffectComposer/RenderPass/ShaderPass/CopyShader/LuminosityHighPassShader/UnrealBloomPass）全部**内联**进 HTML，产出 `-inlined.html`（2.44MB、0 外部脚本、完全自包含），update_artifact 推送。
- **产出**：CDN 版 `yuantu-starry-skies-v3-full.html`（双击用）+ 内联版 `-inlined.html`（喂 artifact）；outputs 各一份；GOTCHAS +1（ERR-20260705-001 gl_VertexID）。

## 做出的决策

| 决策 | 原因 | 影响 |
|------|------|------|
| gl_VertexID → position 派生随机 | WebGL2-only，r128 WebGL1 编译失败致全黑 | 模板永久修复，双击文件恢复渲染 |
| artifact 用内联版、双击用 CDN 版 | 面板 CSP 挡外部 Three.js，内联绕过；文件在真浏览器 CDN 无碍 | 星空维护两份 HTML（CDN/内联） |
| 星图基于 enrich 候选(含炬光) | Doctor 要预览炬光改动进星空 | promote 后正式版一致 |
