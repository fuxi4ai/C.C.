# 会话日志 — 2026-05-23

## 完成的工作

- 给出 MiroFish 质量评估：工程层面中上、整合能力强（建在 OASIS + Zep 上，自动化产品层是真增量）；但「预测万物」未经验证、热度/实质比偏高，定位为情景生成器而非已验证预测器（咬合 PEC G-01/G-04/G-10）。
- Doctor 已注册 Zep API + 已有 DeepSeek 密钥；选定：DeepSeek 作 LLM + 源码部署。
- 生成 `.env` 配置参考：`~/Documents/MiroFish-env配置参考.txt`（DeepSeek base_url=https://api.deepseek.com/v1 / model=deepseek-chat + Zep + OASIS_DEFAULT_MAX_ROUNDS=10，刻意不含 LLM_BOOST_*）。
- 部署位置定为 `~/Documents/projects/MiroFish`（Doctor 指定）。
- 给出 Mac 终端 runbook：环境检查 → clone + heredoc 写 .env → 填密钥 → `npm run setup:all` → `npm run dev`。
- 排障：`uv` 未装 → 官方脚本 `curl -LsSf https://astral.sh/uv/install.sh | sh` 装好。
- 排障（关键）：系统 Python 3.13.3 太新（项目要 3.11–3.12）→ 用 uv 管版本绕过：`uv python install 3.12` + `cd backend && uv python pin 3.12`，再 `npm run setup:all`。
- 结果：前三步完成，`npm run dev` 跑通，项目网页 localhost:3000 已打开（尚未跑首个预测）。

## 做出的决策

| 决策 | 原因 | 影响 |
|------|------|------|
| LLM=DeepSeek + 源码部署到 ~/Documents/projects/MiroFish | Doctor 已有 key、源码便于调 N/R 省钱 | .env/runbook 全按此配 |
| 不在 sandbox 替 Doctor clone/跑服务 | 服务需本地起 3000/5001 + 浏览器 + 密钥留本地；遵 G-X2 不在 sandbox 跑 git 写 | 改为给终端 runbook |
| 密钥用占位符、本地填，不进对话 | 安全：API secret 不留聊天记录 | .env 用 `<...>` 占位 |
| Python 3.13 用 uv pin 3.12 绕过，不动系统 Python | camel-oasis/camel-ai 在 3.13 大概率装不上 | backend/.python-version=3.12 |
