## 完成的工作

### 一、UI 元素 7 处全面升级（一次落）
- **Logo 真发光星**：单星 → 多层叠加 + 呼吸动画（后改为三星轨道系统）
- **KPI bar**：列间 1px 竖线分隔 + 数字下 0.5px 发丝下划线 + hover title 提示
- **类型过滤**：色点 hover 1.4s 脉动 + 活动行 inset 0.5px 内描边
- **全景看点卡（最大改动）**：默认态从 1 句话扩展为四块——行业分布迷你 bar / Top 3 hub 可点小卡 / 5 超图 chips / 4-pill 轮播指示
- **焦点卡精炼**：上下游链路从纯文字 → 小玻璃卡 hover 加深；chip 加玻璃高光层；退出 X 图标 hover 旋转 90°
- **Tooltip 玻璃化**：从方形黑底 → 4 层玻璃语言（顶光 radial + inset 高光 + 微外阴影）
- **操作提示 kbd chips**：5 个按键玻璃小键（最终 Doctor 决定整段删掉）

### 二、焦点卡内容升级"本体 + 关系"
- 加 **本体 type chip**：标题下方一颗彩色 chip（蓝青 / 嫩绿 / 琥珀 / 品紫 / 月白），显示行业 + 原 12-类型
- 上下游每条目末尾加 **22 种 relation 的中文 pill**：驱动 / 依赖 / 支持 / 使能 / 演进至 / 供应 / 采购自 / 制造 / 用于 / 是组件 / 设计服务 / 竞争 / 替代 / 约束 / 受益于 / 相似 / 相关 / 需要 / 监管 / 产出
- focusOnIdx 函数的 `incoming/outgoing` 从 `[idx]` 改为 `[{idx, rel}]` 携带关系类型

### 三、焦点卡升级为"透明玻璃板"
- right: 32 → 44px（离右边框）
- padding: 16/20 → 20/22（内呼吸）
- border-radius: 10 → 14
- 4 层玻璃叠加（顶光 radial + inset 高光 + 0.5px 内描边）
- **外阴影 0 4px 24px rgba(0,0,0,.45)** 让卡片从星空中浮起

### 四、视觉重心配平
- info 卡占 ~340px vs filter 占 ~230px，差出 110px → 整体右倾
- 相机 + OrbitControls target 一起向右偏 6 世界单位（屏幕上 stars 整体左移 ~75px）
- autoRotate 仍以新 target 为中心旋转，交互不破坏
- 底部胶囊 `left: 50%` → `calc(50% - 35px)` 同步左移

### 五、伪三体 logo
- 中心引力星 + 3 颗轨道星（蓝青 / 嫩绿 / 琥珀，半径 10/7/5，周期 5.5s/7.2s/9.3s 不同方向）
- Doctor 决定隐藏中心 + 3 道彩色轨道环显形 → 3 颗小星环绕空中心
- 不真混沌，规则圆轨道+不同周期+错开起始角制造"近似不严谨三体"视感

### 六、kami 字体借入 · 三层字体系统
- 借 kami 的 `TsangerJinKai02 仓耳今楷` 作为 serif 主字体（不商业分享 → fallback 链精简为 `"TsangerJinKai02", serif`）
- jsdelivr CDN 拉 `JetBrains Mono` 作 mono
- 三层用法：
  - **Serif** → logo h1 / 焦点节点名 h2 / Top hub 名 / 上下游链路名 / 全景大数字 / 分类组名 / 按钮 / 搜索框（最终 Doctor 让按钮 + 搜索框也用 serif）
  - **Mono** → KPI 4 数 / stats 4 格 / 行业 count / hub magnitude
  - **Sans** → 副标 small caps / breadcrumb / 关系 pill

### 七、卡片定位最终调整
- 左侧类型卡 → 纵向居中 `top: 50%; transform: translateY(-50%)`
- 右侧知识卡 → top 120 → 148px（"一点点点点"的 28px 下移）

### 八、收尾
- v1.1 双 frozen 快照（brain + 渊图）
- GOTCHAS 加 G-16 ~ G-20 五条新沉淀
- brain stub `星空.md` 加 v1.1 milestone

## 做出的决策

| 决策 | 原因 | 影响 |
|------|------|------|
| 一次性升级 7 处 UI（不分批） | "买椅子配地毯" 是整体调性问题，分批做容易留下不协调感 | 一次 13 个补丁全部到位 |
| 操作提示整段删除 | Doctor 觉得 5 个 kbd chips 在底部还是显得"教程化"——星空美感优先于功能提示 | 画面更干净 |
| 视觉重心用相机偏移而非 CSS 移动 canvas | 保留 OrbitControls 的旋转中心一致性；CSS transform 会让黑边露馅 | 6 世界单位偏移 = ~75px 屏幕位移 |
| 焦点卡上下游加关系类型 pill | Doctor 反馈"看不出关系是什么"——节点列表没传达边的语义 | 22 种 relation 全中文化；图谱"故事性"出来 |
| 隐藏中心星，只留三体 + 轨道环 | Doctor 觉得中心星太"权威"，纯轨道更动态 | logo 变成纯运动的伪三体 |
| serif fallback 链精简 | Doctor 不商业分享 | 从 7 个 fallback 砍到 1 个 + `serif` 通用回退 |
| 按钮 + 搜索框也用 serif | Doctor 觉得 sans 跟其他楷体不协调 | 全 UI 统一到 kami 楷体气质 |
| Filter 卡纵向居中 + Info 卡微下移 | Doctor 视觉直觉：左右两卡基线不对称 | 配平 |
