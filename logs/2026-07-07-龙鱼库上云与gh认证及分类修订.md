**E. 龙鱼库首次上云 · gh 认证踩坑链**
- push 连环失败逐层剥出真因：① `.git/index.lock` 陈旧锁（清后可 commit）→ ② `master` 无 upstream（首推）→ ③ SSH `Permission denied (publickey)`：ssh-agent 空 + 钥匙串无 identity → ④ HTTPS `Repository not found`：**远端仓 `fuxi4ai/longyu-analysis-lib` 压根没建**。
- 澄清「github 每天推」＝推的是别的仓（brain 等，走钥匙串 osxkeychain 存的 github.com PAT）；此 longyu 库 SSH 克隆且从未推过，成特例。
- 解法：装 gh CLI（`brew install gh` + `gh auth login` HTTPS）→ gh 接管 git 凭证助手 → `gh repo create` 建库 → `git push` 上云。Doctor 「都搞定了」。
