## 完成的工作

- **preflight 报告成形**（VV 首轮前）：`--dry-run` 输出 VV 要求的复核项（DRY_RUN/local_delete=false、session/作者/文件数字节、source_root、远端 incoming/committed、manifest schema、前3后3、dup/collision/missing、F: 余量前后）。
- **VV 第一轮（7 项）→ 两阶段提交**：执行门（preflight 异常清零 + F:余量≥150GiB fail-closed）；ps1 根前缀漏洞修（`==root` 或 `root+sep`）+ reparse 链检查 + proof create-new + Status 全量 scrub；**逐条 Commit → FinalizeSession+StatusSession 全量核验 → 才批量写本地 pointer/DB**；`sh` 非零硬退出；session-id 不可复用时间戳。
- **VV 第二轮（8 项）**：missing_source 真门（不再先 skip 缺失，校 symlink/普通文件/realpath 在根/字节==manifest）；session-id 含时分秒；Status 校 verification.json；**StatusSession 改读 committed manifest.json（SHA 绑定 SESSION.COMMITTED）脱离可变 _plan**；proof/manifest create-or-verify；远端 `Assert-PlanValid`；Phase 3 pointer create-or-verify + DB 单事务 + receipt + 分阶段报告。
- **VV 第三轮（6 项）**：`--resume` 接入（原 plan 不重生成 / 远端 _plan 只校哈希不覆盖 / Phase1-Status 跳过已提交 / finalized 补齐 Phase3）；Prepare 拒脏目录+已认领（无 claim≠干净）；reparse 覆盖全 proof（`Read-TextChecked`）；Phase3 只读预检——任一 pointer 冲突则 pointer+DB 一律不写；receipt create-or-verify 绑定 committed manifest SHA-256；DB autocommit 下迁移与事务分离、回读核缺行。
- **VV 第四轮（5 项）**：本地 plan create-new（拒重名 session）；`_plan.json` 上传可恢复（scp 临时名 → 远端 InstallPlan 校验后 create-or-verify 原子发布，永不直覆）；pointer/receipt no-replace 发布（`O_EXCL`+fsync+`os.link`，竞态复读校验不覆盖）；receipt 拆 `attempts/`（append-only）+ 最终 receipt（仅全成功、只记稳定事实）；Prepare 枚举前 no-reparse + 所有 action 先 `Assert-Claim` + 哈希 manifest 前拒 reparse；DB 回读加 `offsite_uri` 核对。
- **VV 第五轮（5 项）**：同上 no-replace/InstallPlan/receipt 拆分/Claim 门定稿并沙箱验证（首个 written/再次 ok/不同 conflict·内容不覆盖）。
- **VV 第六轮 + 终审（5+3 项）**：session-id 严格 `^[A-Za-z0-9][A-Za-z0-9_-]{0,95}$` 两端（防 `..` 逃逸，Python 路径构造前先校验）；rel_path 拒控制命名空间（`.incoming`/`.committed`/`_tools`/点开头）/Windows 保留设备名/段尾空格句点/内部反斜杠/`.` 段——两端；load_manifest 严格（坏行报行号退出、报非空行 vs 有效条目、不静默丢行）；manifest_sha 硬门（Finalize/Status 双返回须均 64hex 且相等才进 Phase3）；**发布最终 receipt 前 `final_scrub` 逐条回验 pointer+DB 与 committed plan 一致**；**dest_win 必须==GetFullPath(Root+rel_path) OrdinalIgnoreCase**（不信任两套路径语义）；size 必填非负整数（拒 null/浮点/布尔）；`--list-batches` 批完整性（总数/四批和/aid 并集/交集空/总字节一致）；收紧：Prepare 要求根已存在（不建根）、link 后 fsync 父目录、attempt 拆 start+result。
- **每轮沙箱验证**：py_compile + AST + ps1 纯 ASCII（`grep -P '[^\x00-\x7f]'`=0）+ 括号引号配平；执行门/碰撞/容量/resume 门/DB 事务/no-replace/plan-dup/manifest 坏行 逐一实测退出码。
- 每轮改完 push 到 `github.com/fuxi4ai/DVA`（Doctor Mac 侧跑 git）。

## 做出的决策

| 决策 | 原因 | 影响 |
|------|------|------|
| 采纳 VV 全部六轮意见、不反驳 | 66GB 不可逆搬迁，护栏越细越稳；VV 抠的都是真实注入/逃逸/静默丢数据/半写不可恢复面 | 脚本从「能跑」到「抗损坏 plan / 抗中断 / 抗竞态 / 可恢复」 |
| 两阶段提交（远端定案后才写本地真相） | 定案前写 pointer/DB 会在 session 未全 committed 时留半批本地状态 | pointer/DB 移到 Phase 3，前置 manifest_sha 硬门 + 最终 final_scrub |
| committed manifest 为 session 真相源（非 _plan） | `_plan.json` 可变、可被替换 | StatusSession 读 committed manifest + SHA 绑定 SESSION.COMMITTED |
| no-replace 原子发布（O_EXCL+link） | `Path.replace()` 会覆盖竞态目标，违背「不覆盖不同内容」 | pointer/receipt/plan 全走 create-or-verify no-replace |
| session-id/rel_path 严格白名单 + dest 绑定 | `..`/控制命名空间/两套路径语义 = 注入面 | 恶意/损坏 plan 无法写进 `.committed`/`_tools`/manifest |
| smoke 用真实最小视频（待 Doctor 定） | 真 aid 才有 DB 行、能走全绿；合成假文件 DB 缺行不绿 | 已问 Doctor 二选一（真文件=第一件真外移 / 合成=纯抛弃） |
