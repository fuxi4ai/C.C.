### B · 渊图技术栈重大调整：默认引擎切换 V4 Pro + KG_* env 解耦

- 实战触发：apiyi sonnet 通道当晚 4/5 Connection error 失败、零 patch 沉淀；切到 DeepSeek 官方 V4 Pro 直连后 13/17 批量 ingest 成功，schema follow 远好于 schema v3 那次
- 决策：所有渊图脚本默认引擎 claude-sonnet-4-6 → **deepseek-v4-pro**；默认 base_url apiyi → **deepseek.com/anthropic**
- **env 解耦**：发现 Doctor 用 CC-Switch 通过 `~/.claude/settings.json` 管 ANTHROPIC_* env，shell export 会覆盖之 → 渊图脚本改用独立的 `KG_API_KEY` / `KG_BASE_URL` / `KG_MODEL`（fallback ANTHROPIC_* 向后兼容）
- 改动：3 个脚本（kg_ingest/reclass_edges/tag_market_signals）+ .zshrc 配置块（4 个 KG_* env + 3 个 shell 函数 kg-use-deepseek/kg-use-apiyi/kg-engine）
- 安全：Doctor 在终端贴 .zshrc 时旧 apiyi key 暴露在对话历史，提醒 rotate

## 做出的决策

| 决策 | 原因 | 影响 |
|------|------|------|
| 默认引擎 V4 Pro + KG_* env 解耦 | apiyi sonnet 通道实战不稳；CC-Switch 警告 shell export 会覆盖 settings.json | 3 脚本默认值改 + .zshrc 加 KG_* 配置 + 不动 ANTHROPIC_*/CC-Switch；副线随时可切 |
| 金刚石散热深度专题完整入库（13 节点） | 图谱真实空白（金刚石只有 3 个钻针节点全是机加工）+ 韬定律 3D Folding 散热瓶颈直接对位 | 与韬定律 used_in hedged 连接（weight 0.65，不过度声明华为采用） |
| "技术合理 ≠ 已被采用" hedged 原则 | 投资叙事过度演绎需甄别（华为官方未明确采用金刚石） | _meta 分层标注 P1 真技术 / 未证实叙事 |
| Tier A 三家全入 + 沃尔德加品格典范 _meta | 沃尔德主动 caveat 信披行为是甄别范式正面对偶第一案例 | 双向甄别公式闭环 |
| Tier B（力量钻石/四方达）暂不入 | "NVIDIA 实验室测试通过/45万片2.8亿订单/12英寸数字"等公司方密集利好需 P1 验证 | 等帕米尔产业纪要/招股书 cross-check 后升 Tier |
| 国机精工/中兵红箭"供华为"边不直接建 | 公司方说法默认 P2，需 P1 cross-check 才正式 supplies HiSilicon | properties + description 转述，不污染边语义 |
| 帕米尔 batch 中医药/消费 3 篇仍入 | Doctor 主动判定 AI 制药=AI 价值链下游应用兑现；泡泡玛特=公司品格候选 | 渊图主题边界从"AI 上游算力"扩到"完整 AI 价值链 + 公司品格甄别" |
| V4 schema follow 偏差用手工 patch 修补 | LLM 偶发中文 id 重复节点 + 自环边（漏 source 前缀），kg_rel_classify 修不了 | <5 行 Python 即修；沉淀为 batch 后处理 SOP |
