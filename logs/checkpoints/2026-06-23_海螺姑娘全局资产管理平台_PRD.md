# PRD · 海螺姑娘全局资产管理平台（升格大活）

## §一 · 任务目标

把「全局资产管理」**内化为海螺姑娘的能力**：现有 `conch_engine.py`（dry-run 默认 + 多维 `is_obsolete` + 审计/rollback + 项目类型自适应）从「单项目维护」**升格**出一个**全局盘点模式**，遍历本地所有项目级资产（不含 VV）产出一份资产 **manifest**；新建一个**看板 artifact** 作为海螺姑娘的 view 层——把资产按 **Resource / Database / Project（+ Infrastructure 子组）** 铺成「上下游关系星河」，平时以**五态发光**显健康、点击弹详情卡；对**待修 / 断链**资产提供「审查并修复」按钮，触发 conch 对该资产做**专项 dry-run pass** 出分级方案，**Doctor 逐项批准才 `--apply`**（conch 自带 rollback）。目的是全局资产可视化 + 后续对出问题的资产专项整理/修复。

**Doctor 原始指令**（逐字引用）：
> "做一个AI powered Artifact；目的是盘点我们本地（不含VV）所有项目级别资产；分成"Resource"，"Database"，"Project"；按"上下游"顺序排列并展示互相调用/消费的关系线；点击实现弹出详情卡；不点击的时候，以发光的不同颜色，或类似加某种符号的形式直观展现状态；对需要修复的资产有"审查并修复"按钮；海螺姑娘这一类的project单成一组，命名为"Infrastructure"。"
> "能不能把这个看板内化成海螺姑娘的一部分——目的是全局资产管理，后续可以对出问题的资产专项整理/修复"
> "升格大活"

**对齐结论摘要**（2026-06-23 三轮对齐确认）：
- 数据来源 = **策展 manifest（单一真源）+ 定时探活回写**；上下游关系线**手工策展**（conch 不臆造 edges）。
- 「审查并修复」= **审查→分级方案，不自动改**；引擎复用 conch（dry-run 默认、`--apply` 必经 Doctor、可 rollback）。
- 载体 = **新建独立 artifact**，复用现有 `brain-vault-dashboard` 的星河/详情卡/新鲜度渲染代码；**不改**现有金融看板。
- 三类定义：Resource=外部信源/原料；Database=本地数据库/存储；Project=代码项目。**Infrastructure = Project 子组**，成员 = {海螺姑娘, 司南}。
- 状态 = **五态**：健康(绿)/陈旧(黄·按表级新鲜度非纯 mtime)/断链(红)/待修(橙)/孤儿(灰)；审修按钮**仅在 待修/断链 亮**。
- 海螺姑娘**升格**为全局资产管理平台（mission 重定义 + 真新代码）。
- 数灵转移在 manifest 中标 **archived**（已退役为文档记录）。
- 实时性现实形态：conch 全局盘点（定时任务/手动）回写 manifest → 看板读快照 + Reload；**非页面级秒级实时**（artifact 沙箱不能直接读本地 .db）。

**任务规模估算**：
- 预计涉及文件：~6–9（`conch_engine.py` 加全局盘点命令；新 `data/asset_manifest.json` schema；新看板 artifact HTML；海螺姑娘 `系统概览.md`/`决策记录.md`；可能 `.conchconfig.yml` / 探活定时任务 SKILL）
- 预计耗时：跨小时（多组件）
- 涉及项目：海螺姑娘（主·写）+ 被盘点的 14 项目（只读）

## §三 · 非交付项（本任务不做）

1. **不做一键自动修复**——conch 只 dry-run 出分级方案，`--apply` 一律待 Doctor 批准。
2. **不做全自动上下游关系检测**——`edges`（调用/消费/反哺）手工策展进 manifest；conch 不产关系。
3. **不保证页面级秒级实时**——走 conch 全局盘点（定时/手动）回写 manifest + 看板读快照 + Reload。真·页面级实时待有可读本地资产的 MCP 另议。
4. **不含 VV 任何资产**。
5. **不改现有 `brain-vault-dashboard`**（新建独立 artifact）。
6. MVP 不含：历史趋势/多期对比、跨设备同步、访问权限、关系线自动校验。
