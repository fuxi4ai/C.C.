# PRD · AI硬件机柜逐层可视化

## §一 · 任务目标

**动机**：为 AI 算力硬件做一套面向**对外展示/传播**的"逐层拆解"可视化——从机柜（rack-scale 超节点）一路下钻到芯片 Die，覆盖 NVIDIA / Google / 华为三家、可横向对比。目标受众是外部观众，故以**直觉冲击力与叙事节奏**为第一位，同时数据须准确并**分级标注来源与置信度**。

**范围（本次 = MVP）**：先做 **NVIDIA GB200 NVL72 单家纵切到底**（机柜 → 1U Compute Tray → 板/GB200 Superchip 板卡 → Superchip → Blackwell GPU Die），同时把"三家统一数据 schema"立好、给 Google TPU Pod 与华为 CloudMatrix 384 留占位。技术路线为**数据驱动叙事**：数据层负责结构与标注，**视觉表层用真实图像/写实渲染**，不用抽象形状近似。

**Doctor 原始指令**（逐字引用）：
> "CC，我如果想做AI硬件的可视化，要求能做到NVdia，Google，和华为韬定律三个系列的，从"机柜"开始，从大到小的产品结构拆解和信息呈现。我们不从头开发，有什么能借鉴的项目么？"

> "对齐一下，视觉上，现在我们的方法呈现效果是什么？要求是：可以没有3D模型，但起码要有机柜/板/芯/连接等照片渲染，不接受形状近似模拟。"

**对齐结论摘要**（六轮 AskUserQuestion + 收口确认）：
- 技术路线：**数据驱动叙事**（数据作结构+标注层，非画硬件）。
- 华为系列锚定：**CloudMatrix 384 超节点**（昇腾 910C，对标 NVL72；"韬定律"为 Doctor 既有华为 AI 链框架名）。
- 用途受众：**对外展示/传播**（重叙事美感+直觉，数据准但标注来源/置信度）。
- MVP 边界：**NVIDIA 单家纵切到底** + 统一 schema 给另两家占位。
- 视觉硬约束（关键新增）：**可以没有实时 3D 模型，但每一层必须是真实图像/写实渲染（机柜/板/芯/连接），不接受抽象形状近似模拟**。真正的轴是"忠实图像 vs 抽象方块"，不是"3D vs 非 3D"。
- 图像来源策略：**混合**——能拿到真图的层用真图（官方渲染/实拍/可授权 Die 图）为主体；拿不到的层用 **AI 写实渲染并打"示意"水印**；全部按"官方实拍 / 官方渲染 / AI 示意"三级标注置信度。
- 下钻引擎：先做**两版最小 demo 对比手感再定**（候选：OpenSeadragon 图像深缩放 vs Apple 式分层交叉淡入），不预先锁定。
- 可借鉴底座：OpenSeadragon（图像深缩放）、Scrollama/GSAP（滚动叙事）、SVG/HTML 热点浮层（标注层）、NetBox 的层级数据模型形状（rack→device→module→part）作 schema 参考、Bartosz Ciechanowski 长文作叙事工艺标杆。

**任务规模估算**：
- 预计涉及文件数：~8-12（index.html、schema、nvidia 数据、assets 图像若干、2 版引擎 demo、选型结论 progress）
- 预计耗时：数小时（大头在 NVIDIA 各层真实图像搜集/授权核查 + 两版 demo 实测）
- 涉及项目：AI硬件可视化（新建，单项目）
