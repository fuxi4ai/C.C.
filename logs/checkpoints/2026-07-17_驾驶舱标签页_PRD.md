## §附 · 三时钟判定口径(冻结 · 逐条注来源 · 分级不升格)

> 数据源:recap.db(**只读**,URI `mode=ro`)的 yuantu_buy_signals + escalation_shadow;market_data.db(只读);overheat_current_holdings.csv(A1 输出,只消费不重算);holdings.json。
> 分级铁律沿用交接消化报告 §二:A=可消费输出;B=可提示不可触发动作;C=禁用。**任何下游引用不得跨级升格。**

**信息时钟**(per case,枚举:新信息/边际升级/持续失衡/反转/无信号)
- 新信息 = signal_node 当日首见(yuantu_buy_signals 新行)。来源:信息差回测——新发信号 10d 中位 +3.4% vs persistent −2.4%。
- 边际升级 = escalation_shadow.escalated=1(四类量值新高:缺口%/涨价档/交期/锁定年限)。**07-23 影子复核放开前,前端一律标注「影子·未放开」。**来源:A3 PRD 2026-07-16。
- 持续失衡 = 同 node 重复且无 escalated。来源:persistent 无信息差双口径确认(兑现 10d 中位 −7.55 n=25 × 信号级 −2.4% n=30)。
- 反转 = direction_flip_date 当日 / 方向字段翻转。
- 显示语义:持续失衡 = "不能支持开仓/加仓,也不能单独触发离场"(Doctor 终局文档统一解释)。

**价格时钟**(per case×标的,枚举:未定价/兑现/过热/松动/中性)
- 未定价 = 信号活跃且 date_realized IS NULL。
- 兑现 = date_realized 已至(excess_cum/excess_peak 记账)。来源:兑现确认后 3d 胜率 84.5%(n=181)。
- 过热 = **仅持仓**,消费 overheat_current_holdings.csv 的 overheat_state ∈ {黄,橙}(A1:黄=停止追价+复核,橙=压低新增仓位上限,**非离场信号**)。观察股不在 A1 覆盖内 → **不给过热硬态**,只给 B3 个性化温度徽章(20日涨幅/自身Q90,候选级·仅提示)。
- 松动 = relit_date 存在且 ≤10 交易日(relit 二段点亮 10d 期望 48%/−1.19,已移出日报强信号栏)。
- 中性 = 其余。
- C 级禁用照抄:z<1 解除、"持续 8 日"、红灯自动减仓,一概不进代码。

**波动/执行时钟**(per 标的,枚举:压缩/常态/扩张/压力/冷却 + 执行旗)
- 压缩 = vol5 < 自身 244 日 P20 连续 5 日(E2 冻结口径,N=122:20 日 |Δ| 中位 5–14%,方向≈掷硬币)。显示语义:"检查仓位,不预测方向"。
- 压力 = vol20 ≥ 自身历史 P90;扩张 = P75–P90;常态 = <P75。⚠ P75/P90 切点为**工程默认档位,非实证冻结值**,Doctor 审 PRD 时可改。
- 冷却 = 曾处压力档、现回落 <P75 且在 20 日内(B4:持仓篮冷却中位 11–12 日)。
- 执行旗 = 当日涨停/跌停(limit_list_daily)/停牌/港股无本地行情;T+1 恒提示。
- 收益口径统一 A2:pre_close+pct_chg 连续财富,收盘确认信号、次日执行,无前视。

**行动权限编译**(联合状态→七态,确定性映射表,LLM 不参与):
研究 / 观察 / 等待确认 / 限制新仓 / 检查减仓 / 人工复核 / 证据不足。映射表按 Doctor 终局文档 §二联合状态表落码(新信息+未定价→研究优先;新信息+过热→不追涨等待窗口;升级+过热→强化跟踪不自动加仓;持续+过热→限制新仓查兑现;持续+不热→只作背景;压缩→检查仓位不预测方向)。**铁律落码:信息钟任何状态变化不重置价格钟与波动钟。**

**P0 三表**(cockpit.db,append-only,建 UPDATE/DELETE 拒绝触发器):
- case:case_id、signal_node、opened_at、available_at、事实摘要、新增事实类型、受益因果链(beneficiaries_detail)、失效条件、run_id、来源引用。首版由 yuantu_buy_signals 自动派生(1 node = 1 case)。
- clock_state:snapshot_date、case_id、ts_code、info_clock、price_clock、vol_clock、执行旗、action、依据摘要。每日一快照,只 INSERT。
- oos_ledger:case_id、ts_code、entry_date(次日执行口径)、horizon(1/3/5/10/20)、raw_ret、bench_ret、excess、成熟标记、样本三分(insample/shadow/oos)、run_id。到期自动成熟,只 INSERT。
