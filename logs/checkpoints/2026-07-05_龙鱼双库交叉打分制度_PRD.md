# PRD · 龙鱼双scorer交叉打分制度（全库 94 全量）

## §一 · 任务目标

把龙鱼五力从"单一评分者拍脑袋判分"升级为**双 scorer 交叉验证制度**：同一标的由两个独立 scorer **独立盲打**，用分歧当信号、触发人工复核校正，既防 CC（自上而下）放水、也防 LLM（自下而上）兜底噪声。

**存储形态（Doctor 2026-07-05 定）**：**不建新目录**，在现有 `Database/龙鱼-标的分析库/` 里**加 `scorer` 维度**区分两套分。DeepSeek 分作为 `scorer=deepseek` 的**新增 analysis 条目**并存，与 `scorer=claude` 条目并列、**互不覆盖**（保盲打独立性）。

- **Claude scorer（自上而下）**：维度级整体判分（reviewer=research-CC / `scorer:"claude"`），偏战略/叙事权重。现有 τ12 的 2026-07-05 分析归此维。
- **DeepSeek scorer（自下而上）**：逐子项打分（加锚点版 prompt）累加成维度分与总分（reviewer=DeepSeek-v4-pro / `scorer:"deepseek"`），偏基本面/纪律。deepseek 条目额外带 `subscores`（六维子项分）。
- **交叉校正机制**：两 scorer 互不看对方独立打分 → 系统算 |diff| → **总分差 ≥15 的标的进"待校正"队列** → CC/Doctor 逐个定夺（分歧不自动合并、不自动收敛成单一"真分"）。
- **客观锚**：财务维（5）两 scorer 都取引擎 `financial_score.breakdown` 真值。

**全库 94 家全量跑**（Doctor 2026-07-05 定：scorer 维度并存不覆盖 CC 分，风险已消除 → 直接全量）。对全部 94 只标的补 deepseek scorer 分 + 对比校正队列。CC 已有 τ12 的 claude 分保留；其余 82 家 claude 分即库内既有历史分（reviewer/日期照旧），deepseek 为新增维度。

**Doctor 原始指令**（逐字引用）：
> "工程升级：做两个库（Deepseek新建库），然后做对比，形成claude"从上到下"，Deepseek"从下到上"，"交叉打分"，对比校正的制度。"
> 机制="独立盲打+分歧触发复核"；范围="先τ12家跑通制度闭环"。
> 存储形态（2026-07-05 追加确认）="同库加个 `scorer` 维度"（不建新目录）。

**对齐结论摘要**（2026-07-05 多轮对齐确认）：
- 机制：独立盲打 + |总分diff|≥15 触发复核；财务维两 scorer=引擎真值。
- 存储：同库 + `scorer` 维度（claude/deepseek 并存，deepseek 条目带 subscores），不建新目录、不覆盖 CC 条目。
- 前期已验证：DeepSeek 无锚点版海光 44.5、加锚点版 50.5（差异化改善），寒武纪 74.0；"参差"主因是 prompt rubric 缺口（已加锚点修复），残留系统性偏严=判分立场差异（DeepSeek 更贴"不放水"纪律）。
- 首期范围：τ12；非首期：其余82家 / 看板双分并排 / 改生产脚本 / 反哺生产 prompt。

**任务规模估算**：
- 预计涉及文件数：schema/制度文档 2 · 脚本 1–2 · τ12 记录改写 12 · 对比产物 1 · 档案更新 2
- 预计耗时：1–2 小时（含 12 家 DeepSeek 打分调用）
- 涉及项目：龙鱼五力（存储在渊图 monorepo `Database/龙鱼-标的分析库/`）
