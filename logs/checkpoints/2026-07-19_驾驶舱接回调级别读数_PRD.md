# PRD · 驾驶舱接入回调级别读数

## §一 · 任务目标

**动机**：回调级别判别器（nowcast L1/L2/L3）已于 2026-07-18 接入烛照九阴日报并验收（9/9，2026-07-19 Doctor 批）。当时明确列为**非交付项**的「接驾驶舱」现由 Doctor 拍板启动。影子驾驶舱看的是**标的级**三时钟，缺一条**市场级**的环境读数——同样是「限制新仓」，发生在 L1 平静期和 L3 急跌期，含义完全不同。

**范围**：在 `longyu-holdings-board` artifact 第二标签页「影子驾驶舱」**顶部**加一条全局状态横幅，显示当前回调级别 + 确认态 + 最近邻 + 同级历史中位。数据经 `get_cockpit_payload` 新增顶层键 `market_grade` 下发。

**Doctor 原始指令**（逐字引用）：
> 「3、起草」（对「级别读数是否接驾驶舱」）
> 「全局横幅·纯展示」（对「级别读数在里面扮什么角色」）

**关键边界（本 PRD 的核心约束）**：级别读数是**市场级**，驾驶舱是**标的级**，粒度不同。故本版**只做展示，不介入任何标的级判断**——不进 `compile_action`、不改三时钟、不做第四只时钟。依据：纪要 §一 将级别判别器定性为**弱信号 risk-overlay、非 alpha**，`auto_trade_allowed: false`；且 L3 事件级样本 n=13、本轮 episode 尚未完结，不具备门控真实行动的成色。

- 预计涉及文件：修改 3（`cockpit_data.py` 加 `market_grade` 组装、`longyu_holdings_mcp.py` 透传、artifact index.html 加横幅），新建 0，新建 DB 0，新建定时任务 0
