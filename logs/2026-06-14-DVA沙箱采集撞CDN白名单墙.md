## 完成的工作

- `/resume` 恢复上下文：定位活跃线为 DVA Update（Cowork 定时任务定时触发但失败，Doctor 上次驳回暂停，任务保持启用）。
- 验证上次悬而未决的「douyin 是否在新会话已通」：`curl -I https://www.douyin.com` 直连与显式走代理**均 HTTP/2 200**——证实上次 403 只是「白名单需新会话才生效」，前提扫清。
- 摸清 `update-all` 执行结构：`handleUpdateAllCommand` 逐作者 spawn `harvest` 子进程；`harvest` = DYD下载(run.py) → seed入库 → ASR(dva_asr.py) → import-transcripts；`--no-analyze` 跳分析。watchlist 6 位（4 full + 2 transcribe-only）。
- 发现 ASR 天然可拆：`dva_asr.py` 有 `--video` 单视频模式，且幂等性强（DB 有 sensevoice-v1 字幕 / txt 已存在即跳过）；`transcribe_video` 内部 `submit_asr_task`(拿 task_id) 与 `poll_asr_task`(每3s轮询、上限300s) **本就分离**——DashScope 异步任务，是绕 45s 的天然切口。
- 实测单条视频下载，逐层撞墙：
  - 墙①（已修）：DYD 自带 SQLite 库在挂载盘报 `disk I/O error`——FUSE/网络挂载不支持 sqlite 文件锁。把 `database_path` 指到 `/tmp` 后过（"Database initialized"）。
  - 墙②（已修）：HOME 必须 `=/sessions/<会话>/mnt`（上次日志含糊写「挂载根」，若指到 `.../Claude` 子目录会 path doubling → 读到 0 作者）。
  - 墙③（**致命·修不了**）：元数据爬取经代理 10s 内 OK，但真正 mp4 字节下载必失败。播放 URL `www.douyin.com/aweme/v1/play` **302 跳到视频 CDN `*.zjcdn.com`**（本次 `v5-dy-o-detect`、`v5-hl-ws-ov-coldy`），这些域经代理一律 `000`（白名单没放）。试 4 个常见 CDN 域 + 真实跳转目标全 000。
- 固化复测脚本 `Projects/DVA/tools/sandbox-cdn-retest.sh`：内置三处沙箱配方（proxy / db_path→/tmp / HOME=mnt），先探针 CDN 域、再计时下载 1 条，供「加白名单后的新会话」一键复测。
