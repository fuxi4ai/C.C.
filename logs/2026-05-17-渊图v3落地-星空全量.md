## 工程沉淀

- **方法论**：LLM 大批量改 schema 时，pilot 50 条 → review → 调 prompt → 全量；avoid "反转推理" 类 prompt 设计
- **schema 设计**：按模态逻辑划分关系类型（充分/必要/负向），enables/requires 反向边对偶不要二选一
- **数据迁移**：双字段（type + relation）schema 不一致时，让一个字段成为权威，工具脚本统一对齐而非维持双轨
- **可视化分类**：行业桶分类用关键词 + 节点 type 优先级（material → 强制 materials_energy），多轮扩词降 other 比例
