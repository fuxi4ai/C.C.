## 做出的决策

| 决策 | 原因 | 影响 |
|------|------|------|
| raw 按核心域/邻域/跨域三分，跨域 hold 不入 canonical | 防 BACKLOG-20260624-001 剂泰科技 bio 子图串入 15 节点重演 | 建立分域准入惯例，`_hold_跨域待定/` 成常设隔离区 |
| 统一以**会议日期**为文件日期 | Doctor 澄清：正文（MMDD）是会议日期，文件名是纪要发布/获取日期，前者语义更准 | 32 篇改名 + 五面同步；但只覆盖 32/276 篇（167 篇正文无日期），无法真正「统一」 |
| provenance 修复与改名**解耦** | LLM 连标题一起重写成「XX-学习材料」，改名后日期对上、标题仍对不上 | 推翻 Doctor 原本「等改名一次到位」的选择，两批分开做 |
| `data_sources.file` 改由**代码强制写入**，而非改 prompt | prompt 是请求、后处理是保证；同文件 cite 路径一直代码直写、从未产生幽灵，可作对照 | 根治入库主链层面的幽灵源头 |
| `technology` 不纳入允许 type 集，改 `concept_` | 这 5 个本质是技术路线，图谱内同类（`concept_LIDETechnology` 等）一律 concept | type 集回到 9 种；与 `equipment_` 2026-06-25 纳入的先例走了相反方向 |
| 帝尔全量归正 `company_DRLaser`，但**产品/概念节点不并进公司** | `product_`/`concept_` 与 `company_` 是不同实体类型，并入即污染 | 4 公司节点合并 + 3 非公司节点仅改名 |
| 幽灵归正强制「候选唯一」，宁可少修 | 归一化后有 2 个候选时机器分不出该配哪个，硬配就是赌 | 实际只修 84 种/2116 处（原估 94/2364），少修 6 种歧义项 |
| 去重一律「信息并入 properties」，不裸删 | 合并方向若判错，原文还在 properties 里可读出重建；裸删则须先知道自己删错了才会去翻 git | 沿 2026-05-23 完整性清理立的规矩 |
