## 做出的决策

| 决策 | 原因 | 影响 |
|------|------|------|
| 市场信号用 _meta tag 而非新 node type | 不动 11 种 schema，灵活多类共存 | 数据层可叠加而不冲突 |
| C 混合方案（规则粗筛 + LLM 精判） | 规则单独覆盖率仅 39%；LLM 单独 215 候选成本可控 | 6 分钟跑完 → 106 信号节点 |
| 三类形状语义差异化（六角/十字/星环）+ 颜色对应（冷/暖/金）| 不只是颜色，形状本身就传递语义 | 视觉一眼能辨别"哪类事件" |
| 星环取消呼吸 | 持续供不应求是"长期状态"非"动态事件"，跟 demand/shock 区分 | 视觉对位："活跃 vs 长期" |
| 搜索别名用 *ds/*ss/*pi 命令式前缀 | 单字简写如 ds/ss/pi 会被普通节点名子串误命中 | `*` 前缀做意图标识，无歧义 |
| 星环 3D 变换顺序：X 透视 → Y 压缩 → Z 旋转 | v3.4-3.7 反向顺序导致 75° 倾斜时戒指畸变 | 修复后旋转/透视/光照在戒指自身坐标系一致 |

## 工程沉淀

- **物理模型 fragment shader**：3D 变换链顺序必须跟物理一致（戒指本平面 → X 透视 → Y 压缩 → Z 旋转）；逆推链反向应用，**视觉问题往往是顺序错误而非角度错误**
- **多层 shape 叠加 vs 单一 type**：复杂语义用独立 layer + 形状差异化 > 把语义压进 type 字段；可独立 toggle 显隐
- **搜索别名表**：用户输入跟内部 ID 不一致时，alias map + 命令式前缀（`*`）比纯子串匹配清晰
- **LLM 批量标记**：pilot-first（50 候选 → 全量）+ 多类输出 + 置信度阈值 → 6 分钟跑完 215 节点准确率 95%+
