- **文档**：最佳实践新增「命名规范」「增量更新与查重」两节；PRD + help 同步；GOTCHAS 追加 INFRA-20260521-002 / 003。

## 做出的决策

| 决策 | 原因 | 影响 |
|------|------|------|
| 命名与寻址键解耦：文件名随意，但末尾恒带 aweme_id | 既要人可读（作者名/标题），又要机器可定位 | import 改为按 id 寻址，换命名不再断链 |
| 幂等/查重锚定 DB 状态而非文件名 | 文件命名易变，DB subtitleSource 是真相源 | 增量更新四环节全程不重复 |
| 孤儿字幕用"恢复 id"而非"重转" | 省 89 条 ASR 费用；附属 json 藏有权威 id | 零成本入库；沉淀"先翻元数据"的方法 |
| 单链接 harvest-links 暂不 seed DB | 定位为快速字幕提取，不污染库 | 单链接视频暂不可 import（需要再加 seed） |
