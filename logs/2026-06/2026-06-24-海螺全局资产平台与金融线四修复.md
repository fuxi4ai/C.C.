## 做出的决策

| 决策 | 理由 |
|------|------|
| 看板内化进海螺姑娘（conch 当审修引擎）·非独立 artifact | conch 本就 dry-run 默认+audit+rollback，天然合规 propose-then-confirm；给「审查并修复」配真后端 |
| 数据=策展 manifest（单一真源）+ survey 回写健康；关系线手工策展 | 上下游关系无法可靠自动检测；状态靠 conch/health 自动 |
| 「审查并修复」=审查→分级方案·不自动改；conch 永远 dry-run 起、--apply 必经 Doctor | 守 propose-then-confirm/可逆铁律 |
| 颜色=类别·外发光=健康度（健康无光） | 一眼分门类、有问题才"亮"；颜色不再背双重含义 |
| conch obsolete_count 纯信息·绝不自动改策展 status | 把知识库正常版本链误判待修=狼来了（渊图 22 旧文件案） |
| 告警走被动（health JSON+看板发光）·主动推送另议 | 纯沙箱可做·零凭证·与看板闭环 |
| 表间一致性判级细化：落后≥2天=fail/1天=warn(stale) | 派生表落后 1 天常是聚合当日步未跑，标红=狼来了 |
| crossval gate：存在但不合格→中止 build；缺失→允许 | 拒绝静默渲染"待补"假象；守「宁留空不造假」铁律 |
