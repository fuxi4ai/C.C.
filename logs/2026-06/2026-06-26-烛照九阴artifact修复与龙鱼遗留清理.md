### B · 烛照九阴 Cowork artifact 全链诊断与修复（本会话核心）
- **症状1 图表空白**：根因 `gen_daily_report.py` L1248 用 CDN 引 echarts（`jsdelivr`），而 Cowork artifact 沙箱只许 Chart.js/Grid.js/Mermaid 走 CDN，其余须内联 → echarts 未定义 → emChart/capGauge 空白。**修**：仿其内联字体/画的做法，新增 `_echarts_inline()` 读 `assets/echarts.min.js` 内联，替换 CDN 标签。
- **症状2 画《果熟来禽图》不见**：逐层排除——base64 解码后 **md5 与原图素材逐字节相同**（966×1495 完整），CSS 全对（`.report-hero` 有 position:relative、`.hero-art` 主规则在、`--laiqin-art` 在 :root）。Doctor 实测：**普通浏览器里画正常显示** → 坐实是 Cowork 渲染器对体积/超大 data URI 的限制（原图 1.73MB→base64 2.3MB，artifact 3.5MB）。**修**：画压成 WebP（保 alpha）646×1000·262KB（原 PNG 保留可逆），生成器 `ART_LAIQIN` 指过去 + mime 按后缀检测，artifact 3.5MB→~1.5MB。Doctor 确认"好了"。
- **生成器自动部署**：原生成器只写 AI4ME、不部署 artifact（"重渲后需手工 update_artifact"是手动缺口，也是"坏了没人盯"的根子之一）。新增 `_deploy_to_artifact()`：套 `cowork-artifact-meta` 头直写 `Artifacts/zhuzhao-jiuyin-daily/index.html`，写前备份、`--no-deploy`/`--output` 跳过、目录不存在告警跳过。一步同步 AI4ME+artifact。
- 临时手工补丁：会话中先用 update_artifact 部署过一版（已被生成器自动部署顶替）。

### C · git 推送排障
- `git push` 报 `curl 52 Empty reply` / `send-pack 意外挂断` → 根因 `http.postBuffer` 默认 1MB 顶不住含 1MB echarts.min.js 的提交。解法 `git config http.postBuffer 524288000`（建议 --global，白泽/行业研究也 vendored 大素材）。
