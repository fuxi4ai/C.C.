## 完成的工作

1. **KV→O MY HTML 对接**：把已入库的弗里曼摄影构图簇（14 卡）提炼转译成 `Projects/O MY HTML/design-ref/visual-eye/构图原理参考.md`——六层语法（框/原则/知觉/词汇/光色/意图与过程），每条「摄影原义→设计转译→落地用法」，附 10 条动手前自检清单；visual-eye README 已登记。
2. **图文卡第一轮**：重抽 PDF images（480 张），精选 12 张入 cooked/images/，7 卡加「图例」。

4. **完好版补齐**：Doctor 另寻完好版 PDF → 校验 99 页全活 → 覆盖 cooked 归档本 → 补抽 11 张图，补齐剩余 7 卡。最终 **14/14 卡全带图例**（22 图，1.4M），健康检查全绿。

## 做出的决策

| 决策 | 原因 | 影响 |
|------|------|------|
| 构图原理参考放 O MY HTML 项目侧而非 KV | KV 管知识资产，项目侧管工作规则；翻译层贴近使用场景 | 以后设计项目直接读此文档，不必回 KV 翻卡 |
| 项目侧引用 KV 卡用「卡名」纯文本不用 wikilink | 跨 vault 链接不解析，避免悬空 | 约定写入对接日志 |
| 卡片嵌图用 markdown 相对路径不用 `[[]]` | KV 健康检查的 `[[]]` 正则会把图片嵌报成未解析 | 写入接手指南铁律区 |
| 图片不全量入库（480张/128M→精选22张/1.4M） | 碎片多、vault 减重；mount 禁 unlink 误拷难删 | cooked/images 只存配卡精选 |
| cooked 归档 PDF 用完好版覆盖 | 归档本应是可重抽的完好原本 | 后续重抽不再踩坏页 |
