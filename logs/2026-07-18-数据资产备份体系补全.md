## 完成的工作

### 一、误诊与真相（起点）
- 剑酒青丘 `git commit` 报 "no changes added to commit"，**CC 连续两次误诊**：先怪 `.gitignore`（查无匹配规则）、再怪 FUSE 挂载可见性（Doctor 实测原生侧文件在）。真相：`git ls-files` 显示**文件早已被跟踪**，`git log` 确认 commit `d8ed770` 已入库——**是 CC 重复发了同一条命令**，第二次自然无事可做。
- 教训：「commit 报 no changes」第一嫌疑应是「已经提交过」，CC 跳过最简解释去追最 exotic 的。

### 二、11 仓远端体检 → 补 5 个裸奔仓
- 扫全部项目仓远端：**5 个无远端**（剑酒青丘 / DVA / 白泽观星 / GlobalPercent / 称象），且 `backup-icloud.sh` 只镜像 `brain/`、不含 `Projects/`，`--exclude='.git/'` 连 brain 的历史也不进镜像。
- **首推前置：全历史密钥扫描**（`git grep` 跨 `rev-list --all`）——5 仓全净；GlobalPercent 一处 `data-token="${m.token_id_yes}"` 是**模板变量插值**、正则误报（预测市场公开 ID，非密钥）。
- `gh repo create --private --source=. --push` 建 5 个私有仓并推送（191/350/66/58/22 objects）。**11 仓现全部有异地版本备份**。
- 剑酒青丘另加 `.fuse_hidden*` 进 `.gitignore`（10 个 FUSE 残留曾被误提交入仓）。

### 三、Database 层：从「全量镜像」收窄到「不可再生档」
- 关键实测：**`Database/` 共 70 GB**，其中 `Douyin/` 66 GB（下载视频）。CC 先前只报了三个 .db 的大小，**给 Doctor 的决策框架是错的**，全量镜像不可行。
- 按**可再生性**重新分层：`Market-Data`(2.4G) tushare 可重拉、`Douyin`/`YouTube` 可重下、117 个 `.bak`(2.0G) 是本地回滚锚异地无意义、`龙鱼-标的分析库/` 自身是 git 仓且有远端 → 全部排除。
- 保留：`recap.db`(4.5M·几个月课件沉淀 API 拉不回) + **`Raw-Recap/`(112M·235 份原始课件＝recap.db 的上游来源)** + `business_breakdown.db`(0.6M)。

### 四、TOS 接入
- 取消 crontab（整条），回滚 `backup-icloud.sh` 中 CC 新加的数据层段（`.bak_20260718-052005` 留档）。
- 工具选型：`tosutil` 不在 homebrew 且其文档页客户端渲染 CC 读不到；**Doctor 无偏好 → CC 定 Python SDK**。写 `brain/.tools/backup-tos.py`：`sqlite3.backup()` 出一致性快照（连 sqlite3 CLI 都不依赖）+ `tos` SDK 上传 + size 比对增量。
- **踩坑**：`TOS_ENDPOINT` 填成控制台的「桶域名」`<bucket>.tos-cn-shanghai.volces.com`，SDK 再拼一次桶名 → `dragonpalace.dragonpalace.tos-...`，报成一堆 SSL EOF / timeout，**伪装成网络故障**。加护栏自动剥离桶名前缀并说人话。
- **首传成功：414 对象 / 115.8 MB / 0 失败**。后续增量只动 ~5MB 数据库。

## 做出的决策

| 决策 | 理由 |
|---|---|
| 5 仓建**私有**远端而非只做本地镜像 | 交易策略(剑酒青丘)/抖音数据(DVA)需异地版本备份但不可公开；`--private` 一个不漏 |
| 首推前必做全历史密钥扫描 | 这 5 仓从没推送过＝从没被审视过；`.gitignore` 挡不住早已 commit 进历史的密钥 |
| 备份作用域按**可再生性**分层，不按目录 | 70GB 里只有 118MB 值得异地；「丢了能不能重建」是唯一判据 |
| Raw-Recap 纳入（112MB） | 它是 recap.db 的**上游**——有它+管线能重建 recap.db，反之不行 |
| 整条 crontab 停，brain 只靠 GitHub | Doctor 定；brain 本就有远端，iCloud 镜像属双保险的第二层 |
| TOS 走 Python SDK（CC 自定） | 去掉 sqlite3 CLI 与外部二进制双重依赖；与 fetch_*.py 同源，将来挂 launchd 顺手 |
| 快照 quick_check 不过就丢弃、保留云端上一版 | **绝不用坏副本覆盖好副本**；Raw-Recap 同步不带 `--delete`，本地误删不传导到云端 |
