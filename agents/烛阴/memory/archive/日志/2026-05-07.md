## 三、改进规则

### 规则 1: 操作幂等性检查
执行任何操作前，先检查是否已经做过：
- 文件上传 → 检查文件是否已存在
- 命令执行 → 检查输出是否已经成功
- 状态查询 → 缓存结果，不要重复查

### 规则 2: 失败快速切换
- 同一命令失败 2 次 → 立即切换方案
- 网络请求超时 1 次 → 切换备选方案
- 不在同一条失败路径上重试超过 2 次

### 规则 3: 进度监控
- 每完成一个 Phase 就汇报进度
- 如果发现自己在做重复的事，立即停下来
- 同一个 tool call 不要连续调用超过 2 次

### 规则 4: 循环检测
如果发现自己连续 3 次输出相同或极相似的内容：
1. 立即停止
2. 检查原因
3. 换方案
