# HEARTBEAT.md — 心跳巡检任务

> **间隔**: 2 小时  
> **最后更新**: 2026-05-14 · 九儿  
> **策略**: 正常静默（HEARTBEAT_OK），异常才主动告警

---

## 巡检清单

### 1. API 连通性（快速检查）
- 执行 `openclaw status`，检查 `model` 字段是否正常
- ⚠️ 仅检查"是否异常"，不做深度 API 调用

### 2. Cron 任务状态
- `cron list` → 检查：
  - 是否有 `consecutiveErrors > 0` 的任务
  - 是否有 `lastRunAtMs` 超过预期间隔 2 倍仍未执行的
  - 是否有 `sessionKey` 不匹配的（对比当前 chat_id）

### 3. 磁盘空间
- `df -h /home` → 使用率 > 85% 告警

### 4. 关键服务
- Gateway 是否在运行（`openclaw gateway status`）

### 5. 文件巡检（轻量）
- `memory/` 当日文件是否已创建（> 12:00 未创建 → 提醒梦境整理）
- `temp/` 文件数 > 10 → 提醒海螺清理

### 6. 回收站自动清理
- 执行 `bash scripts/recycle-clean.sh --auto`
- 超过 7 天的文件自动永久删除
- 清理结果记录到日志（不主动汇报，除非异常）

---

## 汇报规则

| 状态 | 行为 |
|------|------|
| 全部正常 | 回复 `HEARTBEAT_OK`（静默，不发额外消息） |
| 有异常 | 回复告警内容，标注异常项 + 建议操作 |
