### 二 · 沙箱→Tushare 路径打通（本场重点）
- 起：Doctor 要打通沙箱直连 Tushare。我初判「网络 allowlist + Keychain token 两道墙皆在沙箱外、改不了」——**此判错**（见决策/教训）。
- 实探纠偏：沙箱出网必经 `localhost:3128` 代理（`https_proxy` 已设）；经 CONNECT 隧道试 `api.tushare.pro:443` 回 **HTTP 200**——tushare 域名**本在白名单内**。裸 socket 直连才 DNS 失败。无 token 调 `stock_basic` 回 `40101 您上传Token！`→ 证「就差认证」。
- 凭证路线：env 注入在 Cowork 沙箱**不可用**（已知 issue #39125，docs 也无加 env 的口子）→ 改走文件档。引擎 `_load_token` 加 `TUSHARE_TOKEN` env 优先分支（Mac 未设则下沉 Keychain，零影响），但实际落地靠既有 `FF_TOKEN_PATH` 文件档。
- 密钥归置：Doctor 定单独建 `~/Documents/Claude/Env/`（git-free 密钥层，与 Infrastructure 脚本仓分离），token 落 `Claude/Env/tushare.token`(600)；建 `Env/README.md` 立「禁入 git/600/引用非内联/沙箱只读」约定。
- **端到端验通**：`FF_TOKEN_PATH→Claude/Env/tushare.token` + 代理隧道 + 引擎 `_load_token`/`TushareClient` → `stock_basic(600183.SH)` 回 `code 0 · ['600183.SH','生益科技','元器件','广东']`。沙箱可实取 Tushare 数。
