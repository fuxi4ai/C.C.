## 2026-06-03 · 单链接 harvest-links 孤儿字幕统一走 seed+import 正式入库

**背景**：对投知君君跑 harvest 时，`import-transcripts` 递归扫整个 Transcripts 根，连带报出 6 条其他作者（全栈观察员、每日AI评论、独夫之心、矛盾财经史、闭眼Kan世界、黄大圣叨叨叨）的 `无法定位作者`。它们是历史用单链接 `harvest-links.js` 转出来的字幕——按设计单链接只下载+转写+汇总、不 seed DB，故 DB 无视频记录、反查命不中。6 条均有下载 mp4+`_data.json`。

**选项**：
- A：封档止警（`.transcript.*` → `.dupbak`），不入库。
- B：正式 seed+import 入库（本地复刻 `seedDVAFromDYDDir` seed 记录 → Mac 跑 import 注入+重建索引）。

**选择**：B

**理由**：6 条都有完整下载元数据，可正经入库；保留数据可用性优于丢弃。新建 5 个单视频作者可接受。

**影响**：
- DVA 库新增 5 个作者（黄大圣已有记录、并入），各 1 条视频。
- 建议治本：给 `harvest-links.js` 末尾补 seed 步骤，让单链接产物不再成为 import 孤儿。
- 经验固化见 [[经验库]] EXP-20260603-001-P（seed/import 环境分工）。

## 2026-06-05 · 常更作者表 watchlist + 一键批量更新 update-all

**背景**：Doctor 需要一张「需要经常性更新的作者」表，自己手动选作者入表，再用一条命令把全表的视频、字幕、入库、分析全部更新。现有能力已齐：`dva harvest` 做视频+字幕+入库，`dva analyze-level1` 做分析——缺的是「表」和「批量编排」。

**选项**：
- A：watchlist 用 CLI 子命令管理（`dva watch add/remove/list`），存 DB indexes/watchlist.json；批量命令在主进程内直接调 handler。
- B：同 A，但批量命令对每位作者 spawn 子进程跑 harvest/analyze。

**选择**：B

**理由**：`handleHarvestCommand` 内部多处 `process.exit(1)`（下载目录不存在、sec_uid 解析失败等），若在主进程内直接 await，一位作者失败会终止整批。子进程隔离后单个失败只挂掉该子进程，批次继续，并能逐作者汇总成功/失败。

**影响**：
- 新增 `基础模块/watchlist.js` + `dva watch`（add/remove/list/update）+ `dva update-all`（别名 watch update / update-watchlist）。
- 默认每作者抓 20 条新视频（Doctor 选定），`--limit` 可覆盖、作者可设专属 limit，`--no-analyze` 只采集不分析。
- 版本进 v6.6，详见项目 CHANGELOG。

## 2026-06-05 · 分析 LLM 默认切换为 DeepSeek V4 Pro

**背景**：Doctor 指定 DVA 分析步骤改用 DeepSeek V4 Pro。DeepSeek 提供 Anthropic 兼容端点（`https://api.deepseek.com/anthropic`），`llm-client.js` 已走 `@anthropic-ai/sdk` 且透传 BASE_URL / MODEL，故无需改调用代码。

**选项**：
- A：只在 `~/.dva_secrets`（cron）+ shell rc（交互）里设环境变量，代码默认不动。
- B：把代码默认值也改成 DeepSeek，环境变量仅覆盖、唯一必填项缩到 API key。

**选择**：B

**理由**：Doctor 要 DeepSeek 做「标准 API」，覆盖 cron 与手动两条路径。改默认后只剩 key 必填，少一处忘设环境变量的坑；回退原生 Anthropic/APIYI 仍可用三个环境变量覆盖。

**影响**：
- `基础模块/llm-client.js`：MODEL 默认 `deepseek-v4-pro`，BASE_URL 默认 `https://api.deepseek.com/anthropic`。
- `基础模块/dva_config.js`：llm.model 默认同步改，注释更新。
- `run-update-all.sh` 的 SETUP 注释：`~/.dva_secrets` 改为 DeepSeek 三件套。
- ⚠️ ANTHROPIC_API_KEY 现在必须是 **DeepSeek key**；旧 Anthropic/APIYI key 配新默认 base URL 会失败。
- 残留：`3-文本分析/mind-model/mind-extractor.js`。**[2026-06-13 审核更正]** 此前本条记成产物标签 `extractionMethod:'llm-claude-sonnet'`，实为笔误——实际标签是中性的 `'llm-extraction'`（`mind-extractor.js:208`），无需改。真正的 Claude 残留在该文件**头注释**（"改用 Claude claude-sonnet-4-6"）和运行前提里的 `ANTHROPIC_API_KEY`/APIYI 地址，已于审核中改为 DeepSeek/`DVA_LLM_*`。
- DeepSeek 型号经 web 查证（2026-06）：在售为 `deepseek-v4-flash` / `deepseek-v4-pro`；旧 `deepseek-chat`/`deepseek-reasoner` 将于 2026-07-24 弃用。
