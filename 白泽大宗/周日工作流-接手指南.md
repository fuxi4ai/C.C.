## 架构（两半握手 · 沙箱连不上 Tushare/SMM，故拆本机 + Cowork）

```
13:30  Mac 本地 cron   weekly_local_prep.sh（4步+清理+健康行）
  [1/4] Tushare 期货价(P0)  fetch_futures_prices.py     碳酸锂/铜/金/油
  [2/4] SMM h5 现货(P1)     fetch_smm_h5.py --fetch --merge  钨/稀土(无期货品种,沙箱被代理墙故放本机)
  [3/4] 龙鱼 batch(财务+估值) batch_score.py <ts>        → reports/five_forces_data_*
  [4/4] 龙鱼六维 LLM(Deepseek·并行4路) score_with_llm_apiyi.py  → reports/llm_score_*（六维/100）
  + 清理 reports/(每股留最新+近7天) + 落 data/weekly/local_prep_health.json
        ↓ 同 Documents 文件夹握手
14:30  Cowork 定时任务  baize-weekly-report（沙箱 agent）
  Stage0 渊图先验 → Stage1 读health+web补缺价 → Stage1.5 sync_longyu_universe.py ingest 六维/估值
  → Stage2 双面交叉验证(渊图×龙鱼×新闻) → Stage3 build_weekly_report.py → Stage4 present_files 定名版
```

## 凭证（Keychain / Database/.env）

- `tushare_pro`(账户 doctor) — 期货 + 龙鱼引擎共用
- `KG_API_KEY` — Deepseek（六维 LLM，复用渊图凭证，在 `Database/.env`；端点 `https://api.deepseek.com/anthropic`，模型 `deepseek-v4-pro`）
- 缺任一 → 对应步骤跳过、相关字段留『待引擎』/『待核验』，不阻塞

## 铁律（数据真实性）

缺数留空『待核验』、占位禁入；**同比为准·环比参考**；信源分级 P0 期货/P1 权威站/P2 web；web/特气补价须带 `confidence%`(LLM判断) + `asof` + 新鲜度；无现货价品种(AI硬件)走渊图结构先验注记，**不进弹性测算**。名称一致性闸：ts_code 命中但记名≠本公司 → 退『待引擎』，绝不张冠李戴。

## 如何接手（常见动作）

- **改看板样式** → `build_weekly_report.py` 的 `_HTML_TMPL`（勿改产物 HTML，下周重渲覆盖）
- **加信源** → 先查 `docs/大宗信源图谱`，连接器仿 `fetch_smm_h5.py`（config 驱动 + 降级 + 铁律）
- **手动跑一次** → Mac：`bash scripts/automation/weekly_local_prep.sh` → 看 `data/weekly/local_prep_health.json` + `logs/`
- **六维没填** → 多为 ts_code 错/空(核 fundamentals) 或 cron 六维步未跑(缺 KG_API_KEY)
- **出问题先看** → `local_prep_health.json`(各步状态) + `logs/llm_score_<date>.log`
