## 2026-06-25 · LLM 评分通道弃用 APIYI/sonnet → DeepSeek 官方端点

**背景**：发现「默认漂移」——生产周报工作流早已跑 DeepSeek（`api.deepseek.com/anthropic` + `deepseek-v4-pro`，key 走 `.env` 的 `KG_API_KEY`），但独立脚本 `score_with_llm_apiyi.py` 的硬编码默认仍是 `api.apiyi.com` + `claude-sonnet-4-5`，靠周报注入 env 才指向 DeepSeek；Doctor 直接手跑会落回 APIYI/sonnet。本质是把脚本出厂默认对齐到既有生产通道，非高风险切换。

**选项**：
- 默认模型：A `deepseek-v4-pro`（与生产一致）/ B `deepseek-v4-flash`（官方主推·更快更省）
- APIYI 处置：A 软弃用保留可回退 / B 硬清除改名
- DeepSeek 端点：用官方 Anthropic-兼容端点（保留现 /v1/messages 协议）vs 改写 OpenAI 协议

**选择**：默认 `deepseek-v4-pro`；硬清除改名 `score_with_llm.py`（但旧名留薄 shim 不物理删）；端点用 `https://api.deepseek.com/anthropic`，协议不变。

**理由**：
- v4-pro 与生产周报一致、投研质量优先；thinking:disabled 后 ~22s 稳在沙箱 45s bash 限内（GOTCHA-023）。flash 仍可 `--model` 临时覆盖。
- 官方 Anthropic-兼容端点 = 最小改动，现 `/v1/messages` + x-api-key 代码直收，已有 deepseek thinking-disable 分支复用。
- 硬清除是 Doctor 选；但「可逆优先·不删文件」硬约束仍生效，故旧 `score_with_llm_apiyi.py` 改薄 shim（透传 + DeprecationWarning）防周报 SKILL 断档，非物理删。
- DeepSeek 旧模型名 `deepseek-chat`/`deepseek-reasoner` 官方 2026-07-24 弃用，故用 `v4-pro`/`v4-flash` 新名。

**落地**：新建 `score_with_llm.py`（默认 deepseek-v4-pro + 官方端点；key 候选首位 `KG_API_KEY`；`call_apiyi→call_deepseek`；scorer 标签 `DeepSeek/{model}`；APIYI 痕迹清零）；旧名转薄 shim；`白泽大宗/周日工作流-接手指南.md` 引用名更新。

**验证**：沙箱离线自检全绿（语法/导入/默认值/shim 透传 + DeprecationWarning/无 APIYI 残留）。德明利 001309.SZ 端到端实测通过：引擎 leg 9s、DeepSeek leg 22s、key 自动命中 `KG_API_KEY`、scorer = `DeepSeek/deepseek-v4-pro`、六维齐全（总分 43.5/回避）。

**未做**：周报 Cowork SKILL（`Scheduled/baize-weekly-report/SKILL.md` 第 25 行）调用名改 `score_with_llm.py`（非紧急，shim 兜底）；改完撤 shim。两 git 仓提交（渊图 monorepo + brain）。
