# 三灵导出全面审查

> 范围：`4AI/_待审导出/` 下 白泽 / 句芒 / 烛阴 全部包 + 数据库。已全解压核对。

## 一、三灵画像（按导出实物）

- **白泽 · 预测**：观星（中国宏观 v3）+ 大宗（商品弹性）。宏观引擎已归位公共层（见前序）。
- **烛阴 · 复盘**：`烛照九阴` —— A股情绪周期决策工具，四维度（外围定价/行业主线/情绪技术/交易计划），128 复盘记录、双轨一致率 93.2%、121 回测信号 54.5% 胜率、**自带错题本**（10条9改）。成熟、自省。
- **句芒 · 技术（工具匠）**：九大模块——量化脚本集、mootdx 行情、自动深研、共动检测、龙宫梦境、**冥想审计**、**项目审核**、conch海螺、tushare-pro，外加 market-data 公共库。月兔×句芒共维。

## 三、跨灵重复与角色重叠

- **行情/数据访问**：tushare 在 大宗（7+脚本）、句芒（query_tushare_pro ×2处）、白泽数据库全集 各自重写；market_data 句芒/烛阴各一份。→ 应抽**公共数据层**（单一行情库 + 单一 tushare 取数模块），三灵只读共用。
- **龙宫/梦境**：句芒 `05-dream-龙宫梦境`、烛阴 `dragon-palace`、白泽 `数据库全集/梦境状态` 三处都有。→ 归属/用途待你说明，疑似跨灵"状态"机制。
- **conch 海螺**：句芒 `08-conch` 与 brain 的 `海螺姑娘` 项目、已发布 Conch skill 三处并存。→ 宜认 brain/skill 为正源，句芒副本退役。
- **审查/复盘双线（重要）**：句芒 `07-project-review` = **代码/工程审核**（PRD对齐+代码质量+精神洞察三层，"审核者subagent+数据生成器"架构）；烛阴 = **市场复盘**。两者是不同域的"review"。你要的**审查者 subagent，引擎现成就在句芒 project-review**——不论谁戴帽子，都应复用它。
- **冥想审计（对迁移极有用）**：句芒 `06-meditation` 专审 `SOUL.md/MEMORY.md/AGENTS.md/INDEX.md` 的逻辑冲突与失链。→ 正好可做**迁移后各灵性格档案/记忆层的守护工具**。

## 四、安全

- **🔴 P0**：`白泽大宗/.secrets/tushare_token.env` 明文 token **在新导出里仍在**。`.gitignore` 本含 `.secrets/`，是**打包没遵守忽略规则**。请：① 轮换 token；② 导出改用 `git archive` 或 `--exclude .secrets`。
- **✅ 更正**：`feishu-config.json` 其实**干净**（appSecret 走环境变量 `${...}`，非硬编码）。前次列入泄漏是误报，特此纠正。

## 五、建议（承"观星引擎"先例，纳入迁移）

1. **公共数据层**：建单一 `market_data` 行情库（消灭句芒/烛阴漂移双份）+ 单一 tushare 取数模块；三灵只读共用。渊图 `knowledge_graph` 归渊图，烛阴按引用只读、不再拷贝。
2. **归属裁定**：句芒 = 工具/数据/审核基础设施所有者；行情库、tushare、project-review、meditation 归句芒维护，余灵调用（仿"大宗→观星"）。
3. **审查者 subagent**：以句芒 `project-review` 为引擎实现；烛阴专司市场复盘，二者并行不混。
4. **记忆守护**：迁移后用句芒 `meditation` 定期体检各灵 `性格档案.md / 记忆层`，防漂移失链。
5. **去重单源**：白泽自带库的"项目内＋数据库全集"双份合一；烛阴 db-review 明确为"只读快照/复盘用"，不作可写源。
