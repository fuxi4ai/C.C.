# 数据库 ↔ 数据源 API ↔ 环境变量 对应性

## 🔴 需要处置的"阿里依赖没断干净"

1. **百炼（BAILIAN）LLM 依赖**——`BAILIAN_API_KEY/ENDPOINT/DEFAULT_MODEL`（qwen）出现在：烛阴 news 清洗（caixin_bridge/gangtise_bridge）、recap 小宝抽取（xiaobao_extractor）、句芒 project-review 的 `llm_audit_engine`。
   → 这是阿里云 LLM，本地应**remap 到 Claude**（subagent 自带 model），别再依赖百炼额度。
2. **财新 / 冈底斯（GTS）桥**——news.db 的两个数据源是 bridge 服务（`CAIXIN_BRIDGE_PORT`/`GTS_BRIDGE_PORT`），原跑在阿里 host。本地若没这两个桥，**news.db 灌不进新数据**（存量仍可读）。→ 需决定：本地重搭桥 / 换数据源 / 暂停新闻线。
3. **火山 TOS（`TOS_*` 5 个变量）**——原"共享库走对象存储"的传输层。✅ **已随公共数据层本地化而退役**，这些变量不再需要。

## ✅ 本地要配齐的环境变量（金融线跑起来的最小集）
- `TUSHARE_TOKEN`（几乎所有库都要；**旧 token 已泄漏，请轮换后再配**）
- `FRED_API_KEY`（观星宏观预测）
- 选配：`NOTION_TOKEN`+`DATABASE_ID`（大宗对外同步）、`COMMODITY_CSV`（大宗价格输入）
- 待决：`CAIXIN_*`/`GTS_*`（财新/冈底斯桥，依上条 2）

## 建议落点
统一在 `Database/.env`（或各项目 `.secrets/`，入 .gitignore）集中配置；取数仍统一走句芒维护的公共 tushare 模块写入 Market-Data。
