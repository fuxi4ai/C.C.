# 公共预测层落地记录

## 〇、归属决策（2026-06-02）
**宏观引擎归白泽观星**。引擎为观星资产；大宗等数灵如需宏观/Fed 信号，作为消费方调用（**大宗 → 观星**）。这也预演了迁移后 subagent 间的调用关系：大宗 subagent 向观星 subagent 取预测。
- 正式位置：**`Documents/Claude/Projects/Financial/白泽观星/engine/guanxing_engine/`**
- 中立 `_shared/guanxing_engine/` 版已废弃（留 `_SUPERSEDED` 说明，待 Finder 手删）。

## 一、成果
观星宏观预测引擎已落地并验证（v1.0.0，纯相对导入、零 sys.path 污染）。
一包两机制两模型，按 `model` 分派，**忠实复用各自原实现、零数值行为漂移**：
| model | 机制 | 标的 | 默认配置 |
|---|---|---|---|
| `china_macro` | 阈值权重累加（观星 v3 `GuanxingV3Engine`） | 中国：降息/衰退/通胀 | `configs/models/china_macro_v3.yaml` |
| `us_fed` | 基准+系数（大宗 `GuanxingPredictor`） | 美联储：加息/降息/持平 | `configs/extreme/us_fed_v1.yaml` |

用法：`import guanxing_engine as ge; ge.predict('us_fed', data)` / `ge.predict('china_macro', data)`。

## 二、验证（全绿/可复核）
- 隔离 import：两模型均正常返回。
- 中国 v3 测试：**17/17 全过**（行为与观星原项目一致）。
- Fed 测试：6 过 5 败——**与大宗原项目画像完全一致**，5 败均为既有问题（p_hold 设计缺口 + 回测校准/缺 normal 配置），非本次搬迁所致。
- 大宗 repoint 复验：把本地 `src/core/predictor.py` 换成指向公共包的薄 shim 后，`from core.predictor import GuanxingPredictor`、`StargazerPredictor` 别名、`predict()` 均正常，商品主流程不受影响。

## 三、过程中修正的两个真实缺陷（审查即纠偏）
1. **包内模块名冲突**：初版把 `core`/`engine_cn` 经 sys.path 暴露，与消费方 `core` 撞名致循环导入。改为**纯相对导入**（顶层 `engine_cn.py` + 子包 `_fed/`），从根消除。
2. **认知更正**：先前以为"两份 predictor 是同一引擎的重复副本"。实查为——观星已演进到 v3（中国宏观），大宗留着旧 Fed 引擎，**是不同模型、共享架构**，且**打分机制不同**。故公共层按"双机制双配置"实现，而非合并成一个公式。

## 四、消费方接入（待两项目落地 Projects/ 时执行）
- 把 PYTHONPATH 指到 `Projects/Financial/白泽观星/engine/`（或 `pip install -e`）。
- 观星（引擎所有者）：引擎即其自有资产，本地直接使用；旧 `src/engine.py` 收敛进本包。
- 大宗（消费方，大宗→观星）：删本地 `src/core/predictor.py`，用 shim 或直接 `from guanxing_engine import GuanxingPredictor`；call site `fed_prediction.py` 已按数据类读取（无需改）。
- shim 范本见本目录 `大宗_predictor_shim范本.py`（已实测：大宗经 shim 调用观星引擎成功）。

## 五、遗留（需你拍板）
- **`p_hold`**：两模型均无持平概率，policy 却可输出 'hold'；强行令三者和=100 可能为负。属设计缺口，交观星定语义。
- **Documents 旧 `src/` 目录**：沙箱不能删挂载盘文件，已留 `_DEPRECATED_可删除.txt`；请在 Finder 手动删除 `guanxing_engine/src/`。
- **大宗 normal 周期配置**（`configs/normal/v1.1_normal.yaml`）未导出，致部分回测用例报 FileNotFound——需要时补导出。
